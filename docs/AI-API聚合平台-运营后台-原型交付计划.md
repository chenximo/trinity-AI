## 2.1 列表与工具栏 UI 约定（`@trinity/ui` · Excel）

本节为 **运营后台** 内 **「供应商」**（§4.4）、**「模型管理」**（§4.5）及同类 **主数据 / 配置列表** 的 **统一原型与工程约定**。

### 2.1.0 Element Plus 列表（若依式 · 优先）

**`apps/trinity-ai-admin`** 内凡 **检索 + `el-table` + 分页** 的页面（API 密钥、计费、风控、用户、权限、系统等），以 **[`apps/trinity-ai-admin/doc/运营后台-若依式列表规范.md`](../apps/trinity-ai-admin/doc/运营后台-若依式列表规范.md)** 为 **唯一真源**；参考实现 **`admin-keys/KeysPage.vue`**。

| 要点 | 约定 |
|------|------|
| 工具栏 | `AdminSectionHead` **`toolbar-only`** + `AdminListQuery` |
| 表格 | `admin-ep-table-wrap`；数据列仅 `min-width`；**不写** `align` |
| 操作列 | `ADMIN_TABLE_COL_OPS` + `admin-ep-row-actions` + `link` 按钮与图标 |
| 按钮间距 | 工具栏左/右 `#actions` 与操作列统一 **`0.5rem`**；见若依规范 **§2.3** |

供应商 / 模型等若仍使用 **`@trinity/ui`** 药丸筛选（§2.1.2），列表 **表格与操作列** 仍须满足上表；新页默认 **Element Plus 全套**（`el-select` 筛选），与密钥页一致。

### 2.1.1 工具栏布局（一行 · 左右分区 · `@trinity/ui` 画板对齐）

> 适用于仍走 **SearchForm1Fixed + FilterForm2** 的模块；**Element Plus 列表页见 §2.1.0**。

与 **`apps/trinity-design`** 中 **`/design-spec`**（`#ds-search`、`#ds-filters`）对齐，业务控件从 **`@trinity/ui`** 引入。

| 区域 | 内容 | 组件与说明 |
|------|------|------------|
| **左侧** | **搜索** | **`SearchForm1Fixed`**（Vue 中常用导出别名 **`TSearchForm1Fixed`**）：**固定宽度**搜索（默认 `width="17.5rem"`，可按页微调），占位与视觉与设计规范 **形式 1 · 固定宽度** 一致；**不得**用无约束宽度的普通 `input` 顶替。 |
| **右侧**（与左侧 **同一行**、垂直居中齐平） | **筛选** + **主操作** + **导入 / 导出** | 使用 flex：右侧容器 **`margin-left: auto`**，内部 **`gap`** 排列。主操作示例：**「新增供应商」**、**「新增对接配置」** 等，用 **`TButton`**（主操作用 `variant="gradient"`，次要操作用 `outline`）。 |

**工程参考**：`apps/trinity-ai-admin/src/views/admin-suppliers/` 中 **`SuppliersPage.vue`** + **`suppliers.css`**（`.sup-toolbar-row`、`.sup-toolbar-row__start`、`.sup-toolbar-row__end`）。

### 2.1.2 筛选组件（全后台统一）

- **所有筛选**（状态、范围、类型等）一律使用 **`FilterForm2PillListbox`**（药丸触发 + listbox + ✓）。
- **推荐用法**：`managed-panel` + `v-model:open` + `:items`（`FilterForm2ListboxItem[]`，含 `checked`）+ `@select` 更新业务状态；**禁止**在运营后台列表工具栏使用原生 **`<select>`** 作为筛选主形态；**不**用 **`FilterForm3LabeledSegmented`** 承担后台列表筛选（形式 3 保留给设计画板或其它场景，与本后台约定区分）。

### 2.1.3 供应商 / 模型相关列表 · 产品能力（CRUD + 导入 + 导出 Excel）

以下能力为 **列表页 / 配置子列表** 的 **交付目标**（原型阶段可 mock、`localStorage`、占位弹窗；正式版接 API 与权限审计）。

| 能力 | 说明 |
|------|------|
| **增** | 「新增…」主按钮打开表单 / 抽屉 / 弹窗，写入一条配置或主数据。 |
| **删** | 行级 **删除**（确认后移除；原型可同步 `localStorage`）。 |
| **改** | 行级 **编辑** 进入同一套表单或侧栏编辑。 |
| **查** | 列表展示 + **搜索** + **筛选** +（正式版）分页与排序。 |
| **导入** | **导入** 按钮：支持模板文件（如 CSV / XLSX）；无 UI 库「文件选择」原子件时可用隐藏 **`input type="file"`** + **`TButton`** 触发。 |
| **导出** | **导出 Excel**：目标格式 **`.xlsx`**（工程侧可用 SheetJS 等；原型可占位提示「导出当前筛选结果」）。 |

**适用范围（与设计 §4 对齐）**：**§4.4 供应商**（供应商列表、对接配置列表等）、**§4.5 模型管理**（模型列表、供应线路、主数据等以表格 + 工具栏呈现的页面）均应按上表规划工具栏与 CRUD / 导入 / 导出。

### 2.1.4 导出与时间范围（分档，与详设 §2.3 一致）

列表工具栏若提供 **导出**（CSV / Excel / 审计包等），导出语义默认为 **「当前筛选结果」**。是否须提供 **日期范围选择器**（`AdminDateRangePicker` + 快捷项）按数据性质分档，**非**「凡有导出必加」：

| 档位 | 是否加时间范围 | 典型页面 |
|------|----------------|----------|
| **必加** | 是；重置须清空时间范围 | 用量明细、操作审计、密钥审计轨迹、报表中心·导出任务 |
| **建议加** | 是（按申请日 / 账期） | 数据导出审批、账单（按客户导出对账） |
| **不必加** | 否 | 供应商列表、模型列表（主数据快照导出） |

工程参考：`apps/trinity-ai-admin/src/utils/adminDateRange.ts`（`isWithinAdminDateRange`、`isBillingPeriodWithinAdminDateRange`）。

### 2.1.5 列表分页（全后台统一）

- 每个 **`el-table` 数据列表**（含工作台待办表）表格下方须有 **`AdminTablePagination`**（`el-pagination`：总数、每页条数 10/20/50/100、上一页/下一页、跳转）。
- 原型阶段对 **已筛选结果** 做前端切片（`useAdminTablePagination`）；**筛选 / 重置** 后回到第 1 页；正式版改为 API `page` / `pageSize` / `total`。
- **非列表** 页面（实时大盘图表、文档编辑器等）不适用。
