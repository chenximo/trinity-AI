## 1. 约定

| 谁 | 改哪里 |
|----|--------|
| **产品经理** | `Trinity/apps/`、`packages/`、`docs/` |
| **工程师** | `TrinityAI-web/apps/` only |
| **原型镜像** | `TrinityAI-web/apps_ui/` — PM 同步写入，工程师**只读** |
| **提交** | 工程仓禁止同一 commit 含 `apps/` 与 `apps_ui/` |

**平时**：只在 Trinity 改、正常 commit，不必每次同步。  
**要交给工程师时**：产品经理走 §2；工程师走 [如何消费原型.md](./如何消费原型.md)。

---

## 2. 产品经理 · 交付步骤

```text
五件套齐全 → README 写路由 + 映射 → 同步 apps_ui → 工程仓只 commit apps_ui → push
```

**模块 README**：按五件套规范 **§6.2**（`## 工程对齐` 必填；迭代可加 `## 本次变更`）。

**交付前自检**：五件套 · README 路由/映射 · 已同步 `apps_ui` · commit 无 `apps/` · 已 push。

```bash
# 在 TrinityAI-web
git add apps_ui/
git commit -m "proto: sync 2026-05-20"
git push
```

通知工程师 `git pull`。
