# 用户控制台 · 布局与样式规范

> **总览（必读）**：[用户后台管理风格统一规范.md](./用户后台管理风格统一规范.md)  
> **工程真源**：[`apps/trinity-ai/src/views/account/`](../../apps/trinity-ai/src/views/account/README.md)（`ConsolePage.vue`、`account.css`）  
> **静态对照**：`TrinityAI/account/console.html`  
> **规范打样**：`/user-console-spec`（`console-sample/`，引用 `account.css`）  
> **完整原型**：`/trinity-ai/account/console`（`ConsolePage.vue`）  
> **与运营后台分轨**：用户侧用 `@trinity/ui` + `trinity-base`（`or-*` / `.btn`），**不用** Element Plus 若依列表壳。

## 1. 信息架构

| 层级 | 职责 | DOM / 路由 |
|------|------|------------|
| 全站顶栏 | 品牌、主导航、登录态 | `TrinityAiShellLayout` · `header.or-inject`（产品内 `/trinity-ai/*`） |
| 左栏导航 | **API 管理**（密钥、角色）+ 账户 + 产品外链 | `.or-shell > aside.or-side` · hash `#keys` `#preset` `#credits` … |
| 主列 | 五区面板 + 模态 | `.or-shell > .or-main` · `[data-or-panel]` |
| 壳主列 | 占满顶栏下高度 | `main#main` · `.account-console-root` |

五区 hash（与 `ACCOUNT_CONSOLE_HASH` 一致）：

| hash | 面板 | 说明 |
|------|------|------|
| `#keys` | API 密钥 | 默认区；形式 2 筛选、主渐变「创建密钥」 |
| `#preset` | 角色管理 | Preset / 角色弹层 |
| `#credits` | 额度 | 原静态 `billing.html` 并入 |
| `#activity` | 活动 | 活动记录 |
| `#logs` | 用量 | 用量明细 |

## 2. 布局规则（OpenRouter 式控制台）

实现真源：**`account.css`**（勿在 `ConsolePage.vue` 内联大块 layout）。

| 规则 | 说明 |
|------|------|
| 无整页页脚 | 不渲染 `page-foot`；遗留节点 `display: none` |
| 左栏贯通 | 桌面 `.or-side` **220px**，`align-self: stretch`，底 `--surface`，右 `1px` 分割 |
| 主列底 | `.or-main` 底 **`var(--bg)`**，与侧栏 `--surface` 成对 |
| 主列留白 | `padding: 0.75rem 1.25rem 2rem 1.75rem`（窄屏跟 `--page-gutter`） |
| 壳主列 | `#main` 在含 `.account-console-root` 时 **`padding: 0`**，纵向 flex 占满剩余高度 |
| 窄屏 | `<900px` 侧栏改顶栏横条 + 底部分隔线 |

## 3. 样式与 token

| 类别 | 类名 / 组件 | 颜色与高亮 |
|------|-------------|------------|
| 主 CTA | `.btn.btn-gradient` | 底 `var(--grad)`，悬停 `var(--grad-hover)`，字 `#fff` |
| 次要按钮 | `.btn`、`.or-btn-outline` | 灰边 + `--muted`；悬停 `--surface` / `--border-strong` |
| 筛选形式 2 | `.or-app-filter-dd-*` | 药丸 + listbox；选中 ✓；聚焦 `--blue` + `--blue-ring` |
| 搜索 | `.or-keys-search` / `.orc-*-search` | 药丸或矩形见区段；高 `var(--ctrl-h)` |
| 侧栏当前项 | `.or-side a.is-active` | `--surface-2` 底 + `font-weight: 600` |
| 侧栏悬停 | `.or-side a:hover` | `--surface-2` 浅底 |

Token 真源：`packages/tokens/src/core.css` → `assets/trinity-base.css`。改色先改 token，再在 `account.css` 补控制台特例。

## 4. 页面内结构（以 API 密钥区为例）

```
.or-crumb                    ← 面包屑
.or-keys-pagehead
  .or-keys-title-row         ← 标题 + 右侧工具（筛选 + 主按钮）
  .or-keys-lead-row          ← 说明 + details 帮助
.or-keys-table / 列表区
.or-modal-root               ← 弹窗（or-modal-head / 底栏纯文字按钮）
```

| 区域 | 约定 |
|------|------|
| 标题行右侧 | 筛选在左（**形式 2 下拉** `or-app-filter-dd-*`），**主操作在最右**（`btn-gradient`） |
| 表格操作 | 文字链 / 线框，非 EP `link` 按钮 |
| 弹窗底栏 | 主/次按钮**仅文字**，不加图标（与 `design-spec` 弹窗一致） |

## 5. 与运营后台（`/admin-ops-spec`）对比

| 维度 | 用户控制台（Account） | 运营后台 |
|------|----------------------|----------|
| 框架 | 单页 + hash 五区 | 多模块路由 + 侧栏菜单树 |
| 组件 | `@trinity/ui`、`trinity-base` | Element Plus + `admin-shell` |
| 列表 | `or-*` 表格 / 静态表 + 自定义交互 | `el-table` + `AdminListQuery` |
| 主按钮 | `.btn-gradient` | `el-button type="primary"` |
| 样式入口 | `account.css` | `admin-theme.css`、`admin-element-plus.css` |

## 6. 开发检查清单

- [ ] 新面板挂在 `.or-main` 内，带 `data-or-panel` 与侧栏 `href="#id"` 对齐  
- [ ] 左栏 / 主列底色仅用 `--surface` / `--bg`，禁止魔法 hex  
- [ ] 主路径按钮用 `.btn-gradient`，次要操作用线框或文字链  
- [ ] 形式 2 下拉使用 `or-app-filter-dd-wrap` + `adm-form2-dd.js`（见 `accountInteractions.ts`）  
- [ ] 不在用户控制台引入 `admin-shell` / `el-table` 若依样式
