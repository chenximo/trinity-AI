# 用户后台管理 · 风格统一规范

> **版本**：2026-05（与 Trinity AI `account` 控制台对齐）  
> **读者**：产品、设计、前端原型与 AI 云用户后台接入方  
> **母版工程**：[`apps/trinity-ai/src/views/account/`](../../apps/trinity-ai/src/views/account/README.md)  
> **打样站**：开发枢纽 [`/user-console-spec`](http://localhost:5173/user-console-spec)（`apps/trinity-design/src/views/user-admin-system/`）  
> **全站原子**：[`/design-spec`](http://localhost:5173/design-spec) + [`packages/tokens`](../../packages/tokens/src/core.css)

---

## 1. 定位：用户后台 ≠ 运营后台

Trinity 仓库内存在**两条独立的管理端 UI 轨道**，不可混用类名、组件库或打样入口。

| 轨道 | 使用者 | 典型产品 | 技术气质 | 文档 | 打样 / 真源 |
|------|--------|----------|----------|------|-------------|
| **用户后台** | 租户、开发者、客户自助 | Trinity AI 账户控制台；**AI 云用户后台**（规划） | OpenRouter 式控制台 · 单页 hash 分区 | **本文 + `docs/03`** | `@account`、`/user-console-spec` |
| **运营后台** | 平台内部运营、风控、商务 | `trinity-ai-admin` | Element Plus · 若依式列表 · `admin-shell` | `docs/02` | `trinity-ai-admin`、`/admin-ops-spec` |

**统一风格的对象**是**用户后台这一条线**：今后 AI 云用户后台、以及 Trinity AI 内 `account` 的迭代，均遵循本文；**不**以 `trinity-ai-admin` 的 `el-table` / `AdminListQuery` 为母版。

---

## 2. 设计目标

1. **一套视觉语言**：色板、主按钮渐变、药丸筛选、弹窗壳、表格密度在各产品用户后台一致。  
2. **一套布局骨架**：产品顶栏 + 控制台左栏 + 主列内容区；无整页页脚。  
3. **一套组件来源**：门户级原子以 `@trinity/ui` 与 `/design-spec` 为准；控制台布局以 `account.css` 为准。  
4. **可复制的工程结构**：新产品（如 AI 云）按 **account 五件套** 落目录，只改 IA、文案与 Mock，不改 DOM 语义与 token 规则。

## 4. 信息架构

### 4.1 层级结构

```
产品站点壳（如 TrinityAiShellLayout）
└─ header.or-inject          ← 品牌、主导航、登录态、账户菜单
└─ main#main
   └─ .account-console-root  ← 控制台根（account.css 约束 #main padding）
      └─ .or-shell
         ├─ aside.or-side     ← 控制台左栏（工作区 / 账户 / 可选产品外链）
         └─ .or-main         ← 主列：多面板 + 模态根
```

### 4.2 控制台分区（Trinity AI 现行）

侧栏分组与 `href` hash 须与 `ACCOUNT_CONSOLE_HASH` 一致：

| 侧栏分组 | hash | 面板职责 |
|----------|------|----------|
| **API 管理**（工作区） | `#keys` | API 密钥：筛选、搜索、表格、创建/编辑弹窗 |
| | `#preset` | 角色管理（Preset） |
| **账户** | `#credits` | 额度 / 账单 |
| | `#activity` | 活动 |
| | `#logs` | 用量 |
| **产品**（可选） | — | `RouterLink` 回前台（模型、对话、文档等） |

**AI 云** 可调整侧栏文案与模块数量，但须保留：

- 单页 + `data-or-panel` + hash 切换（或等价 Vue 路由，DOM 类名不变）；  
- 左栏 `.or-dash-nav` + `.is-active`；  
- 主列 `.or-*-pagehead` 标题行结构。

### 4.3 路由约定

- **控制台主路由**：一个路由级 Vue（如 `ConsolePage.vue`），对应 `account/console`。  
- **旧路径**：`account/keys`、`account/billing` 等 **redirect** 到 `console` + 对应 hash，避免双轨维护。  
- **登录 / 注册**：归产品首页 hash 或独立页，**不**在控制台壳内重复造登录布局。

## 5. 布局规范

实现真源：**`account.css`**。禁止在入口 Vue 内联大段 layout。

| 规则 | 说明 |
|------|------|
| 无整页页脚 | 不渲染 `page-foot`；遗留节点在壳内 `display: none` |
| 左栏宽度 | 桌面约 **220px**（`flex: 0 0 220px`），与主列同高 `stretch` |
| 左栏背景 | `var(--surface)`，右边线 `var(--border)` |
| 主列背景 | `var(--bg)`，与侧栏形成对比 |
| 主列内边距 | 桌面约 `0.75rem 1.25rem 2rem 1.75rem`；窄屏跟 `--page-gutter` |
| 壳主列 `#main` | 含 `.account-console-root` 时 **`padding: 0`**，纵向 flex 占满顶栏下剩余高度 |
| 窄屏 | `<900px`：侧栏改为顶区横条 + 底部分隔，主列单独滚动 |

细则见：[用户控制台-布局与样式规范.md](./用户控制台-布局与样式规范.md)。

## 6. 样式与 Token

### 6.1 颜色与渐变

所有语义色以 **`packages/tokens/src/core.css`** 为准，业务 CSS 写 **`var(--*)`**，禁止新增魔法 hex（侧栏等控制台特例在 `account.css` 登记）。

| Token | 典型用途 |
|-------|----------|
| `--bg` / `--surface` | 主列底 / 侧栏底 |
| `--border` / `--border-strong` | 分割、线框按钮描边 |
| `--text` / `--muted` | 正文 / 次要文案 |
| `--blue` / `--blue-soft` / `--blue-ring` | 链接、选中、聚焦外环 |
| `--grad` / `--grad-hover` | **主 CTA** 渐变底 |
| `--danger*` | 删除、危险操作 |

### 6.2 控件高度与圆角

- 主按钮、药丸筛选、搜索框高度：**`var(--ctrl-h)`**（36px 量级）。  
- 常规按钮圆角：**`var(--radius)`**；顶栏登录等胶囊：**`border-radius: 9999px`**（见 design-spec）。

### 6.3 样式依赖链

```
packages/tokens (core.css)
    → assets/trinity-base.css（全站）
        → account.css（控制台布局 + or-keys-* 等）
        → 模块内 *.css（仅本模块增量）
```

用户后台页面**不**引入 `admin-theme.css`、`admin-element-plus.css`、`admin-ruoyi.css`。

## 7. 原子组件与 `/design-spec`

用户后台**不使用** Element Plus 业务组件；与门户、对话页共用 **`@trinity/ui`** 与 **design-spec 冻结类名**。

| 场景 | 推荐 | 规范参考 |
|------|------|----------|
| 主操作 | `.btn.btn-gradient` 或 `<TButton variant="gradient">` | design-spec §按钮 · 主要·渐变 |
| 次要 / 线框 | `.btn`、`.or-btn-outline` | design-spec §次要·线框 |
| 表格行内 / 跳转 | 文字链、线框小按钮 | 非 `el-button link` |
| 筛选（控制台） | **形式 2**：`or-app-filter-dd-wrap` + `or-app-filter-dd-item` | design-spec §筛选 · 形式 2 |
| 搜索 | `@trinity/ui` **`SearchForm1Fixed`** | 用户后台主列/toolbar：**形式1 固定宽**（默认 `17.5rem`）；勿用药丸 `SearchForm2Grow` |
| 弹窗 | `or-modal-root` + `or-modal-head` + 底栏纯文字按钮 | design-spec §弹窗；可用 `ModalPanel`（`@trinity/ui`） |
| 对内说明 ⓘ | `details` 或 `InternalHelpTip` | 不占主排版 |

**原则**：新页优先 `import { TButton, ModalPanel, FilterForm2PillListbox, … } from "@trinity/ui"`；历史页面保留 `or-*` / `.btn` 类名时，视觉须与 design-spec 一致，避免第三套按钮。

## 8. 按钮规范

### 8.1 语义分级

| 级别 | 类名 / 组件 | 用于 |
|------|-------------|------|
| 主路径 | `.btn.btn-gradient` | 创建密钥、充值、保存（主流程） |
| 次要 | `.btn`、`.or-btn-outline` | 导出、取消、低优先级操作 |
| 危险 | 线框 + danger 色或 `is-danger` | 删除、吊销（须二次确认弹窗） |
| 导航 | 侧栏 `.or-dash-nav`、面包屑链接 | 不用渐变按钮 |

### 8.2 默认与悬停

| 类型 | 默认 | 悬停 / 聚焦 |
|------|------|-------------|
| 主渐变 | `background: var(--grad)`，白字 | `var(--grad-hover)` |
| 线框 | 灰边、`--muted` 字 | `--surface` 底、`--border-strong` |
| 主渐变禁用 | `opacity` 降低 | 须文案说明不可点原因 |

定义见 `assets/trinity-base.css`（`.btn` / `.btn-gradient`）与 `/design-spec` 四态示意。

### 8.3 弹窗底栏

- 主/次按钮**仅文字**，**不加**图标（与全站弹窗约定一致）。  
- 危险确认可用红色实心或规范中冻结的危险按钮类。

## 9. 页面内结构（列表型面板）

以 **API 密钥** 为范本，其它面板（额度、活动、用量）沿用同一「页头 + 工具 + 内容区」节奏。

```
.or-crumb                         面包屑
.or-{module}-pagehead
  .or-{module}-title-row           左：h1.or-page-title  右：筛选 + 主按钮
  .or-{module}-lead-row            说明文案 + optional details 帮助
.or-{module}-toolbar               搜索条（如有）
.table-wrap / .or-{module}-table   数据表（HTML table，非 el-table）
.or-{module}-summary              条数摘要（可选）
.or-modal-root                    全屏/居中弹层（按需）
```

| 区域 | 约定 |
|------|------|
| 标题行右侧 | **左**：形式 2 筛选等；**右**：主渐变按钮（如「创建密钥」） |
| 表格 | 脱敏列用 `code.or-keys-mask`；操作列可用 ⋯ 菜单或文字链 |
| 数据列 | 左对齐；日期、金额列可用 `or-keys-col-center` 等已有类 |

## 10. 工程结构（五件套）

每个产品的用户控制台模块宜采用与 [`account/`](../../apps/trinity-ai/src/views/account/) 相同的五件套（见原型模块规范 §7.1）：

| 文件 | 职责 |
|------|------|
| `ConsolePage.vue`（或 `XxxConsolePage.vue`） | 路由入口 + 整页 template |
| `account.css` / `{product}-console.css` | 布局与模块样式（可首版复制 account 再改前缀） |
| `mock.ts` | hash 常量、静态路径 |
| `accountInteractions.ts` | hash ↔ 面板、下拉脚本注入顺序 |
| `README.md` | 路由表、依赖、与本文链接 |

**禁止**在用户后台模块使用运营后台的 `AdminListQuery`、`el-table.admin-ep-table-wrap`、`AdminSectionHead toolbar-only` 骨架。

## 11. 新产品接入：AI 云用户后台（清单）

在 `apps/ai-cloud`（或独立 app）落地时，建议按序完成：

### 11.1 规划

- [ ] 确认 IA：侧栏分区名称与 hash 列表（可与 Trinity AI 五区不同，但结构同构）。  
- [ ] 确认**不属于**运营后台需求（若内部运营用，走 `ai-cloud-admin` + `docs/02`，另立项）。

### 11.2 工程

- [ ] 建 `src/views/account/`（或 `console/`）五件套，**复制并对照** Trinity AI `account`。  
- [ ] 建产品壳 `src/views/shell/`，顶栏品牌为 AI 云，账户菜单链到 `account/console#…`。  
- [ ] `main.ts` 引入 `trinity-base.css`；控制台页 `import` 控制台 CSS。  
- [ ] 路由：`/ai-cloud/account/console` + redirect 旧路径；门户 `trinity-portal` 挂载 `/ai-cloud` 子路由（与 `/trinity-ai` 并列）。  
- [ ] `mock.ts` 导出本产品 `CONSOLE_HASH` 常量，供壳层 `RouterLink` 引用，避免硬编码双轨。

### 11.3 视觉

- [ ] Token 不改语义名，仅必要时在 `account.css` 增产品特例。  
- [ ] 主按钮一律 `btn-gradient`；筛选一律形式 2 或 design-spec 等价组件。  
- [ ] 弹窗、表格、页头对照 `/user-console-spec` 与 `/design-spec` 走查。

### 11.4 文档

- [ ] 在本目录或 `apps/ai-cloud` 下增 **模块 README** + 原型总览一行。  
- [ ] 打样站 `user-admin-system` 增 AI 云说明链（可选）。

### 11.5 复用策略（团队择一）

| 策略 | 优点 | 注意 |
|------|------|------|
| **A. 复制 account 目录** | 最快 | 需约定同步 account.css 通用部分的流程 |
| **B. 共享 CSS/TS 包**（中期） | 防漂移 | 需抽 `packages/console-*`，两产品引用 |

首期推荐 **A**；抽包在第二个产品稳定后再做。

## 12. 禁止事项（走查用）

| 禁止 | 应使用 |
|------|--------|
| `admin-shell`、`el-table`、`AdminListQuery` | `or-shell`、`table.data-table`、`or-keys-*` |
| 运营后台蓝色实心 `el-button type="primary"` 作主 CTA | `.btn.btn-gradient` |
| 卡片内重复大标题 + 工具栏（若依页眉） | 页内 `h1.or-page-title` + 标题行工具 |
| 控制台底整站页脚 | 无页脚 |
| 魔法 hex / 独立主色 | `var(--*)` from tokens |
| 在 `user-console-spec` 嵌整页 ConsolePage 当规范真源 | 打样 + 链到产品完整路由 |
