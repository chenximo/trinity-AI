## 2. 布局规则（OpenRouter 式控制台）

实现真源：**`account.css`**（勿在 `ConsolePage.vue` 内联大块 layout）。

| 规则 | 说明 |
|------|------|
| 无整页页脚 | 不渲染 `page-foot`；遗留节点 `display: none` |
| 左栏贯通 | 桌面 `.or-side` **220px**，`align-self: stretch`，底 `--surface`，右 `1px` 分割 |
| 主列底 | `.or-main` 底 **`var(--bg)`**，与侧栏 `--surface` 成对 |
| 主列留白 | `padding: 0.75rem 1.25rem 2rem 1.75rem`（窄屏跟 `--page-gutter`） |
| 壳主列 | `#main` 在含 `.account-console-root` 时 **`padding: 0`**，纵向 flex 占满剩余高度 |
| 窄屏 | `<900px` 侧栏改顶栏横条 + 底部分隔线 |

## 3. 样式与 token

| 类别 | 类名 / 组件 | 颜色与高亮 |
|------|-------------|------------|
| 主 CTA | `.btn.btn-gradient` | 底 `var(--grad)`，悬停 `var(--grad-hover)`，字 `#fff` |
| 次要按钮 | `.btn`、`.or-btn-outline` | 灰边 + `--muted`；悬停 `--surface` / `--border-strong` |
| 筛选形式 2 | `.or-app-filter-dd-*` | 药丸 + listbox；选中 ✓；聚焦 `--blue` + `--blue-ring` |
| 搜索 | **`SearchForm1Fixed`**（用户控制台系统列表/toolbar） | 形式1 固定宽（默认 `17.5rem`）；侧栏等药丸场景用 `SearchForm2Grow`，见 `/design-spec#ds-search` |
| 侧栏当前项 | `.or-side a.is-active` | `--surface-2` 底 + `font-weight: 600` |
| 侧栏悬停 | `.or-side a:hover` | `--surface-2` 浅底 |

Token 真源：`packages/tokens/src/core.css` → `assets/trinity-base.css`。改色先改 token，再在 `account.css` 补控制台特例。

## 4. 页面内结构（以 API 密钥 `#keys` 为范本，对齐图 2）

主列自上而下五块（样例见 `/user-console-spec` §2）：

| 顺序 | 块 | DOM |
|------|-----|-----|
| 1 | 面包屑 | `nav.or-crumb` · 例：`Trinity AI / API 密钥` |
| 2 | 页头 | `header.or-keys-pagehead` · **UC-MAIN-HEAD-01** |
| 2a | 标题行 | `.or-keys-title-row` · **左** `h1` · **右** `.or-keys-title-actions`（形式 2 筛选 + 最右主按钮） |
| 2b | 说明行 | `.or-keys-lead-row` 在标题行**下方** · **左** `p.or-lead` + 小标题后 ⓘ · 勿与右侧工具同一行 |
| 3 | 搜索 | `.or-keys-toolbar` · `@trinity/ui` **`SearchForm1Fixed`**（形式1 固定宽，见 `/design-spec#ds-search`） |
| 4 | 表格 | `.table-wrap.or-keys-table-wrap` · `.data-table.or-keys-table`（§4.1） |
| 5 | 摘要 | `p.or-keys-summary`（可选） |

```
.or-crumb
.or-keys-pagehead
  .or-keys-title-row
  .or-keys-lead-row          ← p.or-lead（小标题句 + ⓘ）
.or-keys-toolbar            ← 搜索（可选）
.table-wrap / 列表区
.or-keys-summary            ← 条数（可选）
.or-modal-root               ← 弹窗（按需）
```

| 区域 | 约定 |
|------|------|
| **页头分区 UC-MAIN-HEAD-01** | **左**：标题 + 其下说明（含 ⓘ）；**右**：筛选/按钮与 **h1 同一行**；说明行不占右侧列 |
| 标题行右侧 UC-MAIN-02 | 筛选在左（**形式 2** `or-app-filter-dd-*`），**主操作在最右**（`btn-gradient`） |
| 说明 + ⓘ UC-MAIN-03 | 小标题句后紧跟 `details.or-keys-info`；勿放 `h1` 后或说明行最右 |
| 表格操作 | 文字链 / 线框 / ⋮ 菜单，非 EP `link` 按钮 |
| 弹窗底栏 | 主/次按钮**仅文字**，不加图标（与 `design-spec` 弹窗一致） |

**唯一风格示范**：`/user-console-spec` §2（API 密钥主列）；§3 仅补充操作列分支，不另起一套界面。若产品面板需两个示范页才能对齐，视为风格未统一。

完整条文索引：`/user-console-spec`（§1 布局 · §2 主列 · §3 表格）。

## 4.1 列表表格 · 表头与操作列

用户控制台系统列表 **风格统一即可**，**类名不必一致**。**Agent 执行真源**：`.cursor/skills/trinity-user-console/SKILL.md`（规则 ID）。**人 + Agent 验收**：`/user-console-spec` §3。勿沿用角色页渐变表头。

| 规则 ID | 必达 / 禁止（摘要） |
|---------|---------------------|
| UC-TBL-COL-01 | `width:100%` + `table-layout:auto`；表头/数据同列自适应；`table-wrap` 可横滚；允许单列 min/max；禁止常规列表 fixed+全列% 锁宽 |
| UC-TBL-ALIGN-01 / 02 | 全表左对齐；禁止 `or-keys-th-center` |
| UC-TBL-HEAD-01 | 扁平表头；禁止渐变表头 |
| UC-TBL-OPS-01 / 03 | &lt;4 线框横排；禁止窄列类挡线框 |
| UC-TBL-OPS-02 / 04 | ≥4 ⋮；禁止与线框混用 |

类名见 Skill **附录 A**。

**工程与样例对照**

| 场景 | 路由 hash | 规范样例 / 产品 |
|------|-----------|-----------------|
| 密钥整页 + 表（⋮） | `#keys` | `#spec-sample-main` · `or-keys-table` |
| 用量宽表（图标操作） | `#logs` | 产品已用 `or-logs-wide`（风格属 A 族，类名不同） |
| 角色列表（3 按钮） | `#preset` | `#spec-sample-table-actions-buttons` |
| 多云账号（1 按钮） | `#accounts` | 同上 |
| 多操作（⋮ 示意） | — | `#spec-sample-table-actions-menu` |

CSS 真源（落地时改）：`assets/trinity-base.css`（按场景选择器对齐视觉，非强制统一类名）。

> **实施状态（2026-05）**  
> - **已定**：本文 §4.1 与 `/user-console-spec` §3 条文 + 内联样例。  
> - **未改**：产品页（`ConsolePage.vue`、`TrinityAI/account/console.html`、AI 云 `#accounts` 等）与全局 CSS **仍可能沿用旧 `or-preset-table` 渐变表头**；对齐规范时再批量改，勿边定规范边改业务页。

## 5. 与运营后台（`/admin-ops-spec`）对比

| 维度 | 用户控制台系统（Account） | 运营后台 |
|------|----------------------|----------|
| 框架 | 单页 + hash 五区 | 多模块路由 + 侧栏菜单树 |
| 组件 | `@trinity/ui`、`trinity-base` | Element Plus + `admin-shell` |
| 列表 | `or-*` 表格 / 静态表 + 自定义交互 | `el-table` + `AdminListQuery` |
| 主按钮 | `.btn-gradient` | `el-button type="primary"` |
| 样式入口 | `account.css` | `admin-theme.css`、`admin-element-plus.css` |

## 6. 开发检查清单

- [ ] 新面板挂在 `.or-main` 内，带 `data-or-panel` 与侧栏 `href="#id"` 对齐  
- [ ] 左栏 / 主列底色仅用 `--surface` / `--bg`，禁止魔法 hex  
- [ ] 主路径按钮用 `.btn-gradient`，次要操作用线框或文字链  
- [ ] 形式 2 下拉使用 `or-app-filter-dd-wrap` + `adm-form2-dd.js`（见 `accountInteractions.ts`）  
- [ ] 不在用户控制台系统引入 `admin-shell` / `el-table` 若依样式  
- [ ] 列表表：扁平表头 + **全表左对齐**（§4.1）；操作 &lt;4 横排线框 / ≥4 ⋮
