3. 设计枢纽落地：[Trinity设计枢纽（色板+规范）落地计划.md](./Trinity设计枢纽（色板+规范）落地计划.md) · 门户操作：[Trinity开发枢纽与AI协作流程.md](../00-协作与工作流/Trinity开发枢纽与AI协作流程.md) §4.1.5  

**仍在本 Skill 会话中主动做的事**

- 改完模块后更新 `views/<模块>/README.md`（§6 + §6.2）  
- 业务说明写 `apps/<app>/doc/` 或 `views/<模块>/README.md`（§5.1），勿与 README 重复抄功能范围  
- 多模态输入：默认 **结构优先**；高保真仅当用户明确要求，色值仍映射 Token（见规范 §12）  

**静态 HTML**（`TrinityAI/` 等）：只作对照；新功能写在五件套，不双轨维护。

其它专题 Skill：`.cursor/skills/trinity-design-tokens`、`trinity-vue-prototype-monorepo` 等，均不得违背上述规范真源。
