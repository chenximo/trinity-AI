## 1. 列表页布局规范

### 1.1 页面结构模板

参考 `apps/trinity-ai-admin/src/views/admin-keys/KeysPage.vue`：

```html
<div class="xxx-page">          <!-- 页面容器：flex column -->
  <section class="xxx-page__panel">  <!-- 面板容器：跟随 .admin-ep-card -->
    <el-card shadow="never" class="admin-ep-card">
      <AdminSectionHead title="列表标题">
        <!-- 工具栏插槽 -->
      </AdminSectionHead>

      <!-- 表格 -->
      <div class="admin-ep-table-wrap">
        <el-table ...>
          ...
        </el-table>
      </div>

      <AdminTablePagination ... />
    </el-card>
  </section>
</div>
```

### 1.2 CSS 规范

对应 `xxx.css`：

```css
/* 页面容器 */
.xxx-page {
  display: flex;
  flex-direction: column;
  min-height: 0;
}

/* 面板跟随 .admin-ep-card 的 flex 布局即可 */
.xxx-page__panel {
  /* 由 admin-ruoyi.css 中的 .admin-ep-card 处理 */
}

/* 表格撑满剩余高度（重要！） */
.xxx-page .admin-ep-table-wrap {
  flex: 1 1 auto;
  min-height: 0;
}
```

### 1.3 关键原则

| 原则 | 说明 |
|------|------|
| **不要** 在 `el-card` 上直接加 `xxx-page__panel` | 用 `<section>` 包裹 |
| **不要** 给 `el-table` 设置固定 `height` | 让 flex 自动撑满 |
| **必须** 在 `admin-ep-table-wrap` 上设置 flex 撑满 | 否则表格内部无法滚动 |

## 2. 表格操作列宽度规范

### 2.1 宽度档位（定义在 `src/utils/adminTableColumns.ts`）

```typescript
export const ADMIN_TABLE_COL_OPS = {
  sm: 120,  // 1-2 个按钮
  md: 180,  // 2-3 个按钮
  lg: 280,  // 4+ 个按钮或有图标链接
};
```

### 2.2 选择指南

| 场景 | 选用的档位 | 示例 |
|------|-----------|------|
| 仅「查看」或「编辑」 | `sm` | 供应商列表 |
| 2-3 个按钮（无图标） | `md` | 审核队列、黑白名单 |
| **4+ 个按钮，或有图标链接按钮** | **`lg`** | **用户列表、API 密钥** |
| 按钮较多且有 dropdown | 考虑自定义宽度 | 参考 keys 页面 |

### 2.3 常见错误

```vue
<!-- ❌ 错误：4个按钮用了 md (180px)，会溢出 -->
<el-table-column label="操作" :width="ADMIN_TABLE_COL_OPS.md" fixed="right">

<!-- ✅ 正确：4个按钮用 lg (280px) -->
<el-table-column label="操作" :width="ADMIN_TABLE_COL_OPS.lg" fixed="right">
```

## 3. 排查问题的正确方法

### 3.1 遇到「列表溢出」时的排查顺序

> **优先级：从高到低**

1. **先检查操作列宽度** — 这是最容易遗漏的配置问题
   - 参考同类型页面的操作列档位（如 keys 页面用 `lg`）
   - 操作按钮数量是否匹配当前档位

2. **再检查布局结构** — 确认是否用了 `<section>` + `<el-card class="admin-ep-card">`
   - 不要直接在 `el-card` 上加 flex 类

3. **最后检查 CSS** — `admin-ep-table-wrap` 的 flex 撑满

### 3.2 对比参考页面

遇到问题时，**先对比参考页面**，而不是从外围样式开始排查：

```
问题：列表溢出
  ↓
1. 用户说「参考 keys」
  ↓
2. 直接对比 keys 和当前页面的操作列配置
  ↓
3. 发现宽度差异 → 一行代码修复
```

**错误的排查路径**：
```
问题：列表溢出
  ↓
1. 检查 flex 布局 ❌
2. 检查 overflow 属性 ❌
3. 检查父容器高度 ❌
4. 检查 min-height ❌
...
浪费大量时间
```

### 3.3 快速定位三问

| 问题 | 优先检查 |
|------|----------|
| 列表整体溢出 | 操作列宽度、布局结构 |
| 表格内容不滚动 | `admin-ep-table-wrap` flex 设置 |
| 按钮挤在一起 | 操作列宽度档位 |

## 5. 新功能开发流程（AI 协作规范）

### 5.1 开发顺序

> **先文档 → 再草图 → 讨论确认 → 最后开发**

| 阶段 | 内容 | 产出物 |
|------|------|--------|
| **1. 写文档** | 功能设计、需求说明、数据结构、API 约定 | 需求文档 / 设计文档 |
| **2. 画草图** | 页面布局、交互流程、组件拆分 | 原型图或 ASCII 草图 |
| **3. 讨论确认** | 双方确认方案可行后再开始 | 确认记录 |
| **4. 开始开发** | 按规范实现 | 可运行代码 |

### 5.2 遇到问题时的正确姿势

1. **用户提到参考页面** → **立刻对比参考页面**，而不是从外围排查
2. **遇到配置问题** → 先检查常量/配置，再查样式
3. **排查路径**：
   - 配置问题 > 布局结构 > CSS 细节
