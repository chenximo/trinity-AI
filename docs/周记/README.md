# Trinity · 周记

> **用途**：每周一填写并**上交**的工作小结（上一周产出 + 下一周计划）。  
> **不替代**：长期真源文档仍用语义文件名 + 文内版本/修订记录。

---

## 命名（固定一种）

| 项 | 约定 |
|----|------|
| **目录** | `docs/周记/` |
| **文件名** | **`YYYY-Www.md`**（ISO 8601 周历，`ww` 两位补零） |
| **周一写哪一周** | 文件名 = **刚结束的那一周**。例：2026-05-18（一）→ 填 **`2026-W20.md`**（05-11～05-17），勿用当天 `date +%G-W%V` 直接当文件名 |

---

## 单文档结构（简版 + 详细版）

**每周只维护一个文件** `YYYY-Www.md`，内含两段（顺序固定）：

```markdown
## 简版（上交）
### 本周工作
### 下周计划

---

## 详细版
### 本周工作
### 下周计划
```

| 段落 | 用途 |
|------|------|
| **简版（上交）** | 四条主线 + 下周计划；**复制整节提交**。正文仅写业务交付，**不写**「维护周记」「待填写」「本地未提交」等协作用语。 |
| **详细版** | 完整条目与路径；团队存档 / AI 续写用，**勿上交**。 |

**本周工作四条主线**（简版、详细版共用逻辑）：

1. 原型 HTML → Vue 技术栈迁移  
2. 设计规范与色板  
3. 原子组件设计与实现（`@trinity/ui`）  
4. 业务原型交付  

**滚动清单**：[`设计待办.md`](../02-后台运营管理系统设计/设计待办.md)、[`产品管理随手记.md`](../00-协作与工作流/产品管理随手记.md) 仍在原文件维护；周记里至多一句带过。

---

## 节奏（每周一）

1. 复制 [`_template.md`](./_template.md) → `docs/周记/YYYY-Www.md`（上一 ISO 周）。  
2. 先写 **详细版**，再收束 **简版（上交）**；对外只提交简版一节。  
3. 更新下方索引表。
