# Trinity 官网首页 · 静态原型提示词（v1）

> **文档状态**：归档（前期方案对比，非交付口径；不默认随现网迭代）  
> **文档类型**：AI 生成提示词  
> **与实现对齐**：**已与当前静态实现脱钩（历史白板 v1）**。本稿描述「总官网首页 + 顶栏 AI 云厂商下拉 + 多语言」等信息架构；仓库内 **`TrinityAI/index.html` 现为 Trinity AI 产品子站首页**（主导航：首页 / 模型 / 对话 / 文档），**多云 B2B 营销落地在 `TrinityCloud/home.html`**。若要用本提示词生成新页，请先按现网结构改写后再执行。

> **定位**：面向 **Trinity** 品牌官网 **首页** 的静态 HTML/CSS 原型（可含少量原生 JS）。  
> **灵感来源**：白板草图（AI 云 / Trinity AI / 多语言 / 登录、服务·架构·咨询、云介绍、OpenRouter 式极简科技风）。  
> **非目标**：本提示词不要求接入真实后端、不要求 React/Vue 工程；多语言可先做成 **切换 UI 占位**（文案仍以中文为主或中英并列示例）。

---

## 一、品牌与调性

- **品牌名**：**Trinity**（全站统一大小写与拼写；Logo 区可用文字标 + 简单图形符号占位）。
- **气质**：企业级 **AI 云与智能基础设施** 方向；视觉参考 **openrouter.ai** —— 顶栏简洁、留白充足、蓝白/浅灰主色、无冗余装饰，强调 **可信、专业、易扫读**。
- **主色建议**：与 OpenRouter 系协调的蓝色（如 `#2563EB`）；背景浅灰白（如 `#F8FAFC` / `#FFFFFF`）；正文深灰（如 `#1E293B` / `#0F172A`）。

## 二、全局顶栏（Header，固定或滚动后吸顶）

从左到右建议结构：

| 区域 | 内容 | 行为与说明 |
|------|------|------------|
| 左侧 | **Trinity** Logo + 文字 | 点击回到首页顶部 `#` 或 `index.html` |
| 主导航 | **AI 云** | **下拉菜单**：展示云厂商/渠道入口，示例项 **`腾讯云`**、**`华为云`**、**`Google Cloud`**（可再增 **阿里云**、**AWS** 等 1～2 项）；每项为链接占位 `href="#"` 或指向占位页 `cloud-tencent.html` 等；**悬停展开 / 点击展开** 二选一，移动端改为 **手风琴或全屏抽屉** |
| 主导航 | **Trinity AI** | 链接到现有产品子站或占位 `trinity-ai.html` / `#trinity-ai`（与「官网」形成产品矩阵叙事） |
| 右侧 | **多语言** | 下拉或图标切换：`中文` / `English`（原型可仅切换按钮状态或 `aria-label`，不必做完整 i18n） |
| 右侧 | **登录** | 按钮样式突出；跳转 **`login.html`** 或站点已有登录弹窗锚点 **`#login`**（与现有 Trinity 站点对齐即可） |

**交互**：顶栏高度适中（约 `56px`～`64px`）；导航项 **Hover** 有下划线或浅色底；当前页（首页）可在「Trinity AI」或首页入口上做弱高亮（若首页即根路径，可仅高亮「首页」占位项，按信息架构自定）。

**无障碍**：`nav` 地标；下拉用 `button` + `aria-expanded`；键盘可关闭下拉（`Escape`）。

## 三、首页主体区块（自上而下）

### 3.1 Hero（首屏）

- **主标题**：突出 **「AI 云」** 或 **「Trinity AI 云」** 价值一句话（示例方向：一条链路纳管多云算力与模型服务）。
- **副标题**：一行说明 **谁在用、解决什么问题**（企业、开发者、多厂商统一接入等）。
- **主行动点（CTA）**：如 **「了解方案」**、**「预约咨询」**、**「登录控制台」** 中任选 1～2 个按钮；主次按钮用主色与线框区分。
- **可选**：右侧或背景 **轻量插画/抽象网格/渐变光晕**（静态图或纯 CSS），避免喧宾夺主。

### 3.2 核心价值 / 优势（对应白板「服务、架构、咨询」）

用 **三列卡片栅格**（桌面一行三卡，移动端单列），每张卡对应一类优势，标题与文案建议：

1. **架构设计**（或「云架构」）：多云统一架构、安全合规、可扩展路由与观测等（2～3 句占位）。
2. **专业咨询**：上云规划、模型选型、成本与 SLA 咨询等（2～3 句占位）。
3. **专属折扣 / 商务权益**（「折扣」）：渠道价、阶梯折扣、企业合约等（2～3 句占位；可加小标签 **「限时」** 等纯展示）。

卡片：白底、圆角、细边框、轻阴影；图标可用线性 SVG 占位（云、架构图、对话/折扣标签等）。

### 3.3 AI 云介绍（「云介绍、说明」）

- **区块标题**：如 **「什么是 Trinity AI 云」** 或 **「AI 云能力一览」**。
- **内容**：2～3 段说明文字 + **可选** 要点列表（算力池、模型网关、计费透明、技术支持等）；可配 **简易架构示意图占位**（框线图）。
- **与下拉联动叙事**：文案中可自然提到 **腾讯云、华为云、Google Cloud** 等已出现在下拉中的名字，强化「多云」心智。

### 3.4 信任与背书（可选模块）

- **客户/合作伙伴 Logo 墙**（灰度占位图 + 文字「占位」）。
- 或 **关键数字**：可用量、可用区、模型数等（假数据，标注)。

### 3.5 页脚 Footer

- 产品链接、文档、隐私与条款、社交媒体图标占位；版权 **© Trinity** + 年份。

## 四、与 OpenRouter 的对齐方式（仅风格与信息层级）

- **参考 openrouter.ai**：顶栏项数量克制（类似 Home / Models / … 的信息密度），**大标题 + 短副标题 + 少量 CTA**；正文区块之间 **间距明显**。
- **差异**：Trinity 强调 **「AI 云」多云厂商入口（下拉）** 与 **「架构 / 咨询 / 折扣」** 的商业叙事，而非模型排行榜为主角（若需模型入口可放在 Trinity AI 子链或次要 CTA）。

---

## 五、视觉与组件规范（原型级）

- **圆角**：按钮与卡片 `8px`～`14px` 统一。
- **按钮**：主按钮主色填充；次按钮线框；Hover 略加深。
- **下拉**：白底、阴影、项 Hover 浅灰底；与顶栏对齐无错位。
- **排版**：中文标题字重 `600`～`700`；行宽正文 `max-width` 约 `42rem`～`48rem` 提升可读性。
- **响应式**：`≤768px` 时顶栏 **汉堡菜单** 展开侧抽屉，内含 AI 云（可展开子项）、Trinity AI、语言、登录。

---

## 六、交付物约定（给生成静态原型的 AI）

1. **文件**：至少 `index.html`（首页）+ `admin.css` 或独立 `site.css`；若有子页 `cloud-*.html` 可选。
2. **技术**：纯静态，**无打包**；图片可用占位符或内联 SVG。
3. **文案**：中文为主；英文仅用于语言切换时可并列小字。
4. **注释**：在 HTML 中标注「白板区块对应关系」（Hero / 三优势 / 云介绍 / Footer），便于后续替换为 CMS 或组件库实现。
