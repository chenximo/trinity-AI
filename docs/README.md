# Trinity · 文档索引

> `docs/` 放 **仓库级规范** 与 **跨 app 协作**；**具体业务** 放各 `apps/<项目>/doc/`；**旧链接**见 [§路径迁移对照](#路径迁移对照)。

---

## 文档怎么放（四条原则）

| 类型 | 放哪 | 例子 |
|------|------|------|
| **规范**（要长期遵守、改代码要对齐） | **`docs/`** 下 `01` `02` `03` `00` 协作 | 五件套、版式、用户/运营后台、双仓 |
| **一次性生成**（AI 提示词、生成稿，不再迭代） | **`docs/06-归档与提示词/`** | Figma Make 提示词、复刻站提示词 |
| **方案对比、思想碰撞、方法论、学习汇报** | **`docs/08-方法论与汇报/`** | 形态对比、五件套价值说明、工作流总结 |
| **具体业务**（某 app / 某产品的 IA、批次、后端对照） | **`apps/<app>/doc/`** + **`views/<模块>/README.md`** | `trinity-ai-admin/doc/创建后端…`、模块路由表 |

**补充**：

- **`docs/05-产品与PRD/`**：**平台级**产品（整盘能力地图、运营后台详设、跨 app 交付计划）；稳定后业务细节以 **app `doc/`** 与模块 README 为准。  
- **`docs/04-工程与迁移/`**：Monorepo / 迁移 **计划**；完成后可迁入 `06/` 或文首标已完成。  
- **`docs/03-用户控制台系统/`**：租户 / 开发者控制台规范索引（与 `02` 运营后台分轨）。  
- **`docs/07/`、`周记/`**：工具产出、过程记录，非规范。

## 迁入 `06` 的时机（一次性生成）

满足 **任一** 即迁入 `docs/06-归档与提示词/`，文件名 **`…（归档）.md`**，文首标 `文档状态：归档` 并链到现网真源：

- AI **一次性生成**的提示词 / 页面生成稿，不再作为迭代输入  
- 静态 HTML 期备忘，且已由 **`apps/*/views/*/README`** 承接  
- 已废止流程、模板  
- **已完成**的迁移计划（无未勾选项）

**不要放进 06**：方案对比、形态选型、协作模式讨论 → 放 **`08`**。仍在维护的规范 → **`docs/01` `02`**；具体业务 → **`apps/<app>/doc/`**。

**升格**（走查条目稳定）→ 写入 **05 详设**、**02 规范** 或 **app `doc/`**，不是归档。
