# 运营面 · 总览

> **更新**：2026-05-26 · **对标**：OpenRouter **无公开运营后台**；本表按「商用聚合平台必备运营能力」梳理，并与 OR 的 Provider / 模型目录 / 计量等 **能力映射**。  
> **分页真源**：`运营面-0x-*.md`（待建）；本页为 **10 大模块** 汇总。  
> **工程**：`apps/trinity-ai-admin/src/views/admin-*` · 详设 §4 · [运营后台原型交付计划](../AI-API聚合平台-运营后台-原型交付计划.md)

## 一句话

面向平台内部运营/财务/风控：**上架供应商与模型 → 配线路与密钥 → 看监控与用量 → 管用户与租户 → 审计可追溯**；内测重 **原型 + 核心 API 联调**，商用重 **RBAC、审计、计费闭环**。

## 与 OpenRouter 的关系

| 说明 | |
|------|-----|
| OR 对外 | 客户自助完成注册、Key、充值、逛模型；**无**对等「运营后台」产品 |
| 本表「对标」列 | 写 OR **能力域**（Provider、Models、Usage…）+ **我们必备运营动作** |
| 6.30 ✅ | 指 **Trinity 商用运营闭环**，非「做出 OR 同款后台」 |

## 与用户面 / 平台面的关系

| 运营面 | 支撑 |
|--------|------|
| 供应商 + 模型 + 线路 | 用户面 **模型广场 live**、平台面 **路由** |
| 密钥 + 用户审核 | 用户面 **控制台 Key**、平台面 **鉴权** |
| 用量与计费 | 用户面 **用量展示**、平台面 **计量扣费** |
| 监控与风控 | 平台面 **限流**、商用 **SLA** |

## 5.31 内测门禁（运营面 · P0）

| 优先级 | 模块 | 门禁 |
|--------|------|------|
| **P0** | 供应商 → 模型 → 平台密钥 | 可配置一条完整上游链路 |
| **P0** | 用量明细 | 可查 `usage_event` 聚合（原型接真 API） |
| **P0** | 监控大盘 | 核心 KPI 接 live 或准 live |
| **P0** | 权限与审计 | 后台登录 + **写操作审计最小集** |
| P1 | 用户审核 | 单条通过/拒绝可走通 |
| P1 | 用户 Key 运营 | 冻结/限额示意接 API |
| P2 | 客户与合同 | mock 演示即可 |
| **不做 5.31** | 报表全量、§4.6 路由策略页、发布中心 | 排 6.30 |
