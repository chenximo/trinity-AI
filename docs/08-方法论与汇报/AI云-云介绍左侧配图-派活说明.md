## 1. 一句话公式

**「参考图」定视觉语言 + 「业务场景」定内容 + 「禁止项」定边界 + 「落点」定改哪里**

**示例：**

> 云介绍左侧配图，**风格参考我发的叠卡 UI 图**（多卡片漂浮、头像、对话、像真产品在用），场景是**阿里云双 11 大促协同**（联系人、架构师聊天、成本看板），**不要**架构拓扑图 / 设备外框，**只改** `#cloud-solutions` 阿里云 Tab 左侧。

**权重（经验）：**

| 输入 | 权重 | 作用 |
|------|------|------|
| **参考图（附图）** | 最大 | 叠层圆角白卡、阴影、营销站 UI 样张感 |
| **业务场景名词** | 第二 | 画什么业务（双 11、直播、专线…） |
| **禁止项** | 第三 | 防止拓扑图、设备框、六家各一套信息图 |
| **落点与验收** | 第四 | 改哪、多高、Tab 切换是否一致 |

无附图时，易滑向「架构示意图 / 导播单屏 / 地球节点网」等常见套路，与目标风格易偏离。

## 2. 参考图：定「长什么样」（最重要）

### 2.1 推荐表述（命中率高）

| 你说 | 协作者会理解为 |
|------|----------------|
| 「风格参考这张图」+ **附图** | 叠层圆角白卡、阴影、浅色底、可有深色侧栏 |
| 「像真实产品界面 / 营销站 UI 样张」 | 不是示意图、不是 PPT 架构图 |
| 「分层 / 漂浮 / 多卡片叠在一起」 | 构图语言；区别于单张大图、全屏拓扑 |
| 「有头像、对话、列表、小浮卡」 | 「人在用」的场景感 |
| 「以阿里云这张为样例，其它云同风格」 | 六家 **视觉统一**，只换业务内容 |

### 2.2 少说或慎用（易带偏）

| 慎用说法 | 风险 | 建议改成 |
|----------|------|----------|
| 「做个架构图 / 拓扑 / 组织树」 | 变成工程示意图主视觉 | 除非真要示意图；否则强调 **卡片 UI** |
| 「每家风格要完全不同」 | 六套信息图，失去统一语言 | 「六家 **同叠卡风格**，**不同业务场景**」 |
| 「参考 OpenRouter / 某官网」（无附图） | 按文字猜，可能对不上叠卡 UI | **截图** + 指出「要像哪一块」 |
| 「做个好看的云介绍图」 | 过于空泛 | 附图 + 场景名词列表 |

### 2.3 参考图视觉要素清单（自检）

- [ ] 多张圆角卡片 **重叠漂浮**，软阴影
- [ ] 浅色底（或局部深色浮卡），**无** 笔记本 / 手机设备外框
- [ ] 头像、列表、对话或控制台片段，像 **SaaS 在用**
- [ ] 偏 **营销落地页插图**，非技术白皮书配图

## 3. 场景：定「画什么业务」（第二重要）

每张补 **3～5 个具体名词**（中文标签即可），比形容词管用。

### 3.1 现网六云（与组件对应）

| 云厂商 | 组件 | 场景关键词（示例） |
|--------|------|-------------------|
| 阿里云 | `AliyunCloudSceneVisual.vue` | 双 11、架构师对话、ECS / 费用中心、弹性预算预警、成本归因柱状图 |
| 腾讯云 | `TencentCloudSceneVisual.vue` | 直播、连麦、TRTC、CDN、PGM 监视器、延迟 / 在线人数（**非**与阿里同款聊天布局） |
| 华为云 | `HuaweiCloudSceneVisual.vue` | 专线入云、政务机房、运营商多线、等保、昇腾节点 |
| AWS | `AwsCloudSceneVisual.vue` | Landing Zone、Control Tower、OU、SCP、CUR；**卡片 UI**，非整屏组织树 |
| Google Cloud | `GcpCloudSceneVisual.vue` | 多 Region、数据流、Vertex / GKE、数据驻留（叠卡/控制台感，非地球拓扑主视觉） |
| Azure | `AzureCloudSceneVisual.vue` | 混合云、Arc、ExpressRoute、OpenAI / AKS、订阅与资源组 |

### 3.2 场景句模板

> 内容贴近「{客户类型}用控制台做 {一件事}」，界面里要有 **{元素 A}、{元素 B}、{数字指标}**，不要画成 **{不要的主视觉}**。

**示例：**

> 内容贴近「政企客户用控制台办专线」，界面里要有 **工单、带宽、时延数字**，不要画成 **网络拓扑主视觉**。

---

## 4. 禁止项：定「不要什么」

本次对话中验证有效的约束：

- **不要**与阿里云一模一样的「左侧联系人 + 中间聊天」布局（腾讯等需单独场景，如直播导播台）
- **不要**外层灰色设备框 / Mac 边框 / 手机壳 mockup
- **不要**抽象 SVG 示意图、占位折线图糊弄主视觉
- **不要**六家都用同一套聊天卡片 —— **同风格**，**不同场景组件**
- **不要**在无要求时大范围改右侧文案区、其它 Tab 文案

**生图时额外 negative（若走 AI 出图）：**  
architecture diagram, org chart tree, network topology map, device frame, watermark, gibberish UI text

## 5. 落点：定「改哪里、验收怎么看」

| 说清楚 | 作用 |
|--------|------|
| 页面：`/ai-cloud` 首页 `#cloud-solutions` | 定位模块 |
| 位置：某云 Tab **左侧图**，右侧文案不动 | 控制范围 |
| 尺寸：与 **Google Cloud**（或其它已对齐 Tab）块 **同高** | 统一占位（现网 lrbt `--home-uc-art-h: 268px`） |
| 验收：切换 6 个 Tab，左侧图 **高度一致**、无外层灰框 | 可测标准 |

**样式约定（现网）：**

- 场景区背景在 `.home-uc-scene--{vendor}`，主图在 `.home-uc-scene-main`
- 图下 **三个** `home-uc-scene-tag` 由 `HomePage.vue` 维护，不必全画进 mockup 内

## 6. 实现方式选型

### 6.1 默认推荐：Vue 组件 + CSS 分层 mockup

- **做法**：每云一个 `*CloudSceneVisual.vue`，`HomePage` 引入替换原内联 SVG
- **派活时写**：「做成参考图那种叠卡 UI」「优先 Vue 场景组件，对齐 Trinity 色板」
- **优点**：中文准确、六云统一、响应式清晰、改文案便宜、无按张生图费
- **缺点**：头像多为示意块，「实拍感」弱于 AI 图（可用少量 PNG 混合）

### 6.2 备选：AI 生图（PNG）

- **做法**：参考图 + 场景 prompt → 导出 PNG → `assets/home/cloud-scenes/` → `<img>` 或背景图
- **派活时写**：「附图 style reference」「4:3 横图约 1200×900」「透明或浅色底」
- **优点**：真人头像、照片质感、第一眼更「真」
- **缺点**：六张风格难完全一致；改字常需重生成；小字可能错；有按张成本

### 6.3 混合

骨架仍用 Vue；仅头像条、照片卡等用 **1～2 张切图**，兼顾维护与真实感。

### 6.4 成本粗比（供决策）

| 维度 | Vue mockup | AI 生图 |
|------|------------|---------|
| 首轮协作 | 对话 tokens 多（写组件） | 对话 tokens 少，**生图按张另计** |
| 迭代改文案/尺寸 | 低 | 高（常整张重出） |
| 六云一致性 | 高 | 中（需统一 prompt + 筛选） |
| 长期维护 | **更省** | 一般不作产品页默认 |

**结论：** 产品页六云 Tab **默认 Vue**；营销单张或强真实感头像再考虑生图或混合。

## 7. 可直接复制的派活模板

```text
【模块】AI 云首页 → 云厂商介绍 → {云厂商名} Tab 左侧配图
【参考】附图：多层圆角 UI 卡片叠放、浅色底、像真实 SaaS 在用（与阿里云现效果一致）
【场景】{3～5 个具体业务元素，中文标签即可}
【统一】构图语言与阿里云一致；只换场景内容，不换整体风格
【禁止】架构拓扑主视觉、设备外框、抽象折线图、与阿里云完全相同的聊天布局（若需区分）
【尺寸】与其它云 Tab 左侧图同高（lrbt 绘图区与 GCP 块一致）
【实现】优先 Vue 场景组件，对齐 Trinity 色板（trinity-base.css）；不要大范围改文案区
【验收】切换 6 Tab 高度一致；#cloud-solutions 左侧 only
```

## 8. 本次对话复盘（有效 vs 歧义）

### 8.1 说得好的 → 结果

| 表述 | 结果 |
|------|------|
| 附图 +「更真实贴近场景」 | 对齐叠卡 UI，而非示意图 |
|「用阿里云为样例」 | 明确视觉锚点 |
|「腾讯云不要和阿里一样，要直播音视频」 | 内容分化，整体满意度保留 |
|「去掉外层灰框」「和 GCP 一样大」 | 约束清晰、改动可控 |

### 8.2 易歧义 → 建议

| 易歧义 | 建议 |
|--------|------|
| 「做个好看的云介绍图」 | 附图 + 场景名词列表 |
| 「参考某网站」 | 截图 + 指明区块 |
| 「六家各做一种风格」 | 「六家同叠卡风格，不同业务场景」 |
