# AI API 聚合平台 · 后台管理系统静态原型 — 完整提示词（样板）

> **用途**：交给 AI 或外包，用于一次性生成 **对标 OpenRouter** 的企业级后台 **静态 HTML/CSS（+ 可选极少量原生 JS）** 可点击原型。  
> **非目标**：本样板不要求 React/Vue、不要求真实后端、不要求 Ant Design / Element Plus 工程化实现；若后续要迁移到组件库，以本文件的 **信息架构与视觉规范** 为准做对照。

---

## 一、交付物（必须写清）

1. **输出形式**：多页静态站点，**无构建工具**；每个一级菜单对应 **独立 `.html` 文件**，共用 **一份 `admin.css`**（或按模块拆 `admin-layout.css` + `admin-components.css`，二选一写清即可）。
2. **可选 JS**：仅用于 **侧栏折叠**、**退出登录确认框**、**移动端侧栏抽屉**；图表若需动效，**优先**使用 **单文件 CDN（如 Chart.js）+ 本地假数据**；若禁止外链，则用 **CSS/SVG 静态示意块** 代替真实图表。
3. **资源**：所有路径使用 **相对路径**；字体可用 Google Fonts（Inter + 思源黑体/Noto Sans SC）或系统字体栈二选一并在说明里固定。
4. **语言**：界面文案 **全中文**；代码注释可用中文或英文，保持一致即可。

## 二、品牌与命名（可替换）

- **系统名称（侧栏顶部）**：默认文案 `Xinghan Admin`；实现时替换为你的平台名（例如 `Trinity 运营后台`），**全站统一**，勿混用。
- **产品定位**：对标 **OpenRouter** 的 **AI API 聚合平台运营后台**，管理多厂商接入、模型目录、用户密钥、调用监控与权限。

## 三、视觉与主题（浅色 / 蓝色系 / 极简科技）

| Token | 建议值 | 说明 |
|--------|--------|------|
| 主色 | `#2563EB` | OpenRouter 系蓝，用于主按钮、链接、选中态、数据强调 |
| 主色悬停 | `#1D4ED8` | 按钮 / 可点击高亮 hover |
| 页面背景 | `#F8FAFC` | 大背景浅灰蓝 |
| 卡片背景 | `#FFFFFF` | 内容卡片、表格容器 |
| 主文字 | `#1E293B` | 标题与正文 |
| 次要文字 | `#64748B` | 副标题、表头、说明 |
| 弱文字 | `#94A3B8` | 辅助说明、占位 |
| 边框 | `#E2E8F0` | 卡片、表格、输入框 |
| 圆角 | 卡片 `12px`～`14px`，按钮与输入 `8px`～`10px` | 全站统一 |
| 阴影 | 卡片轻阴影；hover 时略加深或轻微 `translateY` | 体现层级，忌过重 |

**风格关键词**：极简、留白充足、层级清晰、数据可读、企业后台专业感（参考 OpenRouter 后台气质 + 你提供的侧栏布局截图）。

---

## 四、整体布局

1. **结构**：**左侧固定侧栏** + **右侧主内容区**；**不强制**顶部通栏；若加顶栏，仅保留极简（例如环境标签、面包屑），**主入口仍以侧栏为准**。
2. **侧栏**：固定宽度（建议 `220px`～`260px`），**可选**折叠为图标栏（窄 `64px`～`72px`）；折叠行为若实现，需 **aria-expanded** 与键盘可操作说明。
3. **主区**：`min-width: 0` 防止表格撑破；内容区左右内边距统一（如 `24px`～`32px`）。
4. **响应式**：**桌面优先**；中小屏可将侧栏改为 **抽屉**（按钮打开/关闭），主区全宽；表格区域 **横向滚动**。
5. **无障碍**：每页有 `h1` 与 `main` 地标；侧栏 `nav` 带 `aria-label`；当前页菜单项 `aria-current="page"`。

## 五、左侧侧栏导航（顺序固定）

**顶部**

- Logo 区或纯文字：**系统名称**（见第二节）。

**菜单项（自上而下，文案与路由建议）**

| 顺序 | 名称 | 文件名建议 | 说明（原型页内用副标题或占位段落概括） |
|------|------|----------------|------------------------------------------|
| 1 | 工作台 | `index.html` | 首页 Dashboard：四张核心卡 + 下方扩展看板 |
| 2 | 监控中心 | `monitoring.html` | 调用日志、成功率、延迟、用量趋势（图表占位 + 筛选条占位） |
| 3 | 供应商管理 | `providers.html` | OpenRouter 与各厂商接入配置、状态、权重（表格 + 操作按钮占位） |
| 4 | 模型管理 | `models.html` | 模型列表、参数、上下线、版本（表格 + 筛选占位） |
| 5 | API 密钥管理 | `api-keys.html` | 密钥列表、权限、用量、额度（表格 + 新建占位） |
| 6 | 文档管理 | `documents.html` | Markdown 格式文稿、API 文档模板、分类/版本（列表 + 编辑器占位区） |
| 7 | 用户与权限 | `users.html` | 角色、用户、权限分配、操作审计（Tab 或分块占位） |

**交互与样式**

- **选中项**：浅蓝背景（如 `rgba(37, 99, 235, 0.08)`）+ 主色文字 + 图标与文字同步高亮；左侧可加 **2px～3px** 主色竖条（可选）。
- **未选中**：灰色图标 + 灰色文字（次要文字色）。
- **Hover**：轻微背景变浅灰或极浅蓝，**不改变**当前选中项逻辑。
- **图标**：每项配 **线性 SVG 图标**（统一 `18px`～`20px`，`stroke` 风格一致）：工作台用仪表盘/网格、监控用折线图、供应商用服务器/层、模型用立方体/芯片、密钥用钥匙、文档用文件、用户用多人/用户。

**底部**

- **退出登录**：全宽按钮样式 — **白底 + 主色描边 + 主色文字**（或线框按钮）；点击弹出 **`confirm` 或自绘对话框** 确认后 **跳转 `login.html`**（原型无真实鉴权，仅流程）。
- `login.html`：**独立居中卡片**（账号、密码输入框可为占位），主按钮「登录」跳转回 `index.html`。

## 六、工作台首页（`index.html`，核心页）

### 6.1 页头

- **主标题**：`工作台`
- **副标题**（小字、次要色）：`左侧切换模块，数据来自后端数据库`（或「以下为示例数据」二选一，全站统一口径）

### 6.2 核心指标卡片区（四列栅格）

桌面 **一行四卡**；平板 **两列**；手机 **一列**。每张卡包含：

1. **供应商**：大数字（示例 `9`）；副文案 `厂商/平台`
2. **模型条目**：大数字（示例 `19`）；副文案 `当前激活：全部`
3. **API 密钥**：大数字（示例 `1`）；副文案 `已连接数：全部`
4. **文档页**：大数字（示例 `0`）；副文案 `说明文档存稿`

**卡片样式**：白底、圆角、细边框、轻阴影；**数字**显著大于标题（如 `1.75rem`～`2rem`、字重 `700`）；副文案 `0.75rem`～`0.8125rem`。

**卡片 Hover**（桌面）：轻微 **上浮 `1px`～`2px`** 或阴影略加深（`transition` 约 `0.2s`）。

### 6.3 扩展看板（卡片下方，自上而下）

1. **API 调用趋势（近 7 天）**  
   - 区域标题：`近 7 日 API 调用量`  
   - 内容：**折线图**（CDN+假数据）或 **灰色占位 + 横坐标日期假标签**。

2. **模型用量 TOP 排行**  
   - 区域标题：`模型用量 TOP`（可写 TOP5）  
   - 内容：**柱状图** 或 **横向条形示意** 或静态 **排名列表**（模型名 + 条形比例灰条）。

3. **最近请求日志**  
   - 区域标题：`最近请求`  
   - **表格列**（示例）：`时间` | `模型` | `状态` | `耗时(ms)` | `用户/密钥`（可选）  
   - 至少 **5～10 行** 假数据；状态用 **标签** 样式（成功绿、失败红、灰）。

## 七、各子页面最低要求（除工作台外）

每个子页与工作台 **共用侧栏**；仅 **主内容区** 变化。每页包含：

- 与模块一致的 ** `h1` ** 与 **一段说明**（灰色）。
- **筛选条占位**：时间范围、状态下拉、关键词输入、查询按钮（可无真实逻辑）。
- **数据表格**：**≥5 行** 示例数据；表头固定风格；行尾 **操作** 列：`编辑` `删除` `启停` 等 **链接样式按钮**（`href="#"` + `preventDefault` 或 `button` 禁用均可，需标注)。
- **供应商 / 模型 / Key** 三页：额外放置 **「新建」主按钮**（主色填充），点击可 **弹窗占位** 或跳转 **`edit-*.html` 空表单页**（二选一，在输出说明里写清）。

**监控中心** 可拆块：**日志表格** + **错误率/延迟 KPI 小卡** + **图表占位**，不必一次做全 OpenRouter 深度，但 **信息分区** 要清晰。

**文档管理**：列表 + 右侧或下方的 **Markdown 编辑器占位**（`textarea` + 说明「此处接编辑器」即可）。

**用户与权限**：至少 **用户表** + **角色表** 两个子区块（或 Tab），再加 **操作审计** 表格占位。

---

## 八、交互与反馈（原型级）

1. **统一圆角**：按钮、输入、卡片、表格容器一致（见第三节）。
2. **Hover**：侧栏项、表格行（可选）、卡片、文字按钮均有可见反馈。
3. **Loading**：若某区域预留异步，可用 **骨架屏灰块** 或 **Spin 占位图**（纯 CSS 旋转亦可），并标注「接接口后替换」。
4. **空状态**：文档数为 0 时，文档列表区显示 **空状态插画/图标 + 引导文案**。
5. **错误提示**：原型可不做全局 Message；若做，用 **静态 Toast 示例** 一条即可。

## 九、与 OpenRouter 对标的验收口径（产品向）

| 模块 | 原型中应让人一眼看出 | OpenRouter 参照 |
|------|----------------------|-----------------|
| 工作台 | 一屏看清「规模 + 趋势 + 最近调用」 | Dashboard / Usage 总览感 |
| 监控 | 日志 + 成功率/延迟/用量 | Usage & Logs |
| 供应商 | 多上游、启停、权重联想 | Providers / Endpoints 管理感 |
| 模型 | 目录、价格、上下文、上下线 | Models 目录管理感 |
| API 密钥 | 密钥生命周期、额度 | API Keys |
| 文档 | 与 API 说明强相关 | 可自定义加强 |
| 用户权限 | 企业运营必备 | OR 无完全同款时标注「增强模块」 |

## 十、给 AI 的一行「执行指令」（可原文粘贴）

请根据本文档生成 **多页静态 HTML 后台原型**：共用 **浅色蓝灰科技风**（主色 `#2563EB`，背景 `#F8FAFC`，主文字 `#1E293B`），**左侧固定导航 + 右侧主区**，侧栏含 **七项菜单 + 底部退出登录（确认后跳转登录页）**；**工作台** 含 **四张 KPI 卡**、**近7日趋势区**、**TOP 用量区**、**最近请求表格**；其余模块页含 **筛选占位 + 表格 + 新建按钮占位**；**禁止**使用 React/Vue 与打包工具；图表用 **CDN+假数据** 或 **纯 CSS/SVG 占位** 并在 HTML 注释中说明；所有链接相对路径可本地双击打开浏览。
