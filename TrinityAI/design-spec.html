筛选 · 应用侧

三种形式用于在产品内统一视觉与控件尺寸，避免各页各写一套。约定「形式1 / 形式2 / 形式3」即可：形式1 渐变「更多」+ icon 菜单；形式2 分组药丸 + 列表 + 右侧 ✓；形式3 文案标签 + 芯片分段。色与圆角见 设计色板。PC 与移动端须同一套比例下自适应（优先用 rem 随根字号，窄屏再辅以断点微调字号/内边距/触控面积），脚注为桌面基线的字号与行高。

备注：后台管理筛选、删选用形式2（标签 + 分组下拉；灰阶样式在控制台实现，字号行高与本节形式2 对齐即可）。

弹窗 · 页面内对话层

遮罩与卡片与对话「添加模型」同源（--orc-modal-overlay-*、--orc-modal-card-*）。卡片内从上到下依次为：

  1. 头部（顶栏）：推荐 header 内 div.or-modal-head：左侧为标题区（h2.or-modal-title；说明按钮、文字链如需与标题同一行，用 flex 容器包裹，见对话「添加模型」orc-picker-modal-head + orc-picker-head-main）；右侧为 button.or-modal-close（「×」仅作字形，须配 aria-label="关闭"）。
  2. 中间：可选说明条（如 p.or-keys-editor-banner）+ 主内容（表单、列表等）。
  3. 底部（操作栏）：右下角对齐；次要 or-btn-outline、主路径 btn btn-gradient。底部主/次按钮仅文字，不加图标（不与工具条带图标的「完成」混用）。

画板内为静态示意；业务页使用 or-modal-root 全屏层、body.or-modal-open 等与产品一致。表单项内下拉与筛选形式 2相同（药丸 + listbox + ✓）。弹窗底栏按钮不加 SVG / 图标。

多选列表 · 形式1（模型行）

多选列表用于在同一纵向列表中勾选或取消多项；本节只冻结列表容器与行（item）的结构与样式，作为通用原子，不绑定任一具体业务页或组件实现。

  1. 列表容器：div.orc-model-list；内边距 0.15rem 0.5rem 0.65rem；纵向 flex，分组块之间 gap: 0.2rem。每组为 div.orc-side-model-group（纵向 flex、gap: 0.3rem，含分组标题与多行）。多选行外包 orc-side-model-group--collection 时语义与间距约定相同。列表容器 role="group"，aria-label 须概括用途（如「可多选」）。
  2. 行卡片（item）：button.orc-model-row.orc-model-row--compare，全宽、左对齐、圆角 var(--radius)、默认边框 1px solid rgba(15, 23, 42, 0.06)、背景 var(--bg)、内边距约 0.5rem 0.55rem、字号 0.8125rem、字重 500。从左到右：span.orc-pv（圆标约 18px，可加 orc-pv--google 等品牌修饰类）→ span.orc-model-row-name（flex: 1，过长省略）→ span.orc-model-row-fav（收藏，☆ / ★）→ 已选时 span.orc-model-row-mark 显示「✓」（品牌色）。
  3. 选中与高亮：已选：class="is-selected"、aria-pressed="true"，并渲染 orc-model-row-mark 为「✓」。若业务需要「当前主项」与多选并存，可加 is-active（浅蓝底 + 蓝边框 + 主文案 var(--blue)，与 ✓ 可同时出现）。未选行悬停略加深边框。
  4. 实现注意：收藏星 orc-model-row-fav 点击需阻止冒泡，避免触发行主点击（切换多选）。

画板为静态示意；本节不含集合顶栏等周边模块。业务实现可自行选用或扩展类名，以本节结构与可访问性约定为准。

按钮原子

主渐变表示高成本/主路径，线框表示可反悔或次要操作，文字链表示跳转学习；色跟 --grad 与全局 ring。下表为仓库类名。
