# Trinity AI 运营后台 · 原型总览

> **应用**：`apps/trinity-ai-admin`（亦可嵌套于 `apps/trinity-portal` 的 `/trinity-ai-admin`）  
> **工程真源**：`src/views/<模块>/` 目录；路由真源 `src/trinityAdminRoutes.ts`；侧栏 IA `src/views/admin-shell/adminNavTree.ts`、`moduleSecondaryPages.ts`  
> **交付规范**：各模块按仓库 **[《Trinity 原型模块 · 目录与交付规范》](../../../docs/Trinity原型模块目录与交付规范.md)** 的 **「五件套」** 组织（见下文 §2）。  
> **后端对齐**：页面能力对照见同目录 **[创建后端需实现页面与功能.md](./创建后端需实现页面与功能.md)**。

## 1. 五件套（交付标准）

与全站原型约定一致，**每个业务模块**在 `src/views/<模块文件夹>/` 下尽量具备：

| # | 文件 | 职责 |
|---|------|------|
| 1 | `*Page.vue` | **唯一路由级入口** + 整页 `<template>`（用 HTML 注释分区，**不**在模块内再拆子 `.vue`） |
| 2 | `*.css` | 本模块布局与皮肤（须 **非** `*.module.css`，避免 `or-*` / 业务类名被哈希） |
| 3 | `*Interactions.ts` | 事件、`document`/`localStorage`、与演示相关的副作用 |
| 4 | `mock.ts` | 假数据与**无 DOM** 的纯函数 |
| 5 | `README.md` | 路由表、依赖样式、接 API 清单、二次开发登记（见规范 §6） |

**规范全文**：[`docs/Trinity原型模块目录与交付规范.md`](../../../docs/Trinity原型模块目录与交付规范.md)

### 1.1 运营后台特有约定

| 项 | 说明 |
|----|------|
| **多二级共一入口** | 侧栏「子菜单」多为同一路由组件内的 **Tab / `v-show` 分区**（`route.meta.stubSecondaryId` 或 `billingTab`），而非每个子页单独一个 `*Page.vue`。二级清单真源：`admin-shell/moduleSecondaryPages.ts`。 |
| **壳层不算业务五件套** | `admin-shell/`、`admin-login/` 为布局/登录，结构见 §4。 |
| **共享组件** | 列表查询、分页、日期、弹窗等见 `src/components/`（如 `AdminListQuery`、`AdminTablePagination`），**不**算模块内拆分子组件。 |
| **Element Plus** | 表格/表单等通过 `installAdminUi.ts` 统一注册；业务样式叠加 `src/styles/admin-*.css`。 |
| **若依式列表** | 检索 + 表格 + 分页；真源 **[运营后台-若依式列表规范.md](./运营后台-若依式列表规范.md)**（参考 `admin-keys/KeysPage.vue`） |

### 2.1 API 密钥 · 产品说明（通俗）

> 详版：[`admin-keys/README.md`](../src/views/admin-keys/README.md) **§1.1**

| 子菜单 | 做什么 |
|--------|--------|
| **平台密钥** | 管公司对接**上游大模型**的 API Key：列表查询、新建/编辑/删除、冻结/解冻/吊销；看明文需二次验证。 |
| **用户密钥** | 管**客户**在 Workspace 里的 API Key：按用户/空间查、调 RPM/日限额、冻结/解冻、撤销。 |
| **审计轨迹** | 查**后台谁对哪把 Key 做了什么**（如冻结及弹窗里填的原因/备注），不是 API 被调用了多少次。 |

与 **用量日志**（调用侧）、**系统 → 操作审计**（全后台写操作）的区别见模块 README §1.1。

### 2.2 实时大盘 · 产品说明（通俗）

> 详版：[`admin-ops/README.md`](../src/views/admin-ops/README.md) **§1.1、§2**（2026-05-19 版式定稿）

侧栏顺序：**工作台 → 实时大盘 → API 密钥 → …**。单页 `ops/live`，自上而下：

| 顺序 | 看什么 | 说明 |
|------|--------|------|
| — | **分工说明** | 与工作台差异：本页 = 同窗趋势、占比、Top、供应商 |
| ① | **值班 KPI** | QPS、错误率、P99（大盘独有深度指标） |
| ② | **趋势 + 错误分布** | 近 7 日调用折线、近 24h 错误码饼图 |
| ③ | **模型调用占比** | 小饼 + 表；Token 占比明显高于请求时标黄 |
| ④ | **五类监控时序** | SSE / Token / Spend / 活跃 / 稳定性迷你柱；与工作台当前值同源，可折叠 |
| ⑤ | **异常与根因** | 错误 Top、延迟 Top（可下钻用量日志） |
| ⑥ | **供应商健康** | 线路成功率、P95、容量、状态 |

**不是**：单条 API 请求明细（→ 用量日志）、IP 限流规则配置（→ **风控**）。**工作台**保持短窗摘要 + 待办 + 快捷入口；完整值班排查进本页。

### 2.3 风控 · 产品说明（通俗）

> 详版：[`admin-risk/README.md`](../src/views/admin-risk/README.md) **§1.1**

侧栏模块名 **「风控」**（限流/拉黑；**不是**实时大盘）。

| 子菜单 | 做什么 |
|--------|--------|
| **风控规则** | 配窗口、限流阈值、拉黑阈值、TTL、启停；限流阈值须 ≤ 拉黑阈值。 |
| **风控动作日志** | 查已发生的 IP 限流/拉黑（动作、范围、命中规则、起止时间）。 |

**不是**：QPS/错误趋势（→ **实时大盘**）、单条 API 调用（→ 用量日志）、后台 Key 操作（→ API 密钥 → 审计轨迹）。

## 3. 路由命名速查

- **前缀**：`tai-admin-`
- **单页**：`tai-admin-dashboard`、`tai-admin-ops-live`
- **带子菜单**：`tai-admin-<模块 pathBase>-<二级 id>`，例如 `tai-admin-keys-platform-keys`、`tai-admin-billing-usage`
- **注册**：`getTrinityAdminChildRoutes()` in `src/trinityAdminRoutes.ts`
- **侧栏**：`ADMIN_NAV_TREE` in `src/views/admin-shell/adminNavTree.ts`
