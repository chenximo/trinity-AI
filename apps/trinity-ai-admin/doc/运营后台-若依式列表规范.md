# 运营后台 · 若依式列表规范

> **真源实现**：`src/views/admin-keys/KeysPage.vue`（平台密钥 / 用户密钥 / 审计轨迹三表）  
> **适用范围**：运营后台所有 **「检索 + 筛选 + 表格 + 分页」** 子页（计费、供应商、模型、客户、用户、权限、系统、风控等）。  
> **非列表页**（工作台、实时大盘、文档编辑器等）不适用，见文末 §8。

与 Element Plus / 全局样式关系：

| 层级 | 路径 | 职责 |
|------|------|------|
| 组件约定 | 本文 | DOM 结构、工具栏、表格列、操作列 |
| 列宽档位 | `src/utils/adminTableColumns.ts` | `ADMIN_TABLE_COL` / `ADMIN_TABLE_COL_OPS` |
| 表头与单元格 | `src/styles/admin-ruoyi.css` | `.admin-ep-table-wrap`、`.admin-ep-row-actions` |
| 查询栏布局 | `src/styles/admin-page.css` | `.admin-section-head--toolbar` |
| App 总览 | [`README.md`](../README.md) | UI 栈、色彩 |

交付计划交叉引用：[`docs/AI-API聚合平台-运营后台-原型交付计划.md`](../../../docs/AI-API聚合平台-运营后台-原型交付计划.md) **§2.1**（工具栏齐平、筛选形式）；产品备忘 **B-01、B-05**。

## 1. 页面结构（固定骨架）

面包屑 / TagsView 已由壳层展示 **当前二级页名称**，卡片内 **不再重复大标题**。

```
{module}-page                          ← 模块根，如 key-page / risk-page
└─ section.{module}-page__panel       ← v-show 切换子路由分区
   └─ el-card.admin-ep-card.{module}-page__panel
      ├─ AdminSectionHead [toolbar-only]
      │  ├─ #annot   → AdminInternalTip（ⓘ 产品说明）
      │  └─ #tools   → AdminListQuery
      ├─ el-table.admin-ep-table-wrap
      ├─ AdminTablePagination
      └─ p.{module}-page__hint（可选，底栏弱说明）
```

### 1.1 禁止写法（导致「很 low」的常见原因）

| 禁止 | 应改为 |
|------|--------|
| 卡片内 `AdminSectionHead` **带 title + #desc** 与工具栏并排 | **`toolbar-only`**，说明进 ⓘ |
| 工具栏与表格之间大块 `__context` 黄条 / 重复分工文案 | 收进 `AdminInternalTip` |
| 数据列 `width="88"` / `align="right"` / `align="center"` | 仅 `min-width` + 档位常量，**默认左对齐** |
| 操作列裸放一个 `el-button`，无 `admin-ep-row-actions` | 见 §5 |
| `el-table` 加 `border` / `stripe` | 只用 `admin-ep-table-wrap`（若依横线表） |
| 多列都设很大 `min-width` 把表撑死 | 主列 1～2 个用 `xl`/`primary`，其余 `sm`～`md` |

## 2. 查询工具栏（AdminSectionHead + AdminListQuery）

### 2.1 AdminSectionHead

```vue
<AdminSectionHead toolbar-only>
  <template #annot>
    <AdminInternalTip heading="… · 原型" explain="…">
      <p>…</p>
    </AdminInternalTip>
  </template>
  <template #tools>
    <AdminListQuery … />
  </template>
</AdminSectionHead>
```

- **必须** `toolbar-only`：底部分割线、检索与筛选同一行、右侧主操作齐平（样式在 `admin-page.css`）。
- **不要** `#desc` 槽（副标题由面包屑承担）。
- `title` 属性在 `toolbar-only` 下可不传。

### 2.2 AdminListQuery 布局

| 区域 | 插槽 | 内容 |
|------|------|------|
| 左侧主区 | 默认 + `#filters` + 默认插槽（日期） | 搜索框 → `el-select` 筛选 → `AdminDateRangePicker`（若需要）→ **重置** |
| 右侧 | `#actions` | **新建**（`type="primary"`）、`AdminExportCsvButton` 等 |

约定：

- 搜索 `placeholder` 写清可搜字段（如「指纹前缀、名称、客户…」），宽度由全局 CSS 控制（约 16rem）。
- 筛选下拉 `placeholder` 用「状态」「动作」「客户组织」等短标签；`style="width: 7rem"` 或组织类 `10rem`。
- **重置**：`@reset` 里清空本表所有筛选 ref（含日期范围），不只清搜索词。
- 主操作（新建/导入）放 `#actions`，与密钥页一致，**不要**放在 filters 里。

## 3. 表格（el-table）

### 3.1 容器属性

```vue
<el-table
  :data="pg.paginatedRows"
  row-key="id"
  class="admin-ep-table-wrap"
  style="width: 100%"
  :default-sort="{ prop: 'updatedAt', order: 'descending' }"
>
```

| 属性 | 要求 |
|------|------|
| `class` | **`admin-ep-table-wrap`**（必填） |
| `style` | `width: 100%` |
| `border` / `stripe` | **不加** |
| `table-layout: fixed` | **禁止**（全局勿覆盖） |
| `default-sort` | 建议：时间类列降序 |

### 3.2 数据列

```vue
<el-table-column
  prop="displayName"
  label="密钥名"
  :min-width="ADMIN_TABLE_COL.primary"
  sortable
  show-overflow-tooltip
/>
```

| 规则 | 说明 |
|------|------|
| 宽度 | 只用 **`:min-width="ADMIN_TABLE_COL.*"`** 或 `ADMIN_TABLE_COL_MIN.detail` |
| 对齐 | **不写 `align`**（与 API 密钥一致，全部左对齐） |
| 排序 | 需要则 `sortable` |
| 长文本 | `show-overflow-tooltip` |
| 主列 | 1 列用 `xl` 或 `primary`（名称、客户/项目等） |
| 指纹 / ID / 时间 | 可用 `key-page__mono` 等模块 **等宽字体** 类，**不要**为此单独右对齐 |
| 双行主从 | 模板内 `<br />` + 次要行 `__muted`（见 Keys 客户/项目列） |

**档位速查**（`adminTableColumns.ts`）：

| 常量 | 约宽 | 典型列 |
|------|------|--------|
| `xs` | 80 | 状态、动作、绑定数 |
| `sm` | 96 | 日期、短数字 |
| `md` | 112 | 短文案、策略版本 |
| `lg` | 128 | 时间、指纹 |
| `xl` | 144 | 客户/项目、用户/Workspace |
| `primary` | 120 | 主标题列 |
| `MIN.detail` | 200 | 说明、备注 |

### 3.3 自定义单元格（状态、枚举）

```vue
<el-table-column prop="status" label="状态" :min-width="ADMIN_TABLE_COL.xs" sortable>
  <template #default="scope">
    <template v-if="scope?.row">
      <span :class="statusBadgeClass(scope.row.status)">{{ scope.row.status }}</span>
    </template>
  </template>
</el-table-column>
```

- **必须** `v-if="scope?.row"`（避免 EP 表头占位行报错，备忘 B-05）。
- 状态徽章类名建议：`{module}-page__badge` + `--ok` / `--warn` / `--danger` / `--muted`（参考 `keys.css`）。

## 4. 操作列（唯一可固定 width 的列）

```vue
<el-table-column label="操作" :width="ADMIN_TABLE_COL_OPS.lg" fixed="right">
  <template #default="scope">
    <template v-if="scope?.row">
      <div class="admin-ep-row-actions" @click.stop>
        <el-button link type="primary" :icon="View" @click="openDetail(scope.row)">详情</el-button>
        <el-button link type="primary" :icon="Edit" @click="openForm('edit', scope.row)">编辑</el-button>
        <!-- 危险操作用 type="danger" + 二次确认弹窗 -->
      </div>
    </template>
  </template>
</el-table-column>
```

| 规则 | 说明 |
|------|------|
| 宽度 | `ADMIN_TABLE_COL_OPS.sm`（1 个操作）/ `md`（2～3）/ **`lg`（4+ 或 详情+编辑+冻结…）** |
| 固定 | `fixed="right"` |
| 容器 | **`div.admin-ep-row-actions`** + `@click.stop` |
| 按钮 | `el-button` **`link`** + **`@element-plus/icons-vue`** 图标 + 文案 |
| 危险 | `type="danger"`（删除/吊销），须 `AdminDialog` 二次确认 |
| 禁止 | 单独 `el-switch` 列、无图标纯文字漂在列中 |

## 5. 分页与底栏

```vue
<AdminTablePagination
  v-model:current-page="pg.currentPage"
  v-model:page-size="pg.pageSize"
  :total="pg.total"
/>
```

- 数据源：`useAdminTablePagination(filteredRows)`（`src/utils/adminTablePagination.ts`）。
- 筛选在前端用 `filterByQuery` + 各筛 ref 组成 `computed` 再传入分页。

可选底栏（虚线弱提示）：

```vue
<p class="{module}-page__hint">…</p>
```

样式对齐 `admin-ruoyi.css` 对 `[class$="-page__hint"]` 的处理。

## 8. 不适用本规范的页面

| 类型 | 示例 | 替代 |
|------|------|------|
| 仪表盘 / 图表 | 工作台、实时大盘 | 分区卡片 + 图表占位，无列表查询栏 |
| 编辑器 | 文档编辑 | 富文本 / 表单页 |
| 权限矩阵 | 角色权限勾选 | 表单 + 树，非 el-table 列表 |
| 纯说明 Tab | 数据范围说明 | 静态文案 |
