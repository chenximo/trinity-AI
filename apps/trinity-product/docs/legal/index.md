# 法律与协议 · 总览

> **本目录定位**：给产品、运营、商务、法务协同时用的 **通用内容清单**（写什么、放哪、怎么验收）。  
> **适用范围**：任意 **ToB 产品**（企业 SaaS、开发者平台、API、云服务、行业软件等），不限于某一业务线。  
> **不是挂网真源**：用户可见正文在官网仓库维护，本手册不复制全文。

## 文档集一览（目录树缩略）

完整展开见 **[页面清单](./pages-and-content#1-文档集目录树总览)**。

```text
{官网}/
├── /contact                 # P0 联系我们
├── /legal/
│   ├── /privacy             # P0 隐私政策
│   ├── /terms               # P0 服务条款（含计费、账户、终止…）
│   ├── /acceptable-use      # P0 可接受使用（可合并进 terms）
│   ├── /refund              # P0 退款（可合并进 terms#refund）
│   ├── /cookies             # P2 [可选]
│   └── /dpa                 # P2 [可选]
└── 站内展示/                # 页脚、注册勾选、定价说明…
```

**P0** = 多数 ToB 产品上线前应具备；具体裁剪见 [按产品类型](./pages-and-content#4-按产品类型的裁剪建议)。

## 与对外文档站的关系

| 站点 | 职责 |
|------|------|
| **官网法律页** | 具有法律约束力的完整正文 |
| **开发者文档站** | 用户向说明，**引用**官网法律链 |
| **本产品手册（本目录）** | 内部通用规范 + 工程映射 |

规则：**帮助文档与法律页冲突时，以法律页为准。**

---

## 变更流程（简）

1. 对照 **[目录树](./pages-and-content)** 确认要改哪份文档、哪一节  
2. 编辑挂网真源（Trinity → `TrinityAI-web/packages/ui/src/legal/content/*.ts`）  
3. build → 确认静态法律页可用于审核爬虫  
4. 部署生产 → 更新商务提交材料中的 URL
