# Trinity 产品手册

## 前言

本手册是 Trinity **唯一的产品导航与阶段真相入口**（与对外开发者文档 `trinity-docs`、客户 API 文档分离）。产品经理、研发、测试与对内宣讲，优先从这里对齐 **有什么、本阶段验什么、做到哪了、去哪看原型与 PRD**。

::: tip 一条主线
看清全貌（地图）→ 锁定阶段（Roadmap）→ 拆到可验收（子能力）→ 一键跳到 PRD 与原型（交付包）→ 走查与 Bug 去飞书。
:::

## 子模块页写法

业务线 **总览** 管地图与阶段；**子模块**（侧栏叶子页）统一四段，正文不重复体验/在线地址：

| 段落 | 写什么 |
|------|--------|
| **页头说明** | 与业务线总览「一句话介绍」同级：`> **说明**` + 2～4 句（用户价值、边界）；不写单独 `## 说明` 小节 |
| **工程定位** | `> **工程**`：仓库路径 · 路由 / hash；体验/在线见该业务线总览 |
| **子能力清单** | **一张表**：子能力 · 当前已做 · 5.30 验收 · 6.30 商用 · 说明（复杂模块可按小节拆表，仍属同一张清单） |
| **附录** | 其余补充：页面区块/锚点、5.30/6.30 验收勾选、模块关联、PRD/规范链、变更记录等 |

需求型子页（如模型详情页需求）可省略子能力表，用 **范围边界 + 路由** 替代，补充内容仍放附录。

**YAML 子能力表**：**一个子目录一个表** → `docs/…/子目录/roadmap.yml`（如 `user/models/roadmap.yml`）。叶子页写 `<ProductRoadmap />` 即可（默认同目录 `roadmap.yml`）；多页共用一表时用 `rel` + `prefix`（如 AI 云 `ai-cloud/roadmap.yml` + `prefix="营销首页"`）。编辑/展示逻辑全在共用组件，**不必每页加代码**。localhost 下可 **编辑子能力表**（表格 + YAML）；静态部署只读。
