# 产品经理必备技能

> **文档类型**：全站通用 · **PM 能力自检地图**（看自己具备什么、欠缺什么；不是文档清单，不是 L0 正文）。  
> **适用**：GEO · AI API 聚合 · AI Cloud · Agent · 任意新产品线。  
> **姊妹文档**：[产品经理工作手册](./产品经理工作手册) = **交什么文档、按什么节奏交付**；本文 = **会不会、强不强**。  
> **补强落点**：能力欠缺时，`→` 后链到工作手册、L0–L3 或各线专文对照补课。

**图例**

| 符号 | 含义 |
|------|------|
| **初** | 初级 PM（约 0–1 年 · 带教下能交付） |
| **中** | 中级 PM（约 1–3 年 · 能独立扛模块） |
| **高** | 高级 PM（约 3 年+ · 方向、商用闭环、组织影响） |
| **⭐** | 上岗到能独立扛模块前应掌握 |
| **◆** | 进阶（高级向 · 按需补齐） |
| **○ / ● / ●● / ●●●** | 该职级对此枝的要求强度（见 §二） |
| `→` | 对照工作手册 / 手册真源时的落点 |

**成熟度四档**（自检时每片叶子勾一档）：**不会** → **需带** → **独立** → **能定标/带人**

---

## 一、通用还是偏 ToB？

| 部分 | 通用性 | 说明 |
|------|--------|------|
| 思维、用户、市场、拆解、协作、验证、数据、执行 | **高** | 多数 C 端 / B 端 PM 均适用 |
| 风险（合规、边界） | **中高** | B 端更重；C 端也要，深度因产品而异 |
| **商业化**（计费、COGS、配额、三处口径、对账） | **偏 ToB / 平台** | 按量、Credits、控制台与运营对齐——典型 API / SaaS / 企业产品 |
| 领域判断力、各线 `pm-handbook` | **Trinity 语境** | 各产品线专文补充 |
| `→` 后的 L0–L3、YAML | **Trinity 交付载体** | 能力可迁移；落点按 [工作手册](./产品经理工作手册) |

**消费级 / 纯内容产品**：商业化枝可只保留「商业假设」；其余枝仍可做自检。  
**B2B、API 聚合、按量计费**：整棵树 **高度贴合**，商业化与风险权重应调高。

---

## 二、职级与能力枝对照

职级为 **行业参考区间**，非公司职级 title；自检时按 **行为** 勾成熟度，不按工龄。

| 能力大枝 | 职级侧重 | 初 | 中 | 高 | 标记 |
|----------|----------|:--:|:--:|:--:|------|
| 思维 · 根 | 全阶段基础 | ●● | ●●● | ●●● | ⭐ |
| 用户 | 懂人再写需求 | ●● | ●●● | ●●● | ⭐ |
| 市场 | 知道战场 | ● | ●● | ●●● | ⭐ |
| 拆解 | 想法 → 可开发 | ● | ●●● | ●●● | ⭐ |
| 协作 | 推得动相关方 | ● | ●●● | ●●● | ⭐ |
| 验证 | 证据改需求 | ○ | ●● | ●●● | ⭐ |
| 数据 | 知道做成了没 | ○ | ●● | ●●● | ⭐ |
| 商业化 | 卖得动、算得清 | ○ | ●● | ●●● | ⭐（偏 ToB） |
| 执行 | 跟得住进度 | ○ | ●● | ●●● | ⭐ |
| 风险 | 少踩坑 | ○ | ●● | ●●● | ⭐ |
| 持续 | 日常习惯 | ○ | ●● | ●●● | ⭐ |
| 资深 | 决策与组织影响 | — | ○ | ●●● | ◆ |

● = 该职级应重点具备 · ○ = 可弱或带教 · — = 一般不单独要求

**进阶关系**：初（带教交付）→ 中（⭐ 全枝 **独立**）→ 高（商业化写透 + 领域判断 + ◆ 资深枝）

## 三、自检用法（建议 15 分钟）

1. **快筛**：勾 §六 **10 项**（每档：不会 / 需带 / 独立 / 能定标）。
2. **展开**：10 项里 ≥2 项「不会」或「需带」→ 到 §四 找对应大枝，逐叶子对照 **具备标志**（§五 表）。
3. **定级参考**：⭐ 全枝达 **独立** ≈ 中级可扛模块；商业化闭环 + 领域判断 + 多数 ◆ 达 **独立** ≈ 高级。
4. **补短板**：叶子 `→` 链 [工作手册](./产品经理工作手册) / L0 / 计费专文；区分 **真不会** vs **会做但没写过文档**。
5. **勿用时间自检**：不按入职周数勾；只按当前行为成熟度勾。

## 五、大枝具备标志（自检标尺）

用 **行为** 判断，不绑「是否写过某份文档」。

| 大枝 | 初级（需带） | 中级（独立） | 高级（能定标/带人） |
|------|--------------|--------------|---------------------|
| **思维** | 能说清背景与痛点 | 能写楔子、边界、P0/P1 理由 | 能主持方向讨论、帮他人收窄范围 |
| **用户** | 能写 ICP 与用户故事 | 能独立访谈并产出可执行结论 | 能定义验证计划、识别假需求 |
| **市场** | 能列竞品与一句差异 | 能维护 Tier 跟踪并影响路线图 | 能输出定位叙事、预判竞争走势 |
| **拆解** | 在模板下拆页面 | 独立 PRD：规则/异常/验收齐全 | 能评审他人 PRD、统一模块口径 |
| **协作** | 参会记录、跟排期 | 能主持评审、推动阻塞解决 | 能协调无汇报线、拍板取舍 |
| **验证** | 参与内测 | 能组织内测并改 backlog | 能设计证伪实验、定义种子用户标准 |
| **数据** | 知道核心指标名 | 能定义 KPI 并周更看板 | 能建指标树、纠偏错误口径 |
| **商业化** | 知道卖什么单位 | 能写假设+套餐+试用边界 | 能定 COGS 杠杆、过商用闭环、三处对齐 |
| **执行** | 知道自己模块进度 | 能维护 roadmap/周计划/需求池 | 能多模块排期、管理依赖与风险 |
| **风险** | 知道要拉法务 | 设计期能前置合规与不做清单 | 能定红线、推动可审计方案落地 |
| **持续** | 偶发看竞品/用户 | 周常稳定执行 | 能建立团队 PM 节奏与复盘机制 |
| **资深 ◆** | — | 部分项在带教下参与 | 假设证伪、GTM、Portfolio、向上叙事能独立 |

## 六、10 项快速体检

每行勾一档：**不会** / **需带** / **独立** / **能定标**

| # | 技能 | 一句话标准 | 对应大枝 |
|---|------|------------|----------|
| 1 | 价值判断 | 为谁、解决什么、为何现在做 | 思维 |
| 2 | 用户理解 | ICP、场景、访谈结论可信 | 用户 |
| 3 | 需求拆解 | 大→小→规则/异常/验收 | 拆解 |
| 4 | 信息架构 | P0 模块与页面树可演示 | 拆解 |
| 5 | 文档与 PRD | 研发能开发、测试能验收 | 拆解 + 协作 |
| 6 | 优先级 | P0/P1 有理由，「不做」有记录 | 思维 + 执行 |
| 7 | 验证闭环 | 访谈/内测/数据驱动改需求 | 验证 |
| 8 | 指标 | 北极星 + 核心 KPI，周常在看 | 数据 |
| 9 | 商业化意识 | 卖什么、怎么量、成本是否成立 | 商业化 |
| 10 | 协作与节奏 | 评审、排期、周常、跨团队对齐 | 协作 + 持续 |

**粗判**：10 项中 ≥7 项 **独立** → 偏中级；商业化+验证+资深多项 **能定标** → 偏高级。

## 八、与工作树怎么对应

| 能力树分支 | 主要对应 [工作手册](./产品经理工作手册) 工作树 |
|------------|-----------------------------------------------|
| 思维 / 用户 / 市场 | 【战略 · L0】 |
| 拆解 / 协作 | 【地图 · L1】+【规格 · L2】 |
| 执行 | 【交付 · L3】 |
| 验证 / 数据 / 持续 | 【持续 · 全周期】 |
| 商业化 | 【商业化 · 计费】 |
| 资深 ◆ | 【资深 · 进阶】 |

**口诀**：工作树 = **交什么**；能力树 = **会不会**。欠缺能力 → 先自检 → 再对照工作树补交付物。

## 九、刻意不放进「初级必备」的

重要，但初级不必强求：**Portfolio、向上叙事、精算财务、深度技术架构、视觉设计**（会验收即可）。见 **◆ 资深枝** 与 §二「高」列。
