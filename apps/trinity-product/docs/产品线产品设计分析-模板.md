# 产品线 · 产品设计分析（L0 模板）

> **文档类型**：**产品线级** · 0→1 立项 / 内部讲解 / 评审用 **战略与领域模型真源**（不是子页 PRD，不是 roadmap 进度表）。  
> **适用**：GEO · AI API 聚合 · AI Cloud · Agent · 任意新产品线。  
> **范本**：[GEO · 产品设计分析](./geo/product-design-analysis)（已填完整版）  
> **下游文档**：页面树 → 子页 PRD / 叶子规格 → `roadmap.yml` / 周计划。

---

## 文档分层（写之前先看）

| 层级 | 文档 | 回答什么 |
|:----:|------|----------|
| **L0** | **本文** `product-design-analysis` | 为什么做、做什么形态、领域逻辑、竞品楔子、P0 边界 |
| L1 | 产品总览 / 页面树 / 子总览 | 有哪些模块与页 |
| L2 | 子页 PRD · 叶子产品规格 | 每一屏 / 每一能力怎么做 |
| L3 | `roadmap.yml` · `week-progress` | 做到哪了、本周做什么 |

**篇幅建议**：§0 写扎实（约 15～30 页）即可支撑立项；正文（市场/竞品长文）按需增补。**不要**在 L0 里抄子页交互与验收 checklist。
