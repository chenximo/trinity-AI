# Agent 产品定位与架构（v0.1）

> **文档说明**：顶层定位、分层架构、能力边界、竞品差异、Web 界面 IA；**不含**工程落地步骤。团队对齐、白板评审、需求基线的 **唯一真源**。  
> **评审结论（内部）**：结构 **7.5/10** — 适合对齐与画界面；立项前须遵守 **Phase / MVP** 切分。  
> **关联**：[Agent SDK · 产品设计](./agent-sdk-product-design)（P0 技术）· [行业现状与跨境落地](./industry-cross-border)

### 海报级一句话

**基于多模型聚合网关的可私有化 Agent 运行 SDK + Web 配置台**；服务开发者、中小企业与大型企业；跨境、政企、零售、制造等均为 **插件化场景**，非行业绑定。

### 核心底盘（不变）

| 能力 | Trinity 现状 |
|------|----------------|
| 统一多模型接入、路由、降级、计费 | **已有** · [统一 API](../platform/unified-api)、[运营上架](../operations/models-routes) |
| 天然中立，不绑定单模型 / 单行业 | **产品原则** |

### 产品怎么讲（市场三层 ↔ 技术二分）

| 市场叙事 | 技术落地 | Phase |
|----------|----------|-------|
| **内核**：Agent SDK | P0 循环 · P1 记忆 · P2 工作流 | P0–P2 |
| **应用**：低代码 Web 工作台 | Web 可视化外壳，无独立引擎 | P2 |
| **管控**：企业 Agent Infra | 审计 / 权限 / 私有化 | P3 |

技术栈：**Web 配置台 → 薄适配层 → Agent SDK → 聚合网关**（详见 §二、§八）。

### 目标人群

| 人群 | 交付 | 可只买一层 |
|------|------|------------|
| 开发者 / 集成商 | Agent SDK + 网关 | ✓ |
| 业务 / 运营 | Web 低代码平台 | ✓ |
| 政企 / 集团 | 私有化 SDK + 管控 | ✓ |

### 竞品三角

| 竞品 | 我们差异 |
|------|----------|
| **OpenRouter** | P0 对齐 Agent SDK + **Web + 企业管控** + B2B 运营后台 |
| **Coze / 百炼** | **可私有化 SDK 内核** + 多模型聚合 |
| **HiAgent 等** | 通用中立、轻量灵活；开发者 + 中小企 + 大企业全链条；无云 / 行业绑定 |

### 阶段节奏

| Phase | 交付 |
|-------|------|
| **P0** | Agent 循环 SDK · [技术详设](./agent-sdk-product-design) |
| **P1** | 记忆模块 |
| **P2** | 工作流 + Web MVP |
| **P3** | 审计 / 私有化 / 大客户 |

门禁详见 §七。

### 明确不做

重型行业 AgentaaS 大包 · P0 同时上记忆+工作流+全量 Web · 适配层重复编排 · 绑定单模型/单云 · 混淆 IDE Skill 与 Agent SDK。

## 一、核心顶层定位（展开）

### 1.1 产品二分（对标 OpenRouter）

| 层 | 角色 | 状态 |
|----|------|------|
| **Web 可视化智能体管理平台** | 配置、调试、观测的「操作外壳」 | 规划 · Phase 2 |
| **自研 Agent SDK** | 统一执行内核；所有运行能力由 SDK 承载 | P0 设计稿 · [详设](./agent-sdk-product-design) |

```text
Web 可视化平台（纯配置外壳）
        ↓ 参数下发、结果查询
薄适配服务层（鉴权/租户/JSON→Context，无业务编排）
        ↓
自研 Agent SDK（唯一执行内核）
        ↓
多模型聚合网关（已有底座）
```

### 1.2 差异化（对外叙事 vs 分期真相）

| 对外 | 对内 Phase |
|------|------------|
| SDK「循环 + 记忆 + 工作流」一体化底座 | **P0** 循环 · **P1** 记忆 · **P2** 工作流 |
| 区别于 Coze「引擎锁在云端」 | Web 必须调用官方 SDK |
| 区别于 OpenRouter「轻量循环包」 | P0 对齐 OR；P1/P2 为增强模块 |

## 三、Web 可视化平台（Phase 2）

### 3.1 界面基础版式

左侧固定导航 + 右侧主内容；空白页双新建（顶栏 + 居中）；列表页搜索与状态筛选。

### 3.2 侧边导航（四组可折叠）

| 分组 | 菜单项 | 对应 SDK |
|------|--------|----------|
| **智能体创作** | Agent、对话调试、工作流、知识库 | 循环 + P2 + RAG |
| **工具与记忆** | 插件、自定义工具、定时任务、记忆 | tools + P1 |
| **分发渠道** | 渠道管理 | 发布 / Webhook |
| **运维观测** | 模型、质量报告、日志、监控、设置 | 埋点回流 · P3 审计 |

### 3.3 界面层硬约束

1. Web **无**独立运行引擎。  
2. 配置 → 标准 **`Context`** → 薄适配层 → SDK。  
3. 结果 / 记忆 / 监控 **由 SDK 回流**，口径同源。  
4. 适配层 **不做** 编排（编排仅在 SDK 工作流模块）。

### 3.4 与 Coze 界面对标

| 维度 | Coze | 我方（规划） |
|------|------|--------------|
| 记忆 | 多在 Bot 内嵌 | 独立一级「记忆」，审计友好 |
| 观测 / 模型 | 分散在 Bot/空间 | 独立「运维观测」「模型管理」 |
| 内核 | 云侧引擎；API 偏调用已创建 Bot | **可私有化 SDK** + 可选 Web |

### 3.5 Web MVP（Phase 2 门禁）

单 Agent + 对话调试 + Trinity 模型；简单线性工作流；基础 RAG。**不做**完整插件市场、全渠道、重行业模板。

## 四、自研 Agent SDK（执行内核）

### 4.1 统一 `Context`

循环、记忆、工作流共用一套 Context，避免 Web 与 SDK 格式分裂。

### 4.2 三层模块（分 Phase，非一次上线）

```text
┌─ P2 工作流编排（trinity-agent-flow）
├─ P0 Agent 自主循环（trinity-agent-core）← 对标 OpenRouter
└─ P1 持久记忆（trinity-agent-memory）
```

**P0 · Agent 循环** — [Agent SDK · 产品设计](./agent-sdk-product-design)：多轮循环、tools、停止条件、流式、动态换模型。

**P1 · 记忆** — 短期 / 流程变量 / 长期用户；存储适配器（Redis、向量库等）；P1 MVP：短期 + 一种适配器。

**P2 · 工作流** — 串并行、分支、子 Agent、定时；P2 MVP：线性 + 单分支 DAG。

### 4.3 两种运行形态

| 模式 | 用途 |
|------|------|
| **内置服务** | Web 后台调用 |
| **独立交付** | TS/Python 包私有化，可不依赖 Web |

### 4.4 vs OpenRouter Agent SDK

| 维度 | OpenRouter | Trinity |
|------|------------|---------|
| 循环 + tools | `callModel` / `tool()` | **P0 对齐** |
| 持久记忆 | 进程内会话 | **P1 适配器** |
| 工作流 | 无官方编排 | **P2 子集** |
| 模型 | OR 目录 | **Trinity 网关** + 模型 id |

### 4.5 vs Coze

Web SaaS 为主、无可私有化完整内核；开放 API 偏调用已发布 Bot。我方：**Web 外壳 + 分模块可私有化 SDK**。

---

## 五、企业增强（Phase 3）

审计、角色权限、私有化部署包、长任务（客户侧/混合托管，**不默认**公有云托管）。模型/密钥/计费沿用 [运营后台](../operations/)；Agent 租户与审计 **增量建设**。

---

## 六、阶段节奏（立项门禁）

| Phase | 交付 | 门禁 |
|-------|------|------|
| **P0** | `trinity-agent-core` + 网关 | [SDK 设计](./agent-sdk-product-design) 验收 |
| **P1** | `trinity-agent-memory` | P0 稳定；≥1 存储适配器 |
| **P2** | `trinity-agent-flow` + Web MVP | Web 只调官方 SDK |
| **P3** | 审计 / 权限 / 私有化包 | 企业试点驱动 |

---

## 七、明确不做（展开）

云厂商式重型行业包 · P0 全量上线记忆+工作流+Web · 适配层重复编排 · 单模型/单云绑定 · 混淆 trinity-skills 与 Agent SDK。
