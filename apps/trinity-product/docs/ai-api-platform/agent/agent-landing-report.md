# Agent 项目落地报告（预研汇总）

> **性质**：产品预研结论汇总 · 方向判断与落地路线 · **非**工程实施 PRD、**非**对外销售承诺。  
> **状态**：战略讨论稿 · SDK / 可视化 Web **规划中** · 垂直样板 **待试刀**  
> **真源索引**：[产品定位与架构](./positioning-and-architecture) · [行业现状与跨境落地](./industry-cross-border) · [Agent SDK · 产品设计](./agent-sdk-product-design) · [预研 · 跨境投放](./preresearch-cross-border-ads-agent)

---

## 一、核心结论（Executive Summary）

| 判断 | 结论 |
|------|------|
| **方向** | 依托 **多模型聚合网关（已有）+ 自研 Agent SDK + 可视化 Web 孵化器**，切入跨境垂直 Agent，**方向成立、可实现**。 |
| **路线** | **通用底座 + 行业插件模板**，不自研重型行业中台（对标 HiAgent / 实在 Agent 避坑）。 |
| **差异化** | 相对 OpenRouter 补 **记忆 / 工作流 / Web**；相对 Coze 补 **可私有化 SDK 内核**；相对 HiAgent 补 **轻量、中立、双交付**。 |
| **验证策略** | **双线并行**：① **跨境客服 Agent（模拟店铺 · 可商用样板）** 验 SDK 全套能力；② **跨境投放（公司真业务）** 验真实数据与 ROI 叙事。 |
| **商用目标** | 垂直样板按 **可商用 Agent** 规格定义（非缩水 Demo）；工程上按 SDK **Phase 分期接通**；无正式客服产线时用 **Mock Tool** 模拟。 |

**一句话**：可视化平台做 **行业样板孵化**，Agent SDK 做 **可私有化运行底座**，网关做 **多模型与计量真源**；跨境客服扛 **产品化验证**，跨境投放扛 **业务化验证**。

## 二、产品形态与能力栈

### 2.1 二分架构

```text
Web 可视化智能体孵化器（配置 / 调试 / 模板 · Phase 2）
        ↓ 薄适配层（鉴权 · 租户 · JSON→Context，无编排）
自研 Agent SDK（唯一执行内核）
  ├ P0  Agent 多轮循环 + Tool 调度
  ├ P1  持久分层记忆
  └ P2  工作流编排（含子 Agent / 并行）
        ↓
多模型聚合网关（已有：路由 · 计费 · tools · 流式）
```

### 2.2 验证时要覆盖的能力（五块）

| # | 能力 | 归属 | 验证方式 |
|---|------|------|----------|
| ① | Agent 多轮循环 + tools | SDK P0 | 客服对话、投放日报 |
| ② | Tool 插件协议 | SDK P0 + 网关 | Mock 订单 / 真广告报表 |
| ③ | 持久分层记忆 | SDK P1 | 跨会话买家画像、工单状态 |
| ④ | 工作流编排 | SDK P2 | 客服分支、Listing 并行 |
| ⑤ | Web 可视化孵化器 | Web Phase 2 | 拖拽流程模板、调试面板 |

### 2.3 双交付路径（商业化闭环）

| 路径 | 客户 | 验证点 |
|------|------|--------|
| **路径 1 · SaaS** | 中小跨境卖家 | Web 拖拽搭建 → 平台运行 → 快速试用与演示 |
| **路径 2 · SDK 私有化** | 品牌方 / 独立站 ISV | 纯代码或私有化部署 → **脱离 Web 独立运行** → 嵌入自有后台 |

## 三、行业背景与赛道选择

跨境出海为 Agent **率先商业化** 赛道之一（人力贵、流程标准、合规硬、ROI 易讲）。详见 [行业现状与跨境落地](./industry-cross-border)。

**三大跨境痛点**（与双能力映射）：

| 痛点 | SDK + Web 能承接什么 |
|------|----------------------|
| 长链路 / 多角色不稳定 | 工作流 + 记忆 + 可观测（复杂正式 Agent 为终局，非第一刀） |
| 多语种 / 多系统异构 | Tool 协议 + 插件面板 + 网关多模型 |
| 合规 / 私有化分层交付 | 记忆适配器 + 审计（Phase 3）+ SDK 独立交付 |

## 四、垂直 Agent 选型（可商用样板）

### 4.1 筛选标准

1. **行业主流、付费刚需** — 跨境中小卖家 SaaS 付费意愿强。  
2. **复杂度可控** — 不依赖 RPA、屏幕抓取、垂直行业大模型；靠 **循环 + 记忆 + 工作流 + API 插件**。  
3. **双路径可验证** — 同一样板既能 SaaS 用，也能 SDK 私有化复刻。  
4. **竞品差异清晰** — 见 §七。

### 4.2 三大垂直项目（优先级排序）

#### 项目一 · 跨境多语种智能客服 Agent — **主验证线（模拟 · 可商用）**

| 项 | 内容 |
|----|------|
| **业界对标** | 易智亚马逊客服、CarrotAI、Coze 跨境客服模板 |
| **商用规格（不砍）** | 多语种自动对话；订单 / 物流 / 库存 / 退换 tool；短期会话 + 长期用户 + **工单流程状态**记忆；三分支工作流（咨询办结 / 退换货 / 合规子 Agent）；FAQ + 平台售后规则知识库 |
| **模拟边界** | 无正式客服产线 → **Mock 店铺与订单 API** 验全链路；真连 SP-API / Shopify **单列后期**，不阻塞 SDK 验收 |
| **验证 SDK** | 循环、记忆、工作流、tools、Web 孵化器、双路径 **全覆盖** |
| **难度** | **中等**（模拟偏中低；真渠道上线变高） |
| **商用分层** | 中小卖家 SaaS；大品牌 SDK 嵌入独立站后台 |

#### 项目二 · Listing 合规 & 素材生成 Agent — **第二优先级**

| 项 | 内容 |
|----|------|
| **业界对标** | 卖家精灵 AI、Jungle Scout Listing 助手 |
| **商用规格** | 多语种 Listing 文案；合规子 Agent（违禁词 / GDPR）；工作流 **并行**（关键词 + 合规）→ 串行（生成 → 校验 → 输出） |
| **边界** | 首期 **不做评论爬虫**；关键词用 API / 词表 / 导入；验证 **多 Agent 并行** |
| **难度** | 中等 |

#### 项目三 · 广告基础监控 Agent — **第三优先级 · 纯 SDK 场景**

| 项 | 内容 |
|----|------|
| **对标** | 跨境广告 AI 监控小工具 |
| **商用规格** | 外部 Cron 触发 → 拉广告数据 → ROI 阈值 → **告警**（钉钉 / 邮件）；记忆存日报；周报表 |
| **边界** | **首期只读 + 提醒**；改预算 / 关停须 **人工确认**（与投放安全线一致） |
| **验证重点** | SDK **脱离 Web** 独立运行；外部调度 + 工作流 |

### 4.3 为什么不选重型方案

| 方案 | 不选原因 |
|------|----------|
| **实在 Agent** | RPA + 垂类大模型，验证的是非 SDK 原生能力 |
| **HiAgent** | 重型政企中台，成本与范围超出 Agent SDK 验证 |
| **Coze 泛办公** | 缺垂直付费痛点，难验商业化闭环 |

## 五、双线验证策略（关键）

公司现状：**无正式客服场景（模拟）**；**投放有具体业务（真数据）**。

```text
                    ┌─────────────────────────────────────┐
                    │     Agent SDK + Web（规划中）        │
                    └──────────────┬──────────────────────┘
           ┌───────────────────────┴───────────────────────┐
           ▼                                               ▼
  主线 · 项目一 客服 Agent                          副线 · 投放 Agent
  （模拟店铺 · 可商用样板）                          （公司真业务 · Dogfood）
  验：全套 SDK + 双交付 + Web                       验：真数据 · ROI 叙事 · 案例
           │                                               │
           Mock Tool 全功能接通                          切片：日报解读 → 文案 → 告警+确认
```

| 维度 | 客服主线（模拟） | 投放副线（真业务） |
|------|------------------|-------------------|
| **目的** | SDK / Web **产品化是否够用** | **业务价值是否说得清** |
| **数据** | Mock / 沙箱 | 真实账户与报表 |
| **商用形态** | 可对外卖的 SaaS 模板 + SDK 故事 | 对内试刀 + 对外案例 |
| **风险** | 低 | 写操作须人工确认 |
| **与投放关系** | 投放引流后的询盘承接（终局串联） | 北极星行业场景 |

**投放预研详述** → [预研 · 跨境投放 Agent](./preresearch-cross-border-ads-agent)（切片 A～E、安全线、待确认清单）。

## 六、落地节奏（工程分期 vs 商用规格）

> **原则**：商用 **规格按完整 Agent 写**；工程 **按 SDK Phase 接通**；二者不矛盾。

| 阶段 | Trinity 能力 | 客服主线 | 投放副线（并行） |
|------|--------------|----------|------------------|
| **Phase 0** | P0 循环 + tools + 网关 | 多语种对话 + Mock 订单/物流 tool | 投放 **日报 / 异常解读**（只读） |
| **Phase 1** | P1 记忆 | 跨会话买家画像、工单状态 | 待确认单、跨天策略记忆 |
| **Phase 2** | P2 工作流 + Web MVP | 三分支 + 合规子 Agent + Web 拖拽模板 | 文案合规、线性投放 SOP |
| **Phase 3** | 审计 / 权限 / 私有化包 | 企业租户、审计钩子 | 告警 → **人工确认** → 写操作 |
| **标杆** | 生态 + 真 API | 真连店铺 / IM 渠道 | 多 Agent 投放全闭环 |

### 6.1 统一验证闭环（白板）

```text
自研 Agent SDK（循环 · 记忆 · 工作流 · tools）
        ↓
Web 可视化孵化器（SaaS 样板搭建）
        ↓
三大垂直样板（客服 / Listing / 广告监控）
        ↓
路径 1：中小卖家 SaaS 网页使用
路径 2：ISV / 品牌方 SDK 私有化，脱离 Web 独立运行
```

## 七、竞品差异化（跨境语境）

| 竞品 | 短板（我方切口） |
|------|------------------|
| **OpenRouter** | 无记忆 / 工作流 / 可视化；客户需从零拼流程 |
| **Coze** | 引擎在云端，**无可剥离私有化 SDK**；大客数据难本地部署 |
| **HiAgent** | 重型、生态绑定；中小试用门槛高 |
| **我方** | 网关中立 + **分模块 SDK** + **Web 孵化** + **SaaS 与私有化双路径** |

---

## 八、边界与依赖

| Trinity 自建 | 业务 / ISV 共建 | 明确不做 |
|--------------|-----------------|----------|
| SDK 三模块 + Web 孵化器 | 广告 / 电商平台 API 连接器 | 重型行业 AgentaaS 大包 |
| 网关多模型与计量 | 各国法规库生产级维护 | RPA / 屏幕抓取（首期） |
| Mock Tool 契约（验证期） | 投放优化算法 | 首期全自动改预算（投放） |
| 通用跨境基础模板（演示级） | 垂直技能包（报关、海外仓等） | 混淆 IDE Skill 与 Agent SDK |

**安全线**：凡 **花钱、上线、封号风险** 的操作 → 默认 **人工确认**。
