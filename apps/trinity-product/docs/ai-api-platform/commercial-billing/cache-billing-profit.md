> **日期**：2026-07-15

---

## 0. 一句话

折扣阶梯管「账号打几折」；**缓存盈利管「命中 token 按缓价还是按输入价（及少报 δ）」**。  
用真实命中率反推：若希望 **混合毛利率（GM A）≈10%**（或溢价 B≈10%），DeepSeek 类 δ 大约 **0.7%～1.3%**，**不宜默认 5%**。

---

## 1. 我们在赚什么（缓存利润）

### 1.1 机制

上游真实发生了缓存命中（便宜进货），对客结算时：

- **上报命中** \(h_r\)：按 **缓存价** \(P_{cache}\) 计费的 prompt 占比  
- **实际命中** \(h_a\)：usage / 上游统计的真实命中占比  

当 \(h_a > h_r\) 时，差额 token 实际已命中，但仍按 **输入价** \(P_{in}\) 收取 → 形成 **缓存超额利润**。

定义少报幅度：

\[
\delta = h_a - h_r
\]

（单位：百分点 / 比例，下文写 1% 即 \(\delta=0.01\)。）

### 1.2 与折扣阶梯的关系

| 杆 | 文档 | 作用面 |
|----|------|--------|
| 成本折 × 销售折阶梯 | [证据链](./pricing-strategy-evidence-chain) | 账号档位（1k/5k/…） |
| **缓存 δ** | **本文** | 单次调用命中段计价 |

两杆 **拆开算**；落地维度建议：**模型系列默认 + 模型例外**（不要和 \(d_{cost}\) 六档绑死）。

### 1.3 工程侧对照（现状能力）

内部约定（摘要）：

- 已配置 `token_kind=cached_input`：扣费公式含 cached × \(P_{cached\_input}\)  
- **未配置** `cached_input`：命中回落按 **input** 价（最大缓存利；日志 `pricing_cached_input_fallback_billing`）  

本文测评的 **δ**，是在「已有缓价」前提下，**有意少报命中占比** 的政策旋钮；与「根本没有缓价目」要分开登记。

## 2. 测评策略

### 2.1 目标带（产品意图）

在获得一定缓存利润的同时，控制客户感知与口径风险。  
首轮产品目标：

| 主口径 | 目标带 | 含义 |
|--------|--------|------|
| **GM A（推荐主锁）** | **约 10%**（测评亦给 3% / 5% / 8% / 10%） | 含缓存计价后的 **prompt 混合毛利率** |
| **溢价 B（感知校验）** | 对照 3% / 5% / 8% / 10% | 相对「命中全按缓存价」客户账单贵多少 |

说明：成本=刊例时，**GM A=10% ⇔ 溢价 B≈11.1%**；**溢价 B=10% ⇔ GM A≈9.1%**——不能两个「刚好都是 10%」，需选定主锁。

### 2.2 三个口径（A / B / C）

| 口径 | 公式直觉 | 用途 | 本轮地位 |
|------|----------|------|----------|
| **A. 混合毛利率** | \((R-C)/R\) | 「利润几成」 | **主测** |
| **B. 相对公允溢价** | \((R-R_{公允})/R_{公允}\) | 「比透传贵多少」 | 对照 |
| **C. 超额利 / 总毛利** | 超额 / 总毛利 | 结构占比 | 旁注（成本≈刊例时 C≈100%，区分度差） |

其中：

\[
\begin{aligned}
R &= Q\cdot\big(h_r\,P_{cache}+(1-h_r)\,P_{in}\big)\\
R_{公允} &= Q\cdot\big(h_a\,P_{cache}+(1-h_a)\,P_{in}\big)\\
C &= Q\cdot\big(h_a\,C_{cache}+(1-h_a)\,C_{in}\big)
\end{aligned}
\]

（\(Q\) 为 prompt token 量；单价为 **USD / 百万 tokens** 时，金额先按百万归一。）

### 2.3 首轮测算假设（必须写清）

| 项 | 取值 |
|----|------|
| 命中样本 | **单日**、约 **5000** 请求（运营日志） |
| 聚合 | 表内「实际 / 上报命中率」按运营导出；推算时 \(h_a\) 固定为实际列 |
| 计费范围 | **仅 prompt**（缓存不作用于 output） |
| 售价 | Trinity **刊例** \(P_{in}/P_{cache}\) |
| 成本 | **成本 = 刊例**（透传时 GM A≈0；利润主要由 δ 贡献）。有进货折的模型第二轮再叠 \(d_{cost}\) |
| 未进推算 | 输出毛利、Stripe、账号阶梯折 |

> 单日绝对美元金额很小（价廉模型 × 一天），**政策看 GM/% 与 δ，不看当日 $ 绝对值**。

### 2.4 命中率 \(h\)：为何不用「情景低/中/高」做主锚

曾用过示意带（10～20% / 40～60% / 70～90%）仅作敏感分析。  
**真实样本显示主力模型实际命中多在 ~80%～97%**，应以日志为准；情景带仅作无数据时的兜底。

### 2.5 测评步骤（可重复）

1. 导出模型级：请求数、Prompt 量、\(h_a\)、\(h_r\)（建议连续 **≥7 天** 后再定稿）  
2. 对齐刊例 \(P_{in}/P_{cache}\)（及可选进货成本）  
3. 算现状 δ 与现状 GM A / 溢价 B  
4. 对目标带（3/5/8/10%）**反推 δ**  
5. 按 **系列** 收敛默认 δ 上限；模型例外单独挂  
6. 与对外文档 / `cached_input` 价目完备性对齐后拍板

## 5. 反推公式（成本 = 刊例）

令 \(s = P_{in}-P_{cache}\)，\(R_f = h_a P_{cache}+(1-h_a)P_{in}\)（单位 token）。

**锁 GM A = \(g\)：**

\[
\delta_A = \frac{g\cdot R_f}{(1-g)\,s}
\]

**锁溢价 B = \(p\)：**

\[
\delta_B = \frac{p\cdot R_f}{s}
\]

约束：\(0 \le \delta \le h_a\)，且上报 \(h_r = h_a-\delta \ge 0\)。

## 6. 按模型推算表（主锁 GM A）

**读表约定：**

- **主列** = 目标混合毛利率 GM A → 所需 δ（政策旋钮看这里）。  
- **溢价 B** 不单独反推整表：成本=刊例时，锁住 GM A 后溢价连带固定（见下）。  
- **现状** 只作对照列（是否已高于/低于目标），不再单开一节。

| 目标 GM A | 3% | 5% | 8% | **10%** |
|-----------|----|----|----|---------|
| 连带溢价 B（约） | 3.1% | 5.3% | 8.7% | **11.1%** |

若改锁「溢价 B = 10%」，则 GM A≈9.1%，所需 δ 略小于「GM A=10%」那一列（约低 0.1～0.4pt，视模型而定）——日常以 **GM A=10% 列** 为准即可。

### 6.2 若「主锁 GM A = 10%」——建议 δ 摘要

| 系列 / 模型 | 建议 δ（GM A≈10%） | 现状 δ | 动作倾向 |
|-------------|-------------------|--------|----------|
| deepseek-v4-flash | **≈1.3%** | 3.64% | **下调** |

**明确不建议：DeepSeek 类默认 δ=5%。**  
（flash 上 δ=5% → GM A≈29%、溢价 B≈41%，明显高于 10% 带。）

## 7. 客户感知与风险（定 δ 时并列看）

| 风险 | δ 偏高时 |
|------|----------|
| 账单 vs「缓存很便宜」的预期 | 高命中 Agent / 长会话客户最受伤（恰是本样本主力） |
| 对外文档「有缓价则按缓扣」 | 少报需在对内政策写清，避免口径冲突 |
| 计量可解释性 | 实际 vs 上报长期分叉，客诉与审计成本上升 |

权衡原则：**主利润放折扣阶梯；缓存 δ 做受限副杆**，服务 GM≈10% 带，而不是尽可能榨。
