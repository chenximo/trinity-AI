## 0. 先钉死：折扣用在哪

| | 对内（选模 / 推算） | 对外（官网 / 获客页） |
|--|-------------------|----------------------|
| 上游成本折 \(d_{cost}\) | ✅ 核心参考（厚利优先） | ❌ **不出现** |
| 商务深阶梯（Mid～Enterprise） | ✅ 销售卡 | ❌ **不摊开**（或仅「大客户可谈」CTA） |
| 刊例价 / 浅优惠 | ✅ 算获客毛利 | ✅ 可展示 |
| 场景 Playlist | ✅ 绑定默认线路与成本族 | ✅ **主叙事**（卖场景，不卖折数） |

> **一句话：** 折扣与成本族是 **营销选模的计算器**，不是营销话术。

## 1. 三层结构（已拍方向）

```text
对外营销层（浅、场景化）
├── Trial Credits（试用 · 另文）
├── 场景 Playlist（精选路由列表 · 本期主交付）
├── 浅公开价（刊例为主；可选极浅自助优惠）
└── CTA：充值自助 ｜ Contact Sales

对内商务层（深）
├── 01_商务洽谈总表（成本族 × 阶梯 × 模型）
└── 02_交叉模型（同名多折 · 推荐归族 · P/W）
```

| 层 | 用户感知 | 内部依据 |
|----|----------|----------|
| Playlist | 「性价比 / Coding / 多模态…」 | 成本折 + 交叉推荐线路 + 能力标签 +（后续）测评 |
| 浅公开价 | 「明码标价、好起步」 | 刊例；自助最深宜守 GM≳20%（约 8.2，见证据链） |
| 深商务卡 | 「量大找销售」 | Excel `01` 全阶梯 |

## 2. 对内选模参考（不对外）

从商务总表抽模型进 Playlist 时，按下列 **加权参考**（非对外分数）：

| 因子 | 权重（建议） | 用法 |
|------|-------------|------|
| **成本折厚利** | 高 | 优先 \(d_{cost}≤0.65\)，其次 0.70；薄利（≥0.78）仅作「旗舰点名」或备选 |
| **交叉表推荐归族** | 高 | 同 ID 多折时，默认跟 `02` 推荐成本折与 P1 线路 |
| **刊例相对价格带** | 中 | 同场景内区分「入门 / 主力 / 旗舰」 |
| **能力匹配** | 高 | Coding / 多模态（含理解、生图、生视频）标签必须对口 |
| **试用白名单可进** | 中 | 性价比包尽量与 [试用白名单](./new-user-trial-scheme) 可重叠（便于漏斗） |
| **启停与 P1** | 中 | 只推启用；交叉场景优先 P1 主路 |

**推算示意（对内）：**  
同场景若 A 在 0.65、B 在 0.78，能力接近 → **主推 A**；B 标「旗舰备选 / 点名可用」。  
对外文案只写能力与定位，不写 0.65 / 0.78。

## 3. 浅公开价（草案）

| 项 | 草案 | 说明 |
|----|------|------|
| 标价 | **模型刊例**（挂牌） | 与控制台一致 |
| 自助用量阶梯 | **暂不强推六档**；若要做，最多浅到约 **Growth（~8.2）** 一带 | 更深进 Contact Sales |
| 页面表达 | 「按量付费 · 明码标价」+「用量较大可联系商务获取更多优惠」 | 不展示 Mid/Scale 数字表 |
| 与 Playlist | Playlist **不改计价公式**；只是推荐用哪些模型 | 避免「套餐价」与「按量价」双轨混乱 |
| 预付死套餐 | **本期不做** | Playlist ≠ 包月包 |

## 4. 场景 Playlist 定义

### 4.1 产品形态（本期）

| 项 | 定义 |
|----|------|
| 是什么 | **精选路由列表**（3～8 个 Trinity ID + 默认线路提示） |
| 不是什么 | 预付套餐 SKU、包月、独立计价合同 |
| 控制台 | 「推荐场景」入口；一键切到列表内模型 / Auto 限池（工程可选） |
| 官网 | 卡片：场景名 · 一句话 · 代表模型 2～3 个 · CTA 试用/充值 |

### 4.2 三场景（首版草案）

> 模型为 **草案候选**，须产品/运营按测评与上架再裁；括号内为 **对内** 成本折参考。  
> **多模态** 统一覆盖理解、生图、生视频，不再单独拆卡片。

#### P1 · 性价比最高（获客钩子）

| 角色 | Trinity ID（草案） | 对内成本折 | 备注 |
|------|-------------------|------------|------|
| 主力 | `deepseek-v4-flash`（及当前启用的 flash 线） | 0.70 | 高性价比生文；与试用叙事易对齐 |
| 主力 | `qwen-flash` / `qwen3.x-flash` | **0.65** | 厚利主锚 |
| 主力 | `glm-5` / `kimi-k2.x`（按上架裁） | 0.70 | 丰富国产选择 |
| 备选 | `qwen-plus` | 0.65 | 略强一档 |
| 不宜主推 | Claude 旗舰、Seedance | 0.78 / 0.97 | 薄利或非生文 |

**对外一句话：** 日常对话与批处理，成本友好、响应快。  
**对内纪律：** 尽量 \(d_{cost}≤0.70\)；交叉 Gemini 若进本包，跟 `02` 推 **0.70** 线。

## 6. 与深商务卡的衔接

| 用户行为 | 系统 / 运营 |
|----------|-------------|
| 只用 Playlist 内模型、小额自助 | 刊例（+ 可选浅折）；Trial 可用则优先白名单 |
| 点名 Playlist 外旗舰 / 月耗抬头 | 引导 Contact Sales → 按 `01` 成本族谈 |
| 点名交叉模型 | 销售先查 `02`，按推荐归族报价，不对外解释「为什么是这个折」 |
