## 0. 一句话

在 **Credits 充 $N 到账 $N** 不变的前提下：  
用 **客户档 × 商务成本折档** 做自助折扣；用 **分行费率表** 做点名跨档大客包。  
**进货成本折真源** = [模型原价折扣 · 商务原底](../pricing-sources/supplier-cost-discounts)（台账「X 折」）。  
核算公式见 [折扣差价专文](./discount-spread-tier-pricing)；本页只定**怎么卖、两种场景怎么组合**。

## 1. 共用口径（两场景都遵守）

| 项 | 约定 | 状态 |
|----|------|:----:|
| 结算 | 预充值 Credits；展示/扣费美元口径与 MVP 一致 | 已有 |
| **成本折真源** | **仅** [商务原底台账](../pricing-sources/supplier-cost-discounts)：记法为 **「X 折」**（如 `6.5`、`7.8`）。**不是**运营「产品分组」上的小数折扣字段 | **已更正** |
| 成本系数 \(d_{cost}\)（核算用） | 仅公式换算：\(d_{cost} = X / 10\)（如 6.5 折 → `0.65`）。对内口语与表格列优先写 **「6.5 折档」**，避免「0.65 折」 | 已定 |
| 后台生效折扣 | 运营在后台按商务原底**手工对齐录入**；若后台分组字段与台账不一致，**以商务原底为准并回修后台** | 流程已有 |
| 销售折 \(d_{sell}\) | 对内相对挂牌 \(L\)；或刊例 \(P\) 反推。企业再折不得导致单位毛利率破底线 | 待验证填数 |
| **GM 底线** | \(gm = 1 - d_{cost}/d_{sell}\) **≥ 20%**；破线须财务/产品特批 | **暂定** |
| **阶梯目标带** | 折后有效 GM 约 **20%～25%**（不过厚；Standard 刊例可 >25% 作锚） | **验证表 v1 §2 已按此重算** |
| 默认公开区 | Standard 维持刊例；达档后进入 20%～25% 带 | 见验证表 |
| 最深销售折 | \(d_{sell,min} = d_{cost}/(1-0.20)\)（例：成本 **6.5 折** → 对客最深约 **8.1 折** / 系数 0.8125） | 公式已定 |
| 按成本折档批量 | **同一商务成本折（如都是 6.5 折）共用一套对客销售折策略**，不逐模型拍价；例外进白名单 | 已定原则 |
| 多线路 | 同对外模型可对应台账多行（如 GPT 网聚 6.5 折 vs 南清 7.8 折）；默认一口价 + 选路；毛利按**实际命中供应商/台账行**算；浅成本线不享受深阶梯 | 已定原则 |
| 明确不做（本阶段） | 全账号「一个折」盖所有模型；充值加赠 Credits；试用包细则；缓存进主公式；Wire/月结自动化 | 已定边界 |

## 2. 场景① · 自助充值（不指定模型）

### 2.1 谁适用

- 控制台自助充值、按公开目录调用  
- **不**提交模型清单、**不**要求合同特价  

### 2.2 两维定价

```text
客户档（充值或近 30 日消耗门槛）
        ×
商务成本折档（命中供应对应商务原底「X 折」）
        →
适用销售折 / 是否允许再折
```

**客户档（门槛暂定，验证后可改）**

| 客户档 | 门槛（暂定） | 含义 |
|--------|--------------|------|
| Standard | 默认 / 未达下级 | 公开刊例（已含默认 markup） |
| Plus | 预存或近 30 日消耗 ≥ **$1,000** | 轻让 |
| **Boost** | ≥ **$5,000** | **备选过渡档**（介于 Plus 与 Growth）；对外档位过多时可删除本档 |
| Growth | ≥ **$10,000** | 主力让利档 |
| Scale | ≥ **$50,000** | 近 GM 底线；宜搭配用量观察 |

**商务成本折档（与 [商务原底](../pricing-sources/supplier-cost-discounts) 对齐）**

| 成本折档（台账） | 核算系数 | 台账覆盖（2026-07） | 自助策略原则 |
|------------------|----------|---------------------|--------------|
| **6 折** | 0.60 | 百炼 HappyHorse | 可进阶梯；空间较大 |
| **6.5 折** | 0.65 | 网聚 GPT；百炼 qwen | **样板档**；矩阵优先填此列 |
| **7.5 折** | 0.75 | 网聚 Gemini | 标准/偏紧阶梯 |
| **7.8 折** | 0.78 | YM/南清 Claude；南清 GPT、Gemini | 浅成本；自助轻折或不深折 |
| **9.7 折** | 0.97 | 南清 seedance2.0 | **默认不深折**；接近平进 |
| （未录入） | — | TokenHub、AIGC 等 | 补录商务原底后再进矩阵 |

### 2.3 矩阵（结构已定 · 格内折数待验证）

格内填写 **对客销售折（台账「Y 折」说法）** 或「刊例 / 不折」。验算须满足 GM≥20%。

| 客户档 ↓ \ 成本折档 → | 6 折 | 6.5 折 | 7.5 折 | 7.8 折 | 9.7 折 |
|-----------------------|------|--------|--------|--------|--------|
| Standard | 刊例 | 刊例 | 刊例 | 刊例 | 刊例 |
| Plus | 8.0 折 /≈25% | **8.7 折 /≈25%** | 9.5～9.6 | 不折 | 不折 |
| Boost（$5k 备选） | 7.8～7.9 /≈23% | **8.5 折 /≈23.5%** | ≈9.5 | 不折 | 不折 |
| Growth | 7.7～7.8 /≈22% | **8.3～8.4 折 /≈22%** | 9.4 贴底 | 不折 | 不折 |
| Scale | 7.5～7.6 /≈20% | **8.1～8.2 折 /≈20%** | ≥9.4 | 不折或微折 | 不折；宜加价 |

> 数字真源与卖价举例见 [验证表 v1 §2](./pricing-validation-v1)；上表为回填摘要。

新建供应/模型：先写入或归属商务原底某一「X 折」行，再继承该折档策略；**不以产品分组小数字段为准**。

### 2.4 场景① 落地节奏（验证通过后）

| 阶段 | 内容 |
|------|------|
| P0 | 矩阵作对内政策；运营按客户档**人工**调折扣；同步 §4 销售一页纸（数字验完后再对外） |
| P1 | 按「商务成本折档 × 客户档」自动下发；控制台档位/省钱可见（§4.2） |

## 3. 场景② · 充值点名包（2–5 款，可跨档）

### 3.1 谁适用

- 预存 / 合同充值达门槛（**暂定 ≥ $10,000**，可调）  
- 客户**指定 2–5 款**模型（可属不同 \(d_{cost}\) 档）  
- 需要综合商务报价，而非全站一个账号折  

### 3.2 组合方式（已定：分行报价）

**采用方案 A · 分档/分模型分行费率表**（计费真源）：

```text
充 $X,xxx
清单：
  · 模型 A @ 商务成本 6.5 折（如百炼 qwen / 网聚 GPT）→ 对客销售折 d1（≥该档底线）
  · 模型 B @ 商务成本 6 折（如 HappyHorse）→ 对客销售折 d2
  · 模型 C @ 商务成本 7.8 折（如南清/YM Claude）→ 不折或微折 d3（常单独一行）
未列模型 → 公开刊例，不享受本合同折
```

| 规则 | 约定 |
|------|------|
| 同档同折 | 清单内同一商务成本折（如都是 6.5 折）可合并为一行策略 |
| 跨档不强行统一折 | 禁止用「全局一个销售折」盖 6.5 折档与 7.8 / 9.7 折档 |
| 综合折 | 仅报价单**封面话术**（按承诺用量加权）；**扣费仍按行** |
| 若客户坚持统一折 | 只能取清单内**最紧档**（成本折最高者，如 9.7 / 7.8），或浅成本行必须拆出 |
| 打包再让利交换物 | 提高预存、年 commit、扩大清单用量承诺——不是三行一起打穿 GM |
| 点名线路 | 须写清模型 + **供应商/台账行**；未写则默认走更深成本折（更赚）的可用线路，浅成本线另计价或不进深折 |

### 3.3 合同小模板（字段）

```text
客户：
预存 / 门槛：$________
账期：预充 Credits（本阶段）
清单与费率：
  | 模型 | 供应商（台账） | 成本折（商务原底） | 对客销售折/一口价 | 是否可点名线路 |
  |------|----------------|--------------------|-------------------|----------------|
  |      |                |                    |                   |                |
未列模型：公开价 · 无合同折
commit（可选）：清单合计月耗 ≥ $____，否则下期回调
特批（若某行 GM<20%）：审批人 ____
封面综合折（可选话术）：约 ____折（不写入扣费逻辑）
```

## 4. 客户感知与销售作战（ToB）

> **目的**：把「充得多更便宜」从后台规则变成**客户看得见、销售讲得清**的方案。  
> **原则**：对外不披露商务成本折与上游供应商折扣；只讲客户档、可折范围、相对刊例的优惠与算账。  
> **数字**：门槛与「再减 p%」须样本验证回填后方可对外；本节骨架可先对齐话术。

### 4.1 定位：规则激励 ≠ 已做成性价比套餐

| | 场景①（正文） | 销售可叠加的「打动层」 |
|--|----------------|------------------------|
| 本质 | 达档后，**可折模型**单价更好，平台 GM≥20% | 推荐组合 + 算账表 + 材料 |
| 主动权 | 定价规则（哪档可折、折多深） | 荐什么模型、怎么讲 ROI |
| 未包含 | 强制只卖某套餐 SKU（见附录 A） | — |

ToB 打动客户 = **① 的可视阶梯** + **本节作战包**；需要「仅 Auto / 轻量套餐」时走附录 A（P1）。

### 4.2 「充得多更便宜」对外三处体现

| 触点 | 客户看到什么 | 设计要点 |
|------|--------------|----------|
| **充值前** | Standard / Plus /（可选 Boost）/ Growth / Scale 档位卡 | 每档：门槛金额 + 「可折清单内相对刊例更优」+ 不可折说明一行 |
| **充值后 / 账户** | 当前档位、距下一档差额 | 「当前 Growth」；进度（预存或消耗，见 §4.6） |
| **用量 / 月报** | 省了多少钱 | 「刊例应付 → 档位优惠 → 实付」三行；便于客户内部汇报 |

Models / 目录侧（建议）：

- 可参加阶梯的模型：角标或筛选项「档位优惠适用」  
- 不参加的（典型 7.8 / 9.7 折供应相关）：「刊例价 / 不参加阶梯」或合同另议  

**禁止对外**：展示上游「6.5 折进货」、供应商名与成本公式。

### 4.3 对外话术口径（暂定）

| 对内 | 对外怎么说 |
|------|------------|
| 客户档 × 商务成本折档 | 「预存（或用量）越高，**主力可折模型**单价越优」 |
| \(d_{sell}\) / factor | 「相对公开刊例，最高再享约 **p%** 优惠（以控制台为准）」或「约 **Y 折**起（限可折清单）」——**二选一，对外统一**（开放项 #7） |
| 7.8 / 9.7 不深折 | 「部分旗舰 / 多媒体模型维持刊例，可合同点名另议」 |
| 场景② 分行 | 「按您指定的 2–5 款模型分别报价，综合成本可算清」 |

### 4.4 两条销售轨道（对应①②）

| 轨道 | 适用客户 | 开口方式 | 收口 CTA |
|------|----------|----------|----------|
| **阶梯自助（①）** | 模型不固定、愿自助充值 | 「先定用量档位，主力模型随量更便宜；范围公开」 | 预存达 Plus/Growth；控制台自助 |
| **点名包（②）** | 已锁 2–5 款、要合同价 | 「按模型分行报价，不是虚高一个全局折」 | 预存≥点名包门槛 + 签费率表 |

大客同时要阶梯感 + 锁模：用 **② 费率表** 为主，封面可写「综合约 Y 折（按承诺结构）」；扣费仍分行。

### 4.5 销售最小作战包（材料清单）

验证出数后准备，P0 即可人工使用：

| # | 材料 | 内容 |
|---|------|------|
| 1 | **对外档位说明**（半页） | 门槛、可折范围、不可折说明、升级方式 |
| 2 | **算账表**（一页） | 假设月消耗 × Standard vs Growth，客户约省 $、我方仍 ≥20% GM（对内栏可不给客户） |
| 3 | **推荐模型短名单** | 对齐商务原底 6 / 6.5 折可深折族的具体 model（验证后点名） |
| 4 | **点名包费率表示例** | §3.3 填一版假想客户 |
| 5 | **FAQ** | 例：为什么部分模型不参加阶梯？→ 供应与定价结构，可合同微让；如何算省下的钱？→ 看月报三行 |

**一页纸结构（销售面客）**：

1. 场景（客服/内容/知识库等）  
2. 推荐组合（2–4 个可折模型 ± 是否要点名旗舰）  
3. 算账表（填验证后数字）  
4. 边界一句（不参加阶梯的类型）  
5. CTA：升档预存 / 或点名合同  

打动优先级建议：① 可算清的省钱 → ② 统一 API / 少对接 → ③ 主体与条款可信 → 折扣是 ① 的放大器。

### 4.6 达档判定与包装形态（设计选择）

| 项 | 建议（暂定） | 说明 |
|----|--------------|------|
| **对外主讲达档** | **预存余额达档** | ToB 好立项、好讲 ROI；近 30 日消耗可作辅助升级 |
| **产品形态 P0** | **透明阶梯（形态 A）** | 目录仍开 + 档位优惠可视化 |
| **产品形态 可选** | 充值礼包（形态 B） | 「Growth 包 $10k」= 到账 Credits + 绑定 Growth 策略，更好讲 |
| **打动增强** | 推荐层（形态 C） | 充值页/材料推荐高性价比可折模型，不改结算骨架 |

### 4.7 与附录 A 的衔接（话术勿超卖）

- 现在可讲：「用量档位优惠 + 我们帮您搭性价比模型组合」。  
- **未验证前不承诺**「轻量仅 Auto 套餐价」等附录 A SKU。  
- 场景②点名包可称为「专业清单报价」，避免与未上线的「专业版订阅」混淆。

## 5. 两场景对照

| | ① 自助 | ② 点名包 |
|--|--------|----------|

## 附录 A · 模型组层与企业套餐（P1 方向 · 本阶段不验证）

> **状态**：讨论备忘，计入产品包装演进；**不替代**正文场景①②，也不阻塞商务原底样本验证。  
> **意图**：在钱包与折扣档之上，用「模型组 + 套餐权利」约束流量迁徙与点名权，逼近完整定价结构（原「95 分」四层中的第 2、4 层）。

### A.1 与正文四层的关系（备忘）

| 层 | 内容 | 本期 |
|----|------|:----:|
| （1）钱包层 | 充 $N 到账 $N，按量扣 | ✅ 已有 |
| （2）模型组层 | Auto / 国产簇 / 国际 / 多媒体 + 轻量·专业·旗舰 | **附录 · P1** |
| （3）客户档层 | 预存/月耗 × 商务成本折档 → 销售折；GM≥20% | ✅ 正文① |
| （4）粘性 / 清单权 | 点名常用模型 = 产品或合同权利 | ② 为商务雏形；产品化见本附录 |

### A.2 模型组（命名可再磨）

| 组 | 客户得到 | 毛利机制 | 与商务原底的粗映射（示意） |
|----|----------|----------|----------------------------|
| **Auto / 智能选路** | 统一入口，不保证指定厂商 | 在 SLA/质量门槛内选更优成本线路；组刊例可略低于点名价 | 路由偏向 6 / 6.5 折可用线 |
| **主力国产簇** | 点名 deepseek / glm / qwen 等 | 阶梯折扣主要打在此组；利润贡献主力 | 多为 6 / 6.5 折档（以台账为准） |
| **国际旗舰簇** | GPT / Claude / Gemini 点名 | 成本折往往更浅 → 浅折或不折；靠聚合便利溢价 | 如网聚 6.5～7.5、南清/YM 7.8 等，按台账分行 |
| **多媒体 / 贵单价** | 图、视频等 | 默认不进深折；慎进体验包 | 如 seedance **9.7 折**等 |

「流量迁徙」应发生在**组内/路由策略内**，而不是客户在全目录任意套利。

### A.3 企业套餐形态（示意，对标「仅 Auto vs 可点名」类包装）

| 套餐 | 客户得到 | 与本期场景的关系 |
|------|----------|------------------|
| **轻量** | 钱包 + **仅 Auto**（或 Auto + 极少国产） | 未产品化；依赖稳定 Auto 后再卖 |
| **标准 / 专业** | Auto + **粘性清单点名**（如 glm-5.2、deepseek-v4-pro） | 场景②点名包 = **商务版雏形**（2–5 款分行报价） |
| **旗舰** | 再加国际点名 + 合同折 | 场景②加深 + 销售合同；仍分行守 GM |

### A.4 与客户档折扣如何叠加（P1 原则）

- 客户档带来的销售折：**只作用于「允许打折的组」**（通常国产簇 / Auto 池），并守 GM≥20%。  
- 国际旗舰 / 多媒体：可「有客户档、无折或折幅减半」。  
- **点名常用模型 = 套餐或合同清单权**，不是「全站再打一个统一折」。

### A.5 建议启动条件（何时从附录升正文）

1. 正文①②样本验证通过，矩阵数字与点名包模板已跑通 P0；  
2. Auto 选路有可对外承诺的质量/降级话术；  
3. 商务原底与线路角色稳定，组 ↔ 台账折档映射表可维护。  

升正文前：单独开一页或本节转正，并改状态为「草案 · 待套餐验证」。
