# 用量与活动

> **说明**：租户按 **当前工作区** 查看 API **调用用量**（usage）与 **账户活动**（activity）。数据来自 [平台计量](../../platform/metering-billing)；与运营 [billing](../../operations/billing) 同一聚合源。  
> **工程**：`UsageLogsView.vue` · `ActivityView.vue` · `UsageChartPanel.vue` · 路由 `…/usage` · `…/activity`  
> **域地图** → [身份与组织 · 总览](./index) · 能力域 **F + G**

---

## 1. 用户问题与边界

**问题**：开发者与财务需要知道 **谁调了什么、花了多少**——按工作区、按 Key、按模型。

**本能力是什么**：控制台 **只读** 用量与活动视图；不负责定价规则（→ 刊例与 [commercial-billing](../../commercial-billing/)）。

---

## 2. 功能范围

| 页面 | P0 | P1 |
|------|:--:|:--:|
| **用量** `…/usage` | 调用列表 / 统计摘要 | 图表 · 导出 |
| **活动** `…/activity` | 充值 · 配额变动等事件 | 筛选 · 审计级明细 |

---

## 3. 数据口径

| 维度 | 说明 |
|------|------|
| **工作区** | 仅当前 `lastLink` 下 Key 产生的用量 |
| **模型** | `model` id 与 [模型域](../models/) · 网关一致 |
| **单位** | 生文 token · 生图张 · 生视频秒（与扣费一致） |
| **时间** | 与后端 usage 服务对齐 |

---

## 4. 交互路径

```text
调 API（Key）→ …/usage 查看记录
充值 / 分配配额 → …/activity 可见事件
```

---

## 5. 验收

- 某 Key 调用成功后，在 **对应工作区** usage 可见（允许分钟级延迟以产品为准）。  
- 切换工作区后，usage 数据随上下文变化，不串区。
