# 工作区

> **说明**：企业租户的 **组织单元**：顶级工作区（签约根）与子工作区（部门/项目）。侧栏 **工作区切换器** 决定当前上下文下可见的 Key、配额与用量。  
> **工程**：`WorkspaceSwitcher.vue` · `CreateWorkspaceModal.vue` · 路由 `/account/workspace/:link` · `WorkspaceHomeView.vue` · `WorkspaceSettingsView.vue`  
> **域地图** → [身份与组织 · 总览](./index)

---

## 1. 用户问题与边界

**问题**：企业需要把不同团队或项目的 API 使用 **分开管**——不同 Key、不同配额、不同用量，而不是所有人共用一个账户视图。

**本能力是什么**：**工作区** 作为租户内的隔离边界；顶级持有 **账户余额与可分配池**，子区持有 **专属配额**。

**不是什么**：

- 不是运营侧的 [客户与合同](../../operations/customers)（那是平台方录合同）
- 不是多租户之间的隔离（租户间隔离在 IAM / 后端）

---

## 2. 功能范围

| 做（P0） | 不做（后期） |
|----------|--------------|
| 顶级 / 子工作区层级 | 跨租户联邦 |
| 侧栏切换当前工作区 | 工作区模板市场 |
| 创建子工作区（名称 · slug · 描述） | 复杂组织架构树（BU→部门→项目） |
| 工作区首页摘要（Key · 用量入口） | — |
| 工作区设置（名称等） | SSO 自动 provisioning（P2） |

---

## 3. 数据与规则

| 项 | 规则 |
|----|------|
| **顶级工作区** | 注册开通后默认存在；可充值；可见 **子工作区配额池** |
| **子工作区** | 在 **成员** 页创建；可接收 **分配配额**；不可代替顶级做全账户账单（账单在顶级） |
| **切换** | 切换后 Key、balance、usage 等 API 与页面均按 **当前 lastLink** |
| **请求上下文** | 控制台 API 带 `X-Workspace-Last-Link`（或等价机制） |

---

## 4. 交互路径

```text
登录 → /account → 默认进入上次工作区
  → 切换器选子工作区 → /account/workspace/{link}/…
  → 成员页「创建工作区」→ 新子区出现在切换器
  → 顶级 balance 页 → 分配配额到子区
```

---

## 5. 验收

- **Given** 顶级账户，**When** 创建子工作区，**Then** 切换器可见且可进入 `/account/workspace/{slug}/`。  
- **Given** 在子区，**When** 创建 Key，**Then** Key 仅在该工作区列表出现，调用计量归属该工作区。  
- **Given** 顶级，**When** 分配配额到子区，**Then** 子区 balance 显示「专属配额」。

飞书走查：筛 **工作区 / 企业多工作区**。
