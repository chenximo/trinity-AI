# 成员

> **说明**：在工作区内 **邀请、查看、移除成员**；**创建子工作区** 入口亦在本页（与组织管理同屏）。顶级工作区 **默认不可邀请成员**——需先建子工作区。  
> **工程**：`WorkspaceMembersView.vue` · `WorkspaceInviteAcceptView.vue` · 路由 `/account/workspace/:link/members` · `/invite/:token`  
> **域地图** → [身份与组织 · 总览](./index)

---

## 1. 用户问题与边界

**问题**：企业多个开发者要用同一租户，但 **不能共享同一套 Key 管理权限**；需要按工作区协作。

**本能力是什么**：工作区维度的 **成员名单 + 邮件邀请 + 接受邀请**。

**不是什么**：平台运营审核注册用户（→ [operations/users](../../operations/users)）。

---

## 2. 功能范围

| 做（P0） | 不做（P1/P2） |
|----------|----------------|
| 成员列表（懒加载） | 细粒度自定义角色 |
| 邮件邀请加入工作区 | SCIM 同步 |
| 接受邀请 `/invite/:token` | — |
| 移除成员（Key 随之禁用） | — |
| 创建子工作区 | — |
| 顶级工作区禁止邀请（引导先建子区） | — |

---

## 3. 交互路径

```text
/account/workspace/{top}/members → 创建工作区 → 填写名称/slug
  → 进入新子区 members → 邀请邮箱
  → 受邀人打开 /invite/{token} → 接受 → 切换器选该工作区
```

---

## 4. 异常

| 场景 | 行为 |
|------|------|
| 邀请过期 / 撤销 | 明确错误文案，提示 owner 重发 |
| 非 owner 创建子区 / 邀请 | 拒绝并提示权限 |
| 顶级工作区点邀请 | 引导「请先创建工作区」 |

---

## 5. 验收

- 创建子工作区后，切换器与成员列表一致。  
- 邀请接受后，被邀用户可在切换器看到该工作区。  
- 移除成员后，其在该工作区的 Key 不可用。
