## 二、符号

### 原型 / 后端

| 符号 | 含义 |
|:----:|------|
| ✅ | 100% |
| 🟡 | 进行中（可写 `🟡 60%`） |
| ⬜ | 未启动 |
| ➖ | 本阶段不做 |

### 5.30 / 6.30 节点列（由原型+后端推导）

| 条件 | 符号 |
|------|:----:|
| 原型✅ 且 后端✅ | ✅ |
| 原型✅ 且 后端未✅ | 🔵 |
| 有进展但未全⬜ | 🟡 |
| 全⬜ | ⬜ |
| 本里程碑不做 | ➖ |

公式：综合% = (原型% + 后端%) / 2。

### 周计划 `result` / 总览「当前已做」

⬜ 🟡 🔵 ✅ ➖（与上表语义一致；✅=本周或本节点已达验收）。

---

## 三、里程碑锁死

| 时段 | 5.30 列 | 6.30 列 |
|------|:-------:|:-------:|
| 至 5.30 | 可改 | 可改 |
| 5.30 后～6.30 前 | **锁死** | 可改 |
| 6.30 后 | **锁死** | **锁死** |

锁死后只改「备注」记顺延，不改该列符号。

---

## 四、`week-progress.yml`（仅 `ai-api-platform/`）

展示：`ai-api-platform/index.md` → `<ProductWeekProgress rel="ai-api-platform/week-progress.yml" />`

### 结构

```yaml
months:
  - id: 2026-06
    goal: 本月一句话目标
    weeks:
      - id: W24
        period: 06-08～06-14
        focus: 用户侧模块、平台侧模块
        owner: 姓名或角色
        plan: 本周计划
        result: ⬜
        dependencies: —
        testLink: —
        bugList: —
        blockers: —
```

历史周不删；新月追加 `months` 项。

### `focus` 标准名（可点击，勿改同义词）

- `用户侧模块`
- `平台侧模块`
- `运营后台管理平台`

多模块用中文顿号。具体子项写在 `plan`。

### 扩展信息（组件未支持前写在 `plan` 一行）

截止日 · 风险 · 范围变更（`+新增；-砍掉`）· 子清单

## 五、备注

记：里程碑起止、卡点、关键日期、是否已上线（与 ✅ 区分）。

## 六、半月众测表（产品总览）

每期更新一行；明细在飞书，手册不扩表。

## 七、更新顺序（周会）

与站点总览「管理闭环」引用块一致；**本周排期只动周计划**，不每周扫改子能力每一行。

1. 各模块 `roadmap.yml`：原型 / 后端（有实质交付时）  
2. `week-progress.yml`：`focus` / `plan` → 会后 `result`、`blockers`、`testLink`、`bugList`  
3. 子总览模块表 3 列与节点锁死规则对齐（与 roadmap 一致时改）  
4. 半月众测表（如有走查）
