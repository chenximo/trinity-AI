## 二、符号

### 原型 / 后端

| 符号 | 含义 |
|:----:|------|
| ✅ | 100% |
| 🟡 | 进行中（可写 `🟡 60%`） |
| ⬜ | 未启动 |
| ➖ | 本阶段不做 |

### 5.30 / 6.30 节点列（由原型+后端推导）

| 条件 | 符号 |
|------|:----:|
| 原型✅ 且 后端✅ | ✅ |
| 原型✅ 且 后端未✅ | 🔵 |
| 有进展但未全⬜ | 🟡 |
| 全⬜ | ⬜ |
| 本里程碑不做 | ➖ |

公式：综合% = (原型% + 后端%) / 2。

### 周计划 `result` / 总览「当前已做」

⬜ 🟡 🔵 ✅ ➖（与上表语义一致；✅=本周或本节点已达验收）。

## 四、聚合总览 YAML（仅 `ai-api-platform/`）

### `product-backlog.yml` · 产品待办池

**所属模块**：**AI API 聚合产品 · 产品总览**（`docs/ai-api-platform/index.md`），不是叶子页、不是子总览。真源文件 `docs/ai-api-platform/product-backlog.yml`（当前仅聚合线；AI 云暂无待办池）。

展示：总览 **产品待办池** → `<ProductBacklog rel="ai-api-platform/product-backlog.yml" />`

**用途**：跨用户/平台/运营的发现项、尚未写入本周 `plan` 的事项。每条用 **`module`**（模块名）+ 可选 **`leaf`**（手册叶子路径，看板可点进）标 **归属**；**不写状态、不写负责人**。要排期时迁入同页 **`week-progress.yml`**。

```yaml
items:
  - module: 平台侧 · 统一 API 基座
    leaf: platform/unified-api   # 可选，手册叶子路径，看板模块列可点
    task: 一句话任务
    target: 6.30                 # 或 W25、待定
    note: |-                     # 可多行；看板备注列换行展示，表格不横向滚动
      第一行结论
      第二行附录路径
```

承诺交付并拆子能力后，再在对应叶子 `roadmap.yml` 增行。

### `week-progress.yml` · 周计划

### 结构

```yaml
months:
  - id: 2026-06
    goal: 本月一句话目标
    weeks:
      - id: W24
        period: 06-08～06-14
        focus: 用户侧模块、平台侧模块
        owner: 姓名或角色
        plan: 本周计划
        result: ⬜
        testLink: —
        blockers: —
```

历史周不删；新月追加 `months` 项。

### `focus` · 重点模块（标准叶子页 · 看板可点）

**范围**：仅 **AI API 聚合产品** 下已上架侧栏的 **标准叶子页**（含 `ProductRoadmap` 的五件套叶子；**不含**各层「总览」子页面）。真源清单与跳转路径：

`apps/trinity-product/.vitepress/shared/weekProgressFocusLeaves.ts`（增删叶子时与 `.vitepress/config.ts` 侧栏同步）。

**YAML 写法**：`focus` 为叶子 **`title` 与手册 frontmatter 一致**，多个用 **`、`** 分隔（保存后看板拆成标签）。

```yaml
focus: 模型上架与供应线路、开发者文档
```

**看板**：匹配清单的片段 → **可点击** 链到对应叶子；未在清单中的旧文案 → 灰色虚线标签（无链接），请在编辑里改选标准叶子。

**编辑（localhost · 编辑周进度 · 表格）**：「重点模块」列用 **「+ 添加标准叶子页」** 下拉按层（用户侧 / 平台侧 / 运营后台 / API 测试）多选；已选标签点 **×** 移除。勿手打别名（如「模型上架」），须与清单 **label** 完全一致。

**可选叶子（29，摘要）**：

| 层 | 示例 label（完整见 ts 文件） |
|----|---------------------------|
| 用户侧 | 列表、模型排名、模型详情、Chat 在线体验、开发者文档、用户控制台 |
| 平台侧 | 统一 API 基座、生文 API + 流式、多模态 API、鉴权 · 限流 · 配额、路由与 Fallback、计量与计费、数据策略、标准错误与可观测 |
| 运营后台 | 运营工作台、供应商管理、**模型上架与供应线路**、密钥管理、用量与计费、监控与风控、用户注册与审核、权限与操作审计、客户与合同、文档中心（运营发布）、报表中心 |
| API 测试 | API 内测文档、生文验收台、生文验收路线图、Chat API Test |

**书写约定**：

- **本周主战场叶子** → 只把对应叶子 `title` 写入 `focus`（可多个）
- **子项 / 负责人 / 验收细节** → 写在 **`plan`**，不重复塞满 `focus`
- **风险 · 依赖 · 顺延 · 无法对应叶子**（如「6.30 收口」「法律和合规」）→ 写在 **`plan`** 或 **`blockers`（备注）**，不要写进 `focus`
- **勿**在 `focus` 里写 `http`；走查用 **`testLink`**

### `plan` 与 `blockers`（备注）

**计划**与**备注**共用同一套看板样式与编辑方式（`ProductWeekProgress` 组件）。

| 字段 | 看板列 | 写什么 |
|------|--------|--------|
| `plan` | **计划** | 本周任务与子项 |
| `blockers` | **备注** | 风险、依赖、顺延、范围变更、卡点等 |

**书写约定（`plan` / `blockers` 相同）**：

- 子项用 `①` `②` `③` … **分行**（每行一条任务或一条备注项）
- 负责人写 **`@姓名`** → 看板**蓝色高亮**
- **续行**（`模块：`、`待依赖：`、`顺延：`、`+新增；-砍掉` 等）写在上一行子项**下方**，不要新开 `①`（看板以缩进 + 虚线分隔子项块）
- 行数多时用 YAML **`|-` 多行块**；看板**完整展示全文**（不截断省略），行高随行数撑开

**编辑**：localhost「编辑周进度」→ 表格 → 点 **编辑计划** / **编辑备注** → 弹窗多行编辑（与计划同一弹窗形态）。

示例：

```yaml
plan: |-
  ① 自动化模型上架 @崔宇光
  模块：operations/models-routes
  待依赖：生图/生视频（腾讯云）
  ② 文档站与 api-test @李玲
blockers: |-
  ① 生图/生视频接续接入顺延 W24 @李玲
  模块：platform/multimodal-api
  ② 模型列表体验优化待排期
```

::: info 历史字段 `acceptance`
YAML 中可保留旧 `acceptance`（达标）字段供归档；**看板与编辑表格已不再展示该列**，验收口径请写入 `plan` 或 `blockers`。
:::

### 扩展信息（写在 `plan` / `blockers`）

截止日 · 风险 · 范围变更（`+新增；-砍掉`）· 子清单 · 依赖与顺延说明

## 六、走查 / 众测（非固定结构）

产品总览 **不设** 固定的「半月众测」表。有走查时：明细在飞书；手册只在 **能力主链** 下链一次，或写在当周 `week-progress.yml` 的 `plan` / `testLink`。

## 七、更新顺序（周会）

与站点总览「管理闭环」引用块一致；**本周排期只动周计划**，不每周扫改子能力每一行。

1. 各模块 `roadmap.yml`：原型 / 后端（有实质交付时）  
2. `week-progress.yml`：`focus` / `plan` → 会后 `result`、`blockers`（备注）、`testLink`  
3. 子总览模块表 3 列与节点锁死规则对齐（与 roadmap 一致时改）  
4. （可选）走查结论只记飞书或周计划字段，不加手册固定节
