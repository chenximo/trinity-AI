# 产品手册文档规范

> AI 生成或改写 `apps/trinity-product/docs/**` 时必读。只规定 **写什么、放哪、长什么样**。  
> 符号、锁节点、`roadmap.yml` / `week-progress.yml` 字段、周会顺序 → [产品手册更新规范](./产品手册更新规范.md)。  
> Agent 维护 → `.cursor/skills/trinity-product-handbook`（根总机 `trinity-project` 可封发）。

**真源目录**：`apps/trinity-product/docs/` · 侧栏：`.vitepress/config.ts`

## 〇、冻结样板一览

改页前先 READ 对应真源，禁止凭记忆拼结构。

| 页面类型 | 冻结真源 | 何时用 |
|----------|----------|--------|
| **站点总览** | [`index.md`](./index.md) | 全站首页 |
| **产品总览 · 多分层** | [`ai-api-platform/index.md`](./ai-api-platform/index.md) | 用户/平台/运营三层 + 周计划 |
| **产品总览 · 单层** | [`ai-cloud/index.md`](./ai-cloud/index.md) | 模块少：总览内 `## 模块总览`，**无子总览** |
| **子总览** | [`ai-api-platform/user/index.md`](./ai-api-platform/user/index.md) | 仅多分层业务线；小产品**不要**建 |
| **标准叶子页** | [`user/models/list.md`](./ai-api-platform/user/models/list.md) + [`roadmap.yml`](./ai-api-platform/user/models/roadmap.yml) | 模块运维真源（全站唯一叶子页型） |

**命名**：对外 **AI API 聚合产品**（不说「聚合平台」）；该线产品总览标题 **`# AI API 聚合产品 · 总览`**。

**复杂度分级**：

| 类型 | 结构 | 示例 |
|------|------|------|
| **多分层** | 站点 → 产品总览 → **子总览**（按层）→ 叶子 | AI API 聚合产品 |
| **单层** | 站点 → 产品总览（含模块表）→ 叶子 | AI 云 |

## 一、四种页面 · 职责

| 类型 | 职责 | 不做什么 |
|------|------|----------|
| **站点总览** | 手册是什么、能力边界、业务线入口、维护闭环 | 各线里程碑、周计划、子能力表 |
| **产品总览** | 一条产品的地图、体验/在线、跨模块主链、周计划（聚合线） | 子能力逐条、抄飞书 Bug 表 |
| **子总览** | 一层内的 **模块总览**（3 列进度） | 周计划组件、六列子能力、体验四行全文 |
| **叶子页** | 子清单真源、**工程（含原型五件套路径）**、飞书验收链、PRD 索引 | 周计划、重复总览四行、手册 checklist、PRD 全文 |

**管理闭环与真源**（维护按序，勿混写）：

| 步骤 | 真源 |
|------|------|
| 规划 · 模块树 | 侧栏、`config.ts`、产品总览 / 子总览 **模块表** |
| 规划 · 子清单 | 叶子 **`roadmap.yml`**、`<ProductRoadmap />` |
| 节点 | 子总览 / 叶子 / roadmap 的 5.30、6.30 列；锁节点见更新规范 §三 |
| 实施 | 叶子附录 PRD 链 + roadmap **原型 / 后端** 列 |
| 进度 · 累积 | 改 `roadmap.yml` → 必要时抬子总览 **当前已做** |
| 进度 · 本周 | **仅**产品总览 `week-progress.yml` |
| 验收 | 飞书 [5.30 产品测试体验 / Bug 表](https://qcn81yhei1l2.feishu.cn/sheets/PjnVs7bmphodaKtOkkycpvxmnne) + 节点列；手册不抄表 |

**周会**：有交付才改 `roadmap.yml`；再改 `week-progress.yml`；子总览表与 roadmap 一致时抬行。不得每周逐条扫改子能力表。

## 二、站点总览

**真源**：`docs/index.md`

### 2.1 冻结结构

1. `## 前言`（手册定位；与 `trinity-docs` 分工一句）
2. `::: tip 一条主线（阅读）`（阅读路径，非维护细则）
3. `> **管理闭环（维护时按序）**`（五步 + 周会一句；用 `>` 引用块，**不占** `## 管理闭环` 标题）
4. `<div class="product-handbook-cap-wrap">` + **能力矩阵** 表（六行：产品地图 / Roadmap / 周进度 / 子能力+验收 / PRD·原型索引 / 宣讲导出）
5. `## 业务线`（名称 · 说明 · 入口）

### 2.2 禁止

- 业务线「本阶段」简表、子模块四段全文、符号附录
- 重复链「文档规范 / 更新规范」大表（规范在侧栏 **手册规范**）
- 挂 `week-progress.yml` 或 `<ProductWeekProgress />`

## 三、产品总览

### 3.1 多分层 · 冻结顺序

**真源**：`ai-api-platform/index.md`

1. 页头 **四行** `>`（可连续多行，与聚合样板一致）  
   `一句话介绍` → `工程` → `体验地址` → `在线地址`
2. `## 三层分工`（mermaid + 分层表：分层 · 包含什么 · 手册入口 · 整体符号）
3. （可选）对标 tip，如 OpenRouter
4. `## 周计划与验收看板` + `<ProductWeekProgress rel="ai-api-platform/week-progress.yml" />`
5. `## 5.30 能力主链` / `## 6.30 能力主链`（各一节）
6. `## 相关链接`（PRD 路径，不贴全文）

**5.30 / 6.30 主链写什么**：几句话描述该节点的 **跨模块闭环**（主路径串讲），**不是**模块清单。模块级 ✅🟡⬜ 在子总览 / 叶子 / roadmap。走查 Bug **一行链飞书**，**不设**固定「半月众测」节。

**主链段尾固定一句**（可照搬改链）：

```markdown
各步是否已达，在子总览 / 叶子 **当前已做**、**5.30 能力** 列填写；不在此页铺模块表。

走查与 Bug 明细：[5.30 产品测试体验 / Bug 表](https://qcn81yhei1l2.feishu.cn/sheets/PjnVs7bmphodaKtOkkycpvxmnne)（手册不抄表）。
```

**周计划**：全站唯一真源 `docs/ai-api-platform/week-progress.yml`；产品总览 **周计划与验收看板** 为**重点看板**，**计划 / 达标列完整展示 YAML 全文**（多行换行，不截断）；符号见 [更新规范 §四](./产品手册更新规范.md)。

**重点模块（`focus`）**：从 **标准叶子页** 多选（清单见 `weekProgressFocusLeaves.ts`）；看板标签可点进对应叶子。子项写 `plan`，验收写 `acceptance`。详见 [更新规范 §四 · `focus`](./产品手册更新规范.md)。

```markdown
## 周计划与验收看板

<ProductWeekProgress rel="ai-api-platform/week-progress.yml" />
```

### 3.2 单层产品 · 冻结顺序

**真源**：`ai-cloud/index.md` · 侧栏文案 **产品总览**（非「业务总览」）

1. 页头四行 `>`（同 §3.1）
2. `## 模块总览`（模块 · 说明 · 当前已做 · 5.30 能力 · 6.30 能力 · 备注）
3. （可选）`## 变更记录` 或 `## 修订`

**不要**：三层分工图、子总览目录、`<ProductWeekProgress />`（无多模块周会看板时可省略周计划 YAML）。

### 3.3 产品总览 · 禁止

- 在子总览 / 叶子复制体验·在线四行全文
- 在子总览挂 `week-progress` 或 `<ProductWeekProgress />`
- 固定「半月众测记录」表（走查写飞书或周计划 `plan` / `testLink`）

## 四、子总览（多分层可选）

**小产品不需要子总览**；模块表放在 §3.2 产品总览即可。

**真源（用户侧精简）**：`ai-api-platform/user/index.md`  
平台/运营：`platform/index.md`、`operations/index.md` — 表可增加 **对标 / 说明 / 备注** 列，并加 `## 与 xxx 的关系`。

### 4.1 冻结顺序

1. `# {层名} · 总览` + frontmatter `title` 与侧栏一致
2. 页头两行 `>`（**两行之间可不空行**；与模块表之间**空一行**）  
   `> **工程**` · `> **地址**`（链产品总览；**不要**体验/在线四行）
3. `## 模块总览`（行与侧栏叶子一致，模块名可链叶子）
4. （可选）路径示例一行
5. **一行周计划**（固定句式，见下）
6. （可选）`## 与 xxx 的关系`（平台/运营建议有）
7. （可选）`## 修订`

### 4.2 模块总览表

**用户侧（最小）**：

| 模块 | 当前已做 | 5.30 能力 | 6.30 能力 |
|------|:--------:|:---------:|:---------:|

**平台侧示例**：增加 `对标 OpenRouter` · `备注` 等列。

**表表达什么**（模块级仪表盘，**不是**子能力六列表）：

| 列 | 含义 |
|----|------|
| **当前已做** | 截至今日该模块整体累积交付（✅🟡🔵⬜➖）；叶子 `roadmap.yml` 有实质变化时汇总抬行 |
| **5.30 能力** | 5.30 锁节点上该模块是否达标 |
| **6.30 能力** | 6.30 锁节点上该模块是否达标 |

**不要** 使用叶子表头（功能 / 原型 / 后端 / …）。

**周计划固定句式**：

```markdown
周计划与本周结果见 [AI API 聚合产品 · 总览 · 周计划与验收看板](../#周计划与验收看板)；节点主链与走查见同页 **5.30 / 6.30 能力主链**。
```

### 4.3 子总览 · 禁止

`::: tip` 填写说明 · 重复产品总览四行 · 子页能力主链全文 · `<ProductWeekProgress />` · `## 周进度`

## 五、子模块叶子页（维护重心）

产品总览 / 子总览管地图与汇总；**子清单、原型/后端进度、节点、验收链** 在叶子维护。

**全站仅一种叶子页**：**标准叶子**（无「需求型叶子」）。未拆子能力的范围说明、长需求正文放在 **`docs/05-产品与PRD/`**，本手册 **附录链 PRD** 或在 `roadmap.yml` 加行 + `note` 指向 PRD；侧栏可保留历史 PRD 链，但不单独设页型。

### 5.1 标准叶子 · 五件套 + 冻结结构

**真源**：`ai-api-platform/user/models/list.md` + 同目录 `roadmap.yml`

手册叶子 **五件套**（与工程侧 [`五件套规范`](../../../docs/01-原型与交付规范/Trinity原型模块目录与交付规范.md) 对齐，不重复写 PRD 全文）：

| # | 块 | 写什么 |
|---|-----|--------|
| 1 | **`> **说明**`** | 价值、边界、数据口径（2～4 句） |
| 2 | **`> **工程**`** | **`apps/*/src/views/<模块>/` 原型五件套目录** + 路由；必要时点名 `XxxPage.vue` · `*.css` · `*Interactions.ts` · `mock.ts` · `README.md` |
| 3 | **`> **体验 / 在线**`** | 链产品总览（不写四行全文） |
| 4 | **`## 子能力清单`** | `<ProductRoadmap />` / `roadmap.yml`（**原型 · 后端 · 5.30 · 6.30** 列） |
| 5 | **`## 附录`** | 飞书验收链 · 关联 · **PRD 链**（`docs/05-产品与PRD/…`）· 修订 |

**冻结顺序**（= 上表顺序）：

1. `---` frontmatter：`title` 与侧栏一致  
2. `# {模块名} · {页面名}`  
3. 页头三块 `>`，**块与块之间必须空一行**  
4. `## 子能力清单`  
5. `## 附录`（`### 验收（5.30 / 6.30）` · `### 关联` · 可选 `### PRD / 规范` · `### 修订`）

**页头骨架**：

```markdown
> **说明**：…（价值、边界；子能力见 `roadmap.yml`）

> **工程**：`apps/trinity-ai/src/views/models/`（五件套：`ModelsPage.vue` · `models.css` · `modelsInteractions.ts` · `mock.ts` · `README.md`）· 路由 `/models`

> **体验 / 在线**：见 [AI API 聚合产品 · 总览](相对路径)（用户面 `/models`）
```

**工程行**：必须能一眼跳到 **可运行的原型目录**；子能力表里 **原型** 列对应该五件套的交付程度，**后端** 列对接 API/数据真源。

**验收固定句式**（替换 `{模块名}`、`{用户面|平台侧|运营后台}`）：

```markdown
### 验收（5.30 / 6.30）

走查、体验测试与 Bug 真源：[**5.30 产品测试体验 / Bug 表**](https://qcn81yhei1l2.feishu.cn/sheets/PjnVs7bmphodaKtOkkycpvxmnne)（在飞书按 **时间**、**产品/模块** 筛选；本页对应 **{模块名}** / {用户面|平台侧|运营后台}）。子能力进度与节点列以 **`roadmap.yml`** 为准，手册不抄验收 checklist。
```

**标准叶子 · 禁止**：`::: tip` · `## 说明` · 产品总览四行体验/在线全文 · `week-progress` · 附录 `- [ ]` checklist · 抄飞书 Bug 表

### 5.2 子能力清单（**一律 YAML + 可编辑**）

**标准叶子禁止**在正文手写 Markdown 子能力表；与 [`user/models/list`](./ai-api-platform/user/models/list.md) 相同，用 **`<ProductRoadmap />`** 读 YAML，**localhost 可编辑**（表格 / YAML 双栏），静态 build 只读。

**文件命名**（与叶子 `.md` 同目录，相对 `docs/`）：

| 场景 | YAML 路径 | 叶子嵌入 |
|------|-----------|----------|
| 单页独占（多数叶子） | `…/{slug}.roadmap.yml` | `<ProductRoadmap rel="ai-api-platform/…/{slug}.roadmap.yml" />` |
| 子目录多页共用一张表 | `…/roadmap.yml` | `<ProductRoadmap />`（默认读同目录 `roadmap.yml`） |
| 子目录拆第二张表 | `…/detail-roadmap.yml` 等 | `<ProductRoadmap rel="…/detail-roadmap.yml" />` |
| 跨页 `prefix` 筛行 | 共用一份 yml | `<ProductRoadmap rel="ai-cloud/roadmap.yml" prefix="营销首页" />` |

示例（单页）：

```markdown
## 子能力清单

<ProductRoadmap rel="ai-api-platform/operations/models-routes.roadmap.yml" />
```

- 列定义（功能 · 原型 · 后端 · 5.30 · 6.30 · 备注）→ [更新规范 §一](./产品手册更新规范.md)  
- `roadmap.yml` / `*.roadmap.yml` = 模块内累积分解；**不是**周会排期表

**PRD / 长需求**：放 `docs/05-产品与PRD/`；叶子用 **`### PRD / 规范`** 链出。例：[模型详情页](ai-api-platform/user/models/model-detail-requirements.md) 为标准叶子 + `detail-roadmap.yml`，细表在 `docs/05-产品与PRD/roadmap/`。

## 七、生成时禁止（汇总）

| 禁止 | 应做 |
|------|------|
| 复制 PRD 全文 | 叶子附录链 `docs/05-产品与PRD/…` |
| 复制飞书 Bug / 验收 checklist | 附录只链飞书表 + `roadmap` 节点列 |
| 叶子 `::: tip` / `## 说明` | 并入 `> **说明**` |
| 每模块一份 `week-progress.yml` | 仅 `ai-api-platform/week-progress.yml` |
| 子能力表与周计划混一页 | 分文件、分页面类型 |
| 小产品强行建子总览 | 模块表放产品总览 §3.2 |

## 八、新建 / 改版检查清单

1. **规划**：路径、侧栏 `config.ts`；多分层则在父级 **子总览模块表** 加一行（单层则在产品总览模块表加一行）
2. **叶子**：按 §5.1 对齐 `user/models/list.md`；**必须** `{slug}.roadmap.yml`（或子目录共用 `roadmap.yml`）+ `<ProductRoadmap rel="…" />`，禁止 Markdown 子能力表
3. **节点**：5.30 / 6.30 按当前里程碑与锁规则填写
4. **实施**：附录 PRD 链；`> **工程**` 写清五件套目录；roadmap **原型/后端** 与工程一致
5. **周会**：聚合线纳入 `week-progress.yml` 的 `focus`（标准模块名）
6. **符号与周会顺序**：[更新规范](./产品手册更新规范.md)
7. **改版存量**：去 tip / `## 说明` / 手册 checklist；子总览去重复总览；页头 `>` 块间空行
