# 产品手册文档规范

> AI 生成或改写 `apps/trinity-product/docs/**` 时必读。只规定 **写什么、放哪、长什么样**。  
> 符号、锁节点、周计划字段怎么改 → [产品手册更新规范](./产品手册更新规范.md)。  
> Agent 维护流程 → `.cursor/skills/trinity-product-handbook`（可由根总机 `trinity-project` 封发）。

**真源目录**：`apps/trinity-product/docs/` · 侧栏登记：`.vitepress/config.ts`

---

## 一、页面类型（四种）

| 类型 | 路径示例 | 职责 |
|------|----------|------|
| **站点总览** | `index.md` | 业务线入口；链两份规范；不写周计划 |
| **产品总览** | `ai-api-platform/index.md` | 单条业务线的地图、体验/在线、跨层协同；**写法样板**见 §2.2 |
| **子总览** | `ai-api-platform/user/index.md` 等 | 分层/模块地图：用户侧、平台侧、运营侧等 **模块总览**；不写周计划 |
| **子模块叶子页** | `…/models/list.md` 等 | 单模块说明 + 子能力分解；见 §三 |

命名：**AI API 聚合产品**（不说「聚合平台」）；该线产品总览标题固定为 **`AI API 聚合产品 · 总览`**。

## 二、总览写什么

### 2.1 站点总览 `docs/index.md`

手册站首页：只介绍 **这本手册是什么**，不重复各线里程碑。

- **前言** + **一条主线** tip（阅读）+ **管理闭环** 引用块 `>`（维护五步 + 周会顺序，不占二级标题）
- **能力矩阵**（**产品地图** / Roadmap / 周进度 / 子能力 / **PRD / 原型索引** / 导出）
- **业务线**（名称 · 说明 · 入口）

**管理闭环与页面分工**（维护时按序，勿混写）：

| 层 | 真源 |
|----|------|
| 规划（模块树） | 侧栏、`config.ts`、产品总览 / 子总览模块表 |
| 规划（子清单） | `…/roadmap.yml`、叶子 `<ProductRoadmap />` |
| 节点 | 子总览 / 叶子 5.30、6.30 列；锁节点见更新规范 §三 |
| 实施 | PRD / 原型 / 工程：叶子附录 + roadmap 原型/后端列 |
| 进度（累积） | `roadmap.yml` → 必要时抬子总览「当前已做」 |
| 进度（本周） | 仅产品总览 `week-progress.yml`；`focus` 用标准模块名 |
| 验收 | 节点列 + 叶子附录；飞书走查不抄进手册 |

**禁止**：业务线「本阶段」简表、子模块四段全文、符号附录、首页重复链规范表（规范在侧栏 **手册规范**）；**周会不得逐条改子能力表**（只改周计划 + 有交付时改 roadmap）。

不在此页挂 `week-progress.yml`（真源在 **产品总览**）。

### 2.2 产品总览（样板：`ai-api-platform/index.md`）

标题：`# AI API 聚合产品 · 总览`（其他业务线同理：`# {业务名} · 总览`）。

页头 **4 行** `>` 引用块（顺序固定）：

```markdown
> **一句话介绍**：…
> **工程**：`apps/…` · …
> **体验地址**：本地 URL
> **在线地址**：部署 URL（无则写待补）
```

正文区块（与样板页 **同序、同名**；无内容的区块可省略，勿自创同级标题）：

| 区块 | 必填 | 内容 |
|------|:----:|------|
| 三层分工 | ✅（多分层业务线） | 调用关系图 + 分层 · 包含什么 · 手册入口 · 整体进度符号 |
| 周计划与验收看板 | ✅ 仅 **AI API 聚合产品** | `<ProductWeekProgress rel="ai-api-platform/week-progress.yml" />` |
| 半月众测记录 | 建议（聚合产品线） | 周期 · 范围 · 结论 · Top 问题 · 责任人 · 截止时间 · 复盘 |
| 5.30 / 6.30 能力主链 | 按需 | 跨模块主链草案 + 飞书等外链；模块级达标写在子总览/叶子表 |
| 相关链接 | 建议 | PRD、规范等路径，不贴全文 |

**禁止**：在子总览（`user/`、`platform/`、`operations/`）挂 `week-progress.yml` 或复制产品总览四行体验/在线。

### 2.3 周计划看板（仅产品总览）

```markdown
## 周计划与验收看板

<ProductWeekProgress rel="ai-api-platform/week-progress.yml" />
```

- 真源：`docs/ai-api-platform/week-progress.yml`（全站唯一）
- `focus` 标准名见更新规范

### 2.4 子总览（模块总览 / 平台侧总览 / 运营侧总览等）

标题示例：`# 用户侧模块 · 总览`、`# 平台侧模块 · 总览`。

```markdown
> **工程**：…
> **地址**：见 [AI API 聚合产品 · 总览](../)
```

| 模块 | 当前已做 | 5.30 能力 | 6.30 能力 |
|------|:--------:|:---------:|:---------:|

- 周计划一句：`见 [AI API 聚合产品 · 总览 · 周计划与验收看板](../#周计划与验收看板)`
- **禁止**重复产品总览的体验/在线四行

## 三、子模块叶子页

产品总览管地图与阶段；叶子页 **不重复** 体验/在线（只链产品总览）。

### 3.1 标准四段（顺序固定）

| 段落 | 写法 |
|------|------|
| **页头说明** | `> **说明**` + 2～4 句（价值、边界）；**不要** `## 说明` |
| **工程定位** | `> **工程**`：仓库路径 · 路由；体验/在线见产品总览 |
| **子能力清单** | Markdown 表 **或** `<ProductRoadmap />`（§3.2） |
| **附录** | 验收勾选、关联模块、PRD 链、`## 修订` 表 |

### 3.2 子能力清单

**方式 A · YAML（推荐，子能力 ≥3 条）**

- 一个子目录一个文件：`docs/…/子目录/roadmap.yml`（例 `user/models/roadmap.yml`）
- 叶子页正文：

```markdown
## 子能力清单

<ProductRoadmap />
```

- 多页共用一表：

```markdown
<ProductRoadmap rel="ai-cloud/roadmap.yml" prefix="营销首页" />
```

- localhost 可编辑表格 + YAML；build 后只读

**方式 B · Markdown 表（子能力少或过渡）**

| 子能力 | 当前已做 | 5.30 验收 | 6.30 商用 | 说明 |
|--------|:--------:|:---------:|:---------:|------|

列含义与符号 → 更新规范。

**分工**：`roadmap.yml` = 模块内分解；**不是**周会排期（周会用 `week-progress.yml`，仅产品总览）。

### 3.3 需求型子页（无子能力表）

例：模型详情页需求。结构：

- `> **说明**` · `> **工程**`
- `## 范围与边界` · `## 路由` · `## 与非目标`
- `## 附录`（验收、关联、修订）

**不要** `<ProductRoadmap />`。

### 3.4 叶子页 frontmatter

```yaml
---
title: 列表
---
```

`title` 与侧栏文案一致。

## 四、生成时禁止

| 禁止 | 应做 |
|------|------|
| 复制 PRD 全文进手册 | 附录链 `docs/05-产品与PRD/…` |
| 复制飞书 Bug 表进手册 | 周计划或半月众测表只链 URL |
| 每模块一份 `week-progress.yml` | 仅 `ai-api-platform/week-progress.yml` |
| 叶子页再写体验/在线四行 | 只链产品总览 |
| 子能力表与周计划混在一张表 | 分文件、分页面 |

## 五、新建模块检查清单（走管理闭环）

1. **规划**：路径正确；`.vitepress/config.ts` 侧栏；父级子总览模块表加一行  
2. **规划 · 子清单**：叶子页四段；`roadmap.yml` 或 Markdown 子能力表  
3. **节点**：5.30 / 6.30 列按当前里程碑填好  
4. **实施**：附录链 PRD；`roadmap` 原型/后端列与工程路径一致  
5. **进度 · 周会**：纳入产品总览 `week-progress.yml` 的 `focus`（标准模块名）  
6. 页面类型符合 §一；符号与周会顺序见 [更新规范](./产品手册更新规范.md)
