## 零、视角全景图

一个完整的 GEO SaaS 产品，不仅是一个「用户控制台」。从商业闭环角度，需要覆盖 **五大视角**：

```
                    ┌──────────┐
                    │   官网    │  获客与转化
                    └────┬─────┘
                         │ 注册/登录
          ┌──────────────┼──────────────┐
          ▼              ▼              ▼
   ┌──────────┐  ┌──────────────┐  ┌──────────┐
   │ 用户控制台 │  │  开发者门户   │  │ 微信小程序 │
   │  (核心)   │  │  (API/集成)  │  │ (轻量查看)│
   └────┬─────┘  └──────────────┘  └──────────┘
        │
   ┌────▼─────┐
   │ 运营后台  │  内部管理与运营
   └──────────┘
```

### 五大视角说明

| 视角 | 用户 | 核心目标 | V1 是否必须 |
|------|------|---------|:----------:|
| **官网** | 潜在客户 | 品牌建设、获客转化 | ✅ 必须 |
| **用户控制台** | 付费/试用用户 | 使用产品完成 GEO 工作 | ✅ 必须 |
| **运营后台** | 内部运营/客服 | 用户管理、内容运营、问题排查 | ✅ 必须（可简化） |
| **开发者门户** | 技术集成方 | API 文档、SDK、Webhook | ❌ V2 再做 |
| **微信小程序** | 管理层/移动场景 | 核心指标轻量查看、告警推送 | ❌ V2 再做 |

### V1 能力范围

> V1 只做 3 个核心能力；**V1 仅做「提及 + 位置」**，引用 vs 提及（CCR）、信源页面分析属 **V1.5**。

| 能力 | 对应模块 | V1 策略 |
|------|---------|------|
| **多平台 AI 监测** | 监测模块（P0 共 4 页） | 对标。差异化在 **海外 5 + 国内 5** 同一面板 |
| **SOA 答案份额** | Dashboard | 三层可视化：关键词级 + 平台级 + 时间趋势（见 §5.4） |
| **竞品对比** | 竞品模块（P0 共 2 页） | 同问题集下品牌 vs 竞品 **提及 / SOA** 对比（不含引用页分析） |

V1 差异化：**双市场平台覆盖** + **平价全功能**（$79/月 + 14 天试用，相对 Tier 1 高价或阉割）。

### 页面清单

| 序号 | 页面 | 核心内容 | 优先级 |
|:---:|------|---------|:---:|
| 1 | **首页** | Hero（**双市场一句话** + CTA）、<br>三能力卡片（监测 / SOA / 竞品）、<br>「海外 5 + 国内 5」平台 Logo 墙、<br>出海客户 Logo 墙、<br>数据背书（已覆盖 **10** 个 AI 平台）、<br>底部 CTA。<br>**逐块规格** → [官网首页原型](./prototypes/v1-homepage) | P0 |
| 2 | **产品功能页** | 三大能力详解 + 双市场对比截图；<br>FAQ（为何同时监测豆包与 ChatGPT、SOA 是什么等） | P0 |
| 3 | **定价页** | **试用 14 天（专业版全功能）→ $79/月 专业版 → 企业定制** 两档对比 + 功能矩阵（突出 10 平台 vs 竞品）；<br>升级/退款 FAQ；<br>对标说明链到 [Tier 1 定价带](./competitor-research/tier1-deep-dive#33-定价带分布) | P0 |

## 二、用户控制台（约 18 页）

> **设计目标**：用户登录后 **10 分钟内** 完成「添加品牌 → 配置关键词 → 看到双市场数据 → 对比竞品」闭环。

### 全局框架

```
顶部导航栏（V1）：
  [Logo→Dashboard]  监测  竞品  [账户]

V1 仅监测 + 竞品两个模块入口。审计 / 优化 / 报告在 V1.5 加入导航。
```

## 五、交互设计要点

### 5.1 用户首次使用引导流

```
注册成功（14 天专业版试用生效）
  ↓
[步骤 1] 创建品牌 → 名称、官网、行业（出海相关选项）
  ↓
[步骤 2] 添加别名 → 识别 AI 回答中的品牌表述
  ↓
[步骤 3] 添加关键词 → 行业推荐词 + 手动添加（受套餐关键词上限约束）
  ↓
[步骤 4] 添加竞品 → 2–3 个核心竞品 + 别名
  ↓
跳转 Dashboard →「数据采集中，预计 5 分钟…」（10 平台并行）
  ↓
首次数据就绪 → 刷新 + 提示「品牌已开始被追踪（海外+国内）」
```

### 5.2 页面跳转矩阵

```
Dashboard ──→ 关键词详情 / AI 回答详情 / 竞品概览

监测概览 ──→ 关键词管理 / 采集日志（P1）

关键词管理 ──→ 关键词详情 ──→ AI 回答详情

竞品概览 ──→ 竞品管理 / 竞品详情 ──→ AI 回答详情
```

### 5.3 空状态设计

| 页面 | 场景 | 设计 |
|------|------|------|
| Dashboard | 采集中 | 进度卡片 + 10 平台状态列表 |
| 关键词管理 | 无关键词 | 插图 + CTA + 推荐词列表 |
| 竞品管理 | 无竞品 | 插图 +「看看是否在 AI 中被超越」CTA |
| AI 回答详情 | 未提及品牌 | 说明别名未识别或确未出现 |
| 报告列表 | V1.5 | 「即将推出周报」占位（V1 可隐藏入口） |

### 5.4 SOA 三层可视化 ↔ 页面对照

| SOA 层级 | 主要页面 | 验收要点 |
|----------|----------|----------|
| **时间趋势** | Dashboard SOA 图 | 日/周/月切换 |
| **平台级** | Dashboard 分布、关键词详情各平台摘要 | 海外/国内分组 |
| **关键词级** | 关键词详情、关键词管理列表 | 单关键词 SOA 与变化 |

## 六、认证、套餐与补充视角

### 6.1 认证与账户体系

```
注册：邮箱 / 手机号 / 微信扫码
登录：密码 / 验证码 / 微信 / SSO（企业版预留）
权限：管理员 / 编辑者 / 只读（V1 可简化为单账户 Owner）
多品牌：专业版支持多品牌，上限见套餐（V1 不做代理商多租户）
试用：注册即 14 天专业版全功能（10 平台）
套餐：试用 → $79/月专业版 → 企业定制（无永久免费档）
```

### 6.2 套餐用量边界（草案 · 待商用拍板）

| 维度 | 14 天试用 | 专业版 $79/月 |
|------|-----------|---------------|
| 品牌数 | 1 | 3（示例） |
| 监测关键词 | 20（示例） | 100（示例） |
| AI 平台 | 10（全开） | 10（全开） |
| 竞品数 | 3 | 10（示例） |
