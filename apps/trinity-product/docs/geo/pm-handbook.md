## 一、GEO 领域判断力

这些是 **「外行 PM」和「懂 GEO 的 PM」的分水岭**：

| 问题 | 外行 PM 的回答 | 懂 GEO 的 PM 的回答 |
|------|-------------|-------------------|
| 用户说「我想提升排名」 | 「好的我们做关键词优化」 | 「排名不重要，你真正需要的是在 AI 回答里的答案份额 SOA」 |
| 用户说「帮我多写点内容」 | 「好的我们加内容生成功能」 | 「内容数量不解决问题，关键是证据密度和可引用性」 |
| 竞品出了一个新功能 | 「我们也做吧」 | 「这个功能解决用户的什么问题？和我们路线图的差异点是什么？」 |
| 采集成本高怎么办 | 「砍掉采集功能」 | 「监测是 GEO 的基础，降本可以用采样策略而非砍功能」 |
| 用户说「这个报告不够好看」 | 「好的我们加更多图表」 | 「报告好不好看是次要的，能帮用户回答老板『AI 里能看到我们吗』才是核心」 |

**降本原则**（与 [GEO 业界计费](./geo-billing-industry) 一致）：优先调 **平台 / 问题 / 频率 / 轮次**，不宜砍掉「监测」能力本身。

## 二、迭代节奏（示例）

```
Sprint 5–6：   优化待办 + 效果验证 + 用户反馈
─────────────── P0 闭环可演示
Sprint 7–8：   信源盘 + 页面审计
Sprint 9–10：  报告中心 + LLM 草稿

─────────────── P1 差异化加深
```

## 三、种子用户

**理想画像（P0 闭环验证）**：

- 有 SEO 背景，正在经历「AI 流量超过搜索流量」的焦虑
- 已在关注 GEO，但没用过工具
- 愿意参与产品反馈，对早期不完美有容忍度
- 所在行业 GEO 竞争激烈（电商、SaaS、消费品牌）

**获取渠道**：

- SEO 社群 / 知识星球
- 即刻 / 小红书上的 SEO 和 AI 话题博主
- 清华 GEO 报告读者群
- AI 产品经理社群

**用户画像长文**（品牌营销 / 内容 SEO / 代理商）见 [产品设计分析](./product-design-analysis) 或历史调研附件；新手写 L0 时可对照 [用户画像模板](../产品经理工作手册#三用户验证机制) 结构自行展开。

## 四、GEO 合规补充

在 [通用合规框架](../产品经理工作手册#五合规前置通用框架) 之上，GEO PM 额外关注：

| 合规项 | 设计要求 |
|--------|---------|
| AI 平台服务条款 | 采集行为不违反各平台 ToS（速率控制、用途限制） |
| 内容合规 | 优化建议不能包含虚假、夸大、医疗建议等 |
| 广告法合规 | 优化内容不出现「最好」「第一」「唯一」等极限词 |
