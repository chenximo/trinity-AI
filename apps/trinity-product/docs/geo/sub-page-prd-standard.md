# 子页 PRD · 通用规范

> **说明**：控制台单页 PRD 写法规范。**手册参考样例**（仅 1 页）：[引用与信源](./samples/citations-prd-sample)。其余页面 PRD 真源在工程 **`apps/trinity-geo/marketing/console/{page}.md`**（与 HTML/CSS/Interactions/mock 同目录五件套）。

---

## 一、80% 统一原则（比「全文一致」更重要）

**目标**：存量页在 **原结构基础上** 对齐 **~80%** 即可开写、可验收；**不要求**每页节号、图类型、篇幅一模一样。

### 必须统一（100% · 缺了就不算子页 PRD）

| # | 内容 | 节号 |
|---|------|:----:|
| 1 | 页头 blockquote：原型 · 预览 · **固定样本** | 页头 |
| 2 | **用户问题** + **不是什么**（边界） | §1 / §3 |
| 3 | **用户故事** 1 条 | §2 |
| 4 | **功能说明**（模块表或等价列表） | §5 |
| 5 | **交互读路径**（text 一行或 L2 图，二选一） | §6 |
| 6 | **验收**（GWT ≥1 或 checklist + 样本 ID） | §8 |
| 7 | **修订** 在文末 | 修订 |

### 逻辑型页额外必写（GEO / API 等）

| # | 内容 | 节号 |
|---|------|:----:|
| 8 | **规则 / KPI / 字段** 表（本页核心逻辑） | §4 |

### 灵活处理（有则写，无则省略 · 约占 20% 差异）

| 项 | 说明 |
|----|------|
| **L1 mermaid** | 逻辑简单时 **一句 L1 文字 + 链 L0** 即可（见 [diagnosis](../../../trinity-geo/marketing/console/diagnosis.md)） |
| **L2 mermaid** | §6 用 **text 路径** 也可，不必每页画图 |
| **§7 异常** | 简单页可 2 行或省略 |
| **API / 埋点** | 原型期可整节省略 |
| **Mock 对照** | 有 JSON 样本时建议写；纯 UI 页可只在 §8 checklist |
| **节号顺序** | 允许旧稿保留「规则在 §2」等习惯，**内容块齐** 即可 |

**禁止**（与 80% 无关、一律不要）：页首元信息大表、文档分工表、复制 §0.6.2a 全文。

**真源位置**：

| 位置 | 放什么 |
|------|--------|
| **手册** `geo/samples/citations-prd-sample` | **唯一参考样例**（写法对齐用） |
| **工程** `apps/trinity-geo/marketing/console/*.md` | 各页 PRD 真源（与原型五件套同目录） |

新建页：对照手册样例 + [空模板](./samples/_prd-template.md)（不挂侧栏）或 **复制** `console/` 下邻页 `.md` 改写。

## 二、页型（写 §4 时用 · 不必挂侧栏）

| 页型 | §4 重点 | 工程 PRD 示例 |
|------|---------|---------------|
| 下钻页 | 字段字典 | `answer-detail.md` |
| 规则列表 | D* / S* 规则表 | `diagnosis.md` |
| 汇总读口 | KPI + rollup | `citations.md` · **手册样例** |

## 三、标配清单

| 块 | 子页 PRD | 意义 | 节号 |
|----|:--------:|------|:----:|
| **背景 / 问题 / 用户原声** | ✅ 必填 | 为什么要做 | §1 |
| **用户故事** | ✅ 必填 | 谁、要什么、价值 | §2 |
| **功能范围（做 / 不做）** | ✅ 必填 | 边界 | §3 |
| **数据 / 规则 / 状态机** | 🟡 逻辑型必填 | 规则表 + **L1 位置图**（链 L0） | §4 |
| **功能说明** | ✅ 必填 | 这页有哪些模块 | §5 |
| **交互** | ✅ 必填 | **L2 UI 读路径** + 筛选 / 操作 | §6 |
| **异常 / 空态** | 🟡 建议写 | 避免只做 happy path | §7 |
| **验收标准（GWT 或 checklist）** | ✅ 必填 | 什么叫做完 | §8 |
| **API** | 🟡 有后端时 | 契约 | 文末 |
| **埋点** | 🟡 商用前 | 上线后怎么量 | 文末 |
| **Mock 真源对照** | 🟡 原型期 | HTML + JSON 验收 | 文末 |
| **排期 / 风险** | ➖ 单页省略 | 立项文档层面 | — |

**排版**：§1–§6、§8 在 **正文前段**（打开 2～3 屏能读完核心）；API、埋点、Mock、关联、**修订** 放 **文末**。

**不要**：页首元信息大表、文档分工表、六环/顶栏表 — 见 [原型页面清单](./v1-prototype-pages)。

## 四、冻结节序

```text
页头   blockquote（原型 · 预览 · 样本，≤4 行）

§1  背景与问题（用户原声 1～2 条 + 用户问题）
§2  用户故事（1 条）
§3  功能范围（做 · 不做）
§4  数据与规则（表格 + **L1 小图** · 链 L0，不复制 §0.6.2a 全文）
§5  功能说明（模块表 · 优先级）
§6  交互（**L2 UI 读路径图** · 筛选 · 操作）
§7  异常与空态
§8  验收标准（GWT 2 条 + 区块 checklist + 固定样本）

--- 文末 ---
关联 · API（可选）· 埋点（可选）· Mock 真源对照（原型期）· 修订（固定最后）
```

## 五、各节写法

### 页头

```markdown
# 用户控制台 · {页面名} · 产品需求（PRD）

> **配套原型**：[{page}.html](…) · **预览**：`cd apps/trinity-geo && bun run dev` → …  
> **样本**：Q00×豆包×R1 · [data.json](…)  
> **全景**：[原型页面清单 §…](./v1-prototype-pages)
```

### §1 背景与问题

```markdown
## 1. 背景与问题

> 「用户原声 …」

**用户问题**：…  
**固定样本**：Q00 · …（P0 页必填）
```

### §2 用户故事

```markdown
## 2. 用户故事

作为 …，我想要 …，以便 …。
```

### §3 功能范围

```markdown
## 3. 功能范围

**做**：…  
**不做**：…（→ [邻页](…)）
```

### §4 数据与规则

逻辑型产品（GEO 监测、API 网关、诊断规则）**必填**。链 L0 文档，不复制全文：

```markdown
## 4. 数据与规则

（KPI 定义 / D* 规则 / rollup 表 / 字段字典 / 状态机 — 按页型选一种）

数据流详见 [§0.6.2a](./product-design-analysis#citation-data-pipeline)（一句链即可）。
```

| 页型 | §4 写什么 |
|------|-----------|
| 汇总读口 | KPI + rollup 定稿 |
| 证据下钻 | 字段字典 + 标注规则 |
| 规则列表 | D* / S* 条件表 |
| 审计 / 评分 | 因子维度（见 [audit](../../../trinity-geo/marketing/console/audit.md) §2） |

**§4 末尾 · L1 位置图（逻辑型页建议有）**

本页在 **跨页数据流** 里占哪一段；≤8 节点的 mermaid；**链** [§0.6.2a](./product-design-analysis#citation-data-pipeline)，**禁止复制 L0 全文**。

```markdown
### 4.x 本页在数据流中的位置（L1）

\`\`\`mermaid
flowchart LR
  Up[上游 …] --> This[本页 …]
  This --> Down[下游 …]
\`\`\`

完整数据流：[§0.6.2a](…)（L0 真源）。
```

### §6 交互

**§6.1 L2 UI 读路径（用户向页面必填）**

用户从哪进、在本页做什么、跳去哪；≤7 步，**text 或 mermaid 均可**。

**§6.2 筛选 / 操作** — 表格  
**§6.3 与 §4 的分工**：§4 管 **数据怎么算**；§6 管 **用户怎么读、怎么点**。

| 场景 | 用什么 |
|------|--------|
| 跨 3+ 系统/表/页 | L0 mermaid → `product-design-analysis` / `tech-architecture` |
| 本页在流中的位置 | **L1** mermaid → §4 末尾 |
| 本页内 UI 路径（≤7 步） | **L2** text 或 mermaid → §6.1 |
| rollup / 分平台等易错逻辑 | **表格 + 口诀** → §4（不必画图） |
| D* / S* 规则 | **表格** → §4（不必画图） |

### §5 · §7 · §8

见 [空模板](./samples/_prd-template.md)。

### 文末

```markdown
## 关联
…

## API（有后端时）
…

## 埋点（商用前）
…

## Mock 真源对照（原型期）
| 模块 | 真源 | 状态 |

## 修订
| 日期 | 说明 |
```

## 六、检查清单（80% 达标）

**必过（7 项）**

- [ ] 页头：原型 + 预览 + 固定样本  
- [ ] §1 用户问题 · §3 或 §1 内「不是什么」  
- [ ] §2 用户故事 1 条  
- [ ] §4 规则/KPI（逻辑型）或写明「无独立规则」  
- [ ] §5 功能模块  
- [ ] §6 交互路径（text 或图）  
- [ ] §8 验收 + **修订在最后**

**加分（有则更好，不强制）**

- [ ] L1 / L2 mermaid  
- [ ] §7 异常 · Mock 对照 · API / 埋点
