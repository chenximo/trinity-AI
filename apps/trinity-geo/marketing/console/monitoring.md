## 1. 背景与问题

### 1.1 用户原声

> 「今天数据更新了吗？哪个平台挂了？」  
> 「SOA 是 0，是没人提我还是没采到？」

### 1.2 要解决的问题

| 问题 | 本页职责 |
|------|----------|
| 采集是否成功 | **成功率**、**最近失败** 列表 |
| 数据是否新鲜 | **10 平台** 绿 / 黄 / 红 新鲜度 |
| 与 SOA 混淆 | 明确 **运维视角**，测量下钻链到其他页 |

**不是什么**：SOA 总览、诊断结论、问题 CRUD（后者在问题集管理）。

---

## 2. 用户故事

```
作为 负责 GEO 的运营同学，
我想要 一眼看到 10 个 AI 平台今天的采集是否完成，
以便 在客户问「数据准吗」时能先排除采集故障。
```

---

## 3. 与主链的关系

```text
① 问题集（enabled 问题）
    ▼
② 采集调度（本页可见 · 粒度 question × platform × round）
    │  collection_jobs → raw_answers
    ▼
③ 标注 + SOA（总览 / 引用 / 情感 / 下钻页）
```

### 3.1 本页与平台维度

| 区域 | 平台维度 | 说明 |
|------|----------|------|
| 今日 KPI | 汇总 | 全站：启用题 × 平台数 |
| 10 平台状态卡 | **主维** | 每卡一平台；可筛海外/国内 |
| 最近失败 | **每行** | 平台 + 问题 |
| 最新入库 | **每行** | 平台 + 问题 |

与测量读口不同：本页回答「采到了吗」，不回答「SOA 多少」。

## 4. 功能需求

### 4.1 顶部 KPI

| 指标 | 说明 | 优先级 |
|------|------|:------:|
| 启用问题 | 当前 `enabled` 问题数；链问题集 | P0 |
| 今日采集 | 题数 × 平台数（预期完成量） | P0 |
| 成功率 | 今日成功 / 预期；附失败条数 | P0 |
| 最近全量 | 最近一轮全量完成时间；下次计划 | P1 |

### 4.2 10 平台状态卡

| 字段 | 说明 |
|------|------|
| 新鲜度点 | 绿 = 今日已采；黄 = 逾 24h；红 = 失败 |
| 状态标签 | 正常 / 延迟 / 失败 |
| 最近采集 | 该平台最后一次成功时间 |
| 今日进度 | 已完成题数 / 启用题数 |
| 失败数 | 今日失败任务数 |
| 信源可提取 | `citation_extractable` 渠道配置（链引用页） |

**筛选**：全部 / 海外 / 国内（与总览平台条一致）。

### 4.3 最近失败

| 列 | 说明 |
|----|------|
| 时间 | 任务失败时间 |
| 平台 | AI 平台 |
| 问题 | 链 **关键词详情** |
| 原因 | HTTP 码、超时、限流等 |

### 4.4 最新入库回答

| 能力 | 说明 |
|------|------|
| 列表 | 最近 N 条 `raw_answers` 摘要 |
| 跳转 | 链 **回答详情**；问题链 **关键词详情** |
| 标注徽章 | 进答案 / 未进答案（只读） |

### 4.5 操作

| 操作 | 行为 |
|------|------|
| 立即刷新状态 | 拉取最新采集任务汇总（商用 API） |
| 管理问题集 | 跳转问题集管理 |

### 4.6 页内 Tab · 采集日志

MVP **不单独占侧栏**，与概览同页切换：

| Tab | 内容 | 深链 |
|-----|------|------|
| **概览** | §4.1–4.4 全部区块 | `monitoring.html` |
| **采集日志** | 任务表、筛选、重试/导出 | `monitoring.html?tab=logs` |

日志字段与交互详见 [monitoring-logs.md](./monitoring-logs.md)。概览「最近失败」链至日志 Tab。
