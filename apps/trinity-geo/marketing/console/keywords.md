## 1. 背景与问题

### 1.1 用户原声

> 「你们 SOA 是按哪些问题算的？」  
> 「我只加了品牌名问法，为什么分数看起来很高？」

### 1.2 要解决的问题

| 问题 | 本页职责 |
|------|----------|
| SOA 分母不透明 | 列出所有 **监测问法**，标明类型与启用状态 |
| 问法不真实 | 引导 **用户口语**（品类/对比/场景），非 SEO 关键词堆砌 |
| 采集范围失控 | **暂停 / 删除** 不再进入每日采集与 SOA 分母 |
| 套餐超限 | 展示 **用量上限**（试用 20 / 专业 100） |

**核心公式（分母）**：

```text
SOA 分母采样数 ≈ Σ（启用中的问题 × 10 平台 × 当日采集轮次）
```

没有启用中的问题集 → 无 SOA。

---

## 2. 用户故事

```
作为 品牌营销负责人，
我想要 维护用户在 AI 里会问的真实问题列表，
以便 SOA 反映的是业务关心的场景，而不是任意关键词。
```

```
作为 已配置品牌的客户，
我想要 暂停某条问法或查看每条问题的 SOA，
以便 聚焦品类失声（如 Q00）而不被品牌词 SOA 掩盖。
```

---

## 3. 与测量主链的关系

```text
① 问题集管理（本页）
    │  monitoring_questions（enabled）
    ▼
② 采集调度：按启用问题 × 10 平台生成任务
    │  → raw_answers
    ▼
③ 规则引擎 + 别名库 → annotations → SOA（按 question_id 可下钻）
```

| 输入 | 来源 |
|------|------|
| **分母（哪些问题）** | 本页 `enabled=true` 的记录 |
| **分子识别（是否进答案）** | [品牌设置](./brand-settings.md) 别名库 |

## 4. 功能需求

### 4.1 问题列表

| 列 / 能力 | 说明 | 优先级 |
|-----------|------|:------:|
| ID | 稳定主键，如 `Q00`；报告与采集日志引用 | P0 |
| 类型 | **品类词 / 品牌词 / 对比词 / 场景词** | P0 |
| 问法 | 用户口语全文；链到 **关键词详情** | P0 |
| SOA 7d | 该问题近 7 日聚合 SOA（只读）；暂停显示 `—` | P0 |
| 状态 | **监测中** / **已暂停** | P0 |
| 操作 | 暂停、恢复、删除 | P0 |

**类型定义**（与 [MVP 实践](../../../../trinity-product/docs/geo/mvp-practice.md) 一致）：

| 类型 | 示例 | 产品意图 |
|------|------|----------|
| 品类词 | 「推荐两款 API 聚合平台」 | 发现 D1 品类失声 |
| 品牌词 | 「Trinity AI 好用吗？」 | 品牌认知与口碑 |
| 对比词 | 「Trinity 和 OpenRouter 哪个好？」 | 竞品同题 SOA |
| 场景词 | 「怎么降低多模型接入成本？」 | 场景覆盖 |

**组合建议**：品类 + 对比 + 场景 ≥ 50% 启用条数（页面提示，非硬拦截）。

### 4.2 筛选与搜索

| 能力 | 说明 |
|------|------|
| 类型 Tab | 全部 / 品类 / 品牌 / 对比 / 场景 |
| 类型摘要 chip | 点击等同筛选 |
| 搜索 | 匹配 ID 或问法 substring |

### 4.3 录入方式（手动为主 · AI 为辅）

| 方式 | UI | 优先级 | 说明 |
|------|-----|:------:|------|
| **手动填写** | 列表上方 **常显表单**（类型 + 问法 + 添加） | P0 | 主路径；用户确认后启用监测 |
| **模板填入** | 侧栏 chip → 填入手动表单，可改再添加 | P0 | 冷启动降门槛 |
| **AI 生成建议** | 顶栏按钮展开候选列表，勾选后加入 | P1 | 不自动上线；须人工勾选 |

原则：**采集可自动化，问题集不能黑盒自动化**（见 §4.4 字段校验）。

### 4.4 添加问题（字段与校验）

| 字段 | 必填 | 校验 |
|------|:----:|------|
| 类型 | ✅ | 四选一 |
| 问法 | ✅ | 非空；长度上限（如 500 字）；同品牌内近似去重（可选 P1） |

添加后默认 **监测中**；进入 **下一采集周期**（商用可配置为即时插队一轮）。

### 4.5 暂停与删除

| 操作 | 采集 | SOA 分母 | 历史数据 |
|------|:----:|:--------:|----------|
| **暂停** | 停止新采集 | 不计入 | 保留，详情可查 |
| **恢复** | 下一周期采集 | 计入 | — |
| **删除** | 停止 | 不计入 | 软删保留或硬删（商用需定策；默认软删） |

### 4.6 套餐用量

| 套餐 | 启用问题上限（示例） |
|------|---------------------|
| 试用 | 20 |
| 专业版 | 100 |
| 企业版 | 定制 |

展示：**启用数 / 上限** 进度条；暂停不计入启用数。

### 4.7 侧栏

| 块 | 内容 |
|----|------|
| 采集状态 | 启用问题数、今日采集量、成功率、下次全量时间 |
| 推荐模板 | 品类 / 对比 / 场景示例 |
| 关联配置 | 品牌设置、竞品管理 |

### 4.8 监测子导航

| 项 | 路由 | 本期 |
|----|------|:----:|
| 监测概览 | `/console/monitoring` | 占位 |
| **问题集管理** | `/console/monitoring/keywords` | ✅ |
| 采集日志 | `/console/monitoring/logs` | [monitoring.html?tab=logs](./monitoring.html?tab=logs)（Tab） |

## 5. 数据持久化

### 5.1 `monitoring_questions`

| 列 | 说明 |
|----|------|
| `id` | 主键 |
| `brand_id` | 所属品牌 |
| `question_key` | 对外 ID，如 `Q00` |
| `question_type` | `category` \| `brand` \| `compare` \| `scenario`（存枚举，UI 显示中文） |
| `text` | 问法全文 |
| `status` | `active` \| `paused` \| `deleted` |
| `sort_order` | 列表排序 |
| `created_at`, `updated_at` | 审计 |

### 5.2 相关表（本页不直接写）

| 表 | 关系 |
|----|------|
| `collection_jobs` | 由启用问题生成采集任务 |
| `raw_answers` | `question_id` 外键 |
| `metric_snapshots` | 按 `question_id` 聚合 SOA；列表读近 7 日 roll-up |

### 5.3 MVP / 原型

| 环境 | 存储 |
|------|------|
| HTML 原型 | 页面 Mock + `localStorage` 可选 |
| 样本数据 | `mvp/config/questions.json` |
| 商用 | `monitoring_questions` + API |

## 6. 接口（规划）

```
GET  /api/console/brands/:id/questions?status=active&types=...
POST /api/console/brands/:id/questions
PATCH /api/console/brands/:id/questions/:qid   # 暂停/恢复/改文案
DELETE /api/console/brands/:id/questions/:qid
```

列表响应含 `soa_7d`（只读，来自 metrics 服务）。

## 7. 验收标准

```
Given 品牌已有 10 条启用问题
When  用户暂停 Q00
Then  Q00 status=paused
And   启用计数变为 9
And   总览 SOA 分母不再包含 Q00 的后续采样
And   Q00 行 SOA 列显示 —
```

```
Given 用户添加品类问法且未超套餐上限
When  提交添加
Then  新问题 status=active
And   出现在列表且进入下一采集周期
```

```
Given 启用问题中品类+对比+场景占比 < 50%
When  用户查看页面
Then  展示组合建议提示（不阻断保存）
```
