# 错误码

Trinity 网关对错误采用 **OpenAI 风格 JSON 体**（`error.message`、`error.type` 等，具体字段以网关实现为准）。处理策略分两类：**上游供应商** 与 **网关自身**。

## 上游错误（建议透传）

| HTTP | 典型场景 | 客户端建议 |
|------|----------|------------|
| 429 | 上游限流 | 指数退避重试 |
| 502 / 503 / 504 | 上游不可用或超时 | 有限次重试 |
| 401 / 403 | 密钥或权限（若上游返回） | 检查密钥与模型权限 |

::: tip
429 / 5xx **应透传** 状态码与可解析错误体，便于 SDK 与现有重试逻辑复用。
:::

## 网关错误

网关自身错误（路由失败、参数校验、配额等）使用 **统一错误体**；`type` 与 `code` 枚举由产品补充并在本文档迭代。

常见情况：

- **400**：请求体非法（如 `model` 格式错误）
- **401**：缺少或无效的 `Authorization`
- **402 / 403**：配额或权限（以产品计费策略为准）

::: warning
请勿将上游原始堆栈或内部 host 暴露给终端用户；日志侧记录 `request_id` 供排查。
:::

## 排查清单

1. 确认 `TRINITY_BASE_URL` 含 `/v1` 前缀且与 [快速入门](../quickstart.md) 一致。
2. 确认 `model` 为 `provider/model`。
3. 记录响应头中的 `x-request-id`（若提供）并联系支持。
